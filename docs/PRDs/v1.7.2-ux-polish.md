# v1.7.2 — user-message background · compaction heartbeat

> **Ships as v1.7.2** — the next ascending tag. The still-unbuilt architecture
> doc bumped to v1.7.3. (Renumbered from the provisional v1.7.3 at build time per
> ascending-tags, [[feedback_versioning]].)

## Problem

Two rough edges from live use:

1. **User messages are hard to spot in dense scrollback.** They render with
   `USER_STYLE = "cyan"` (foreground only) — visually similar to assistant/system
   text, so the eye can't quickly find "where did I last say something".
2. **`/compact` looks dead.** `ContextManager.compact()` runs `smart_compact` (an
   **LLM call**, seconds) while the CM is `IDLE`, and the status-bar heartbeat
   returns `[]` when `IDLE` — so the bar goes empty during compaction and the user
   can't tell it's working vs hung.

## Goal

1. Give **real user prompts a non-black background** so they're instantly
   identifiable in the transcript.
2. Show a **live heartbeat while compacting** (manual and automatic), so it reads
   as working, not frozen.

## Non-goals

- **Not a full-width background bar** for user messages (see the split — fragile
  in append-only scrollback). A text-span background only.
- **Not restyling assistant/system/tool turns** — only the real user prompt.
- **Not a new `State.COMPACTING`** — a flag, mirroring `_advisor_busy`.
- **Not changing compaction behavior** (what gets dropped/summarized) — liveness
  surface only.

## Claude Code reference

See `docs/splits/v1.7.2-ux-polish.html` (2 split points, agent-decided under the
user's "decide yourself" delegation, CC-surveyed + advisor-reviewed).

### Split point: user-message visual treatment
- **CC**: styles user/command turns with a foreground color + prefix (Ink
  `Text color=…`, e.g. `components/messages/UserCommandMessage.tsx`
  `color="subtle"`), **not** a background.
- **Decision**: User-directed different — **a text-span background** via a fg/bg
  pair on `USER_STYLE` (the user asked for "bkg color different than black"). The
  full-width-padded bar was rejected: append-only scrollback (v0.6.8) means padded
  lines don't reflow on resize → ragged colored rectangles ([[feedback_ui_hints]]
  "ugly"); span bg looks identical for the common 1–2 line prompt (YAGNI). Real
  user prompts only.

### Split point: compaction liveness
- **CC**: `setSDKStatus('compacting')` (`commands/compact/compact.ts`) + a
  "Compacting conversation" spinner (`screens/REPL.tsx`) — compaction is a
  recognized busy/liveness state.
- **Decision**: Follow CC — a CM **`_compacting` flag** (not a new `State`; mirrors
  the `_advisor_busy` precedent, no cancel/guard audit, and the serial input loop
  makes re-entrancy moot). It must count as busy in **three** places or the
  heartbeat ships frozen: the `_heartbeat_ticker` run/blink condition,
  `format_heartbeat` (early-return + a "Compacting" actor branch), and
  `_above_input` passing the signal. Set on **both** compact paths.

## Design

1. **User background** (`terminal_chat`): change `USER_STYLE` from `"cyan"` to a
   readable fg/bg **pair** — e.g. `"cyan on grey19"` (exact pair tweakable;
   constraint: clearly ≠ black, 256-color-safe, readable, not clashy). It flows
   through `ChatView.print_user_now` (`Text(content, style=USER_STYLE)`); the
   `_render_record` `role=="user" → print_user` branch already sits *below* the
   task-reminder / goal / advisor / summary-marker special-cases, so only real
   prompts get it — confirm and add a test asserting injected user-role turns are
   unaffected.
2. **Compaction heartbeat**:
   - **`ContextManager`** gains `_compacting: bool` (+ reuse `phase_started_at` for
     elapsed). Set `True` around the `smart_compact` call in **`compact()`** and in
     **`_maybe_auto_compact()`** (cleared in `finally`).
   - **`format_heartbeat`** gains a `compacting: bool` param: when set, it does not
     early-return on `IDLE`, and renders the glyph + a `"Compacting"` actor (with
     `phase_started_at` elapsed).
   - **`_heartbeat_ticker`** (and `heartbeat_loop`) treat compacting as busy in
     their run/blink condition (today `state != IDLE`) so the glyph actually
     animates and elapsed ticks during the compact await.
   - **`_above_input`** passes `compacting=self.ctx._compacting` to
     `format_heartbeat`.

## Acceptance

1. A real user prompt renders with the chosen fg/bg pair (background ≠ black);
   assistant/system/tool turns are unchanged; injected user-role turns
   (task/goal/advisor/summary) do **not** get the user background (unit test).
2. `format_heartbeat(..., compacting=True)` returns a non-empty "Compacting"
   readout **even when `state == IDLE`** (unit).
3. The `_heartbeat_ticker` / `heartbeat_loop` busy-condition treats compacting as
   busy (unit on the loop condition) — so the glyph blinks + elapsed ticks during
   compaction.
4. **Live `/compact` watch**: during a real `/compact` (which runs `smart_compact`),
   the status bar shows an animating "Compacting · Ns" heartbeat, then clears — not
   an empty/frozen bar. (The unit tests prove the pieces; this proves they're wired
   so it isn't present-but-frozen.)
5. Both compaction paths set the flag (manual `compact()` + auto
   `_maybe_auto_compact()`); `ruff` clean; full `pytest` green.
