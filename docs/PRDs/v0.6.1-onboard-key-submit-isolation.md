# v0.6.1 - isolate onboarding api_key submits from chat

## Problem

When onboarding is opened from the main chat app with `/onboard`,
typing an `api_key` and pressing Enter can also submit that same key to
the main chat composer. The key then appears as a user message on the
main app screen and can be sent to the language model.

This is both confusing and unsafe: secrets entered in onboarding must
never be rendered in the chat transcript or passed to the assistant.

## Goal

- `api_key` Enter inside onboarding only saves the key in onboarding.
- The main chat app only handles submissions from its own composer.
- No onboarding secret is posted to the chat log, stored in
  `agent.messages`, or sent to the model.
- Existing onboarding save, mask, focus, and YAML behavior stays intact.

## Non-goals

- Redesign onboarding.
- Change provider config or session formats.
- Change model streaming behavior.
- Add secret redaction to arbitrary user chat messages.

## Root Cause

Textual `Input.Submitted` messages bubble through the widget tree.
`OnboardScreen.on_input_submitted()` handles `KeyInput` saves, but the
main app also has an app-level `on_input_submitted()` handler. That
handler does not check whether the submitted input is the main chat
composer, so it treats an onboarding `KeyInput` submission as a normal
chat message.

## Design

1. Stop onboarding `Input.Submitted` events after the onboarding screen
   decides whether they apply to a provider key.
2. Add a source guard to `NeutrixApp.on_input_submitted()` so the chat
   handler only processes the app's own `#input` composer.
3. Add a regression test that opens onboarding from the main chat app,
   types a sentinel key, presses Enter, and asserts:
   - the key input remains masked in onboarding;
   - the chat transcript does not contain the key;
   - `agent.messages` does not contain the key;
   - the model streaming path is not called.

## Acceptance

- Reproducing the `/onboard` Enter flow saves the API key without
  posting it as a chat message.
- Main chat submissions still work from the real composer.
- Onboarding's existing tests continue to pass.
- Full test suite remains green.
