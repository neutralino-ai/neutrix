# v0.5.4 — fix api_key Enter race: commit baseline in action_submit

## Problem

After v0.5.3 the user still reports that typing in an api_key field
and pressing Enter leaves the field showing the `EMPTY ...`
placeholder. The bug isn't reproducible in `pilot.press`-based tests
but is real in some interactive terminals.

Likely cause: ordering of Textual messages around `Enter`.

1. `Input.action_submit` posts an `Input.Submitted` message.
2. Screen's `on_input_submitted` is async; it eventually calls
   `self.focus_next()`.
3. `focus_next()` shifts focus → blur fires on the Input.
4. `KeyInput.on_blur` checks `self.value != self._committed_value`
   and, if so, restores `self.value = self._committed_value`.

If the blur runs **before** `on_input_submitted` had a chance to
promote `_committed_value` (which v0.5.3 did inside the handler),
blur sees stale `_committed_value = ""` and reverts the field to
empty. Once empty, Textual renders the placeholder. End state: user
sees `EMPTY ...` despite having typed and pressed Enter.

## Goal

Make the commit of `_committed_value` happen *before* the Submitted
message is posted, so the order in which Textual dispatches the
follow-up events (blur, focus_next) cannot regress the field.

## Non-goals

- Removing the blur-revert behaviour (we still want Tab/Up/Down
  without Enter to restore).
- Disabling the `on_focus` clear-on-focus behaviour.
- Adding latency or sleeps.

## Design

Override `KeyInput.action_submit` so the buffer is promoted to
`_committed_value` before the Submitted message is posted:

```python
async def action_submit(self) -> None:
    buf = self.value.strip()
    if buf:
        self._committed_value = buf
    self.post_message(self.Submitted(self, self.value, None))
```

Then in `OnboardScreen.on_input_submitted` we no longer need to
set `_committed_value` ourselves — we just trust it's current.

Blur ordering becomes irrelevant: any blur firing after a typed
Enter sees `value == _committed_value` and does nothing.

## Acceptance

- `test_real_keystroke_typing_enter_persists` passes: typing each
  char via `pilot.press(ch)` and pressing Enter leaves the value
  intact and the YAML updated.
- `test_action_submit_commits_baseline_before_message` passes:
  invoking `action_submit()` directly promotes `_committed_value`,
  and a subsequent blur does not revert.
- `test_typed_enter_value_renders_dots_not_placeholder` passes:
  `EMPTY` is not in the rendered output after a typed Enter.
- All previously-green tests stay green (45 total).
- `neutrix --version`: `0.5.4`.

## Out of scope

- A new "draft state" model for the Input that explicitly tracks
  pending edits.
- A different visual affordance for "field is showing empty buffer,
  but has a committed value behind it."
- Restructuring the on_blur logic.
