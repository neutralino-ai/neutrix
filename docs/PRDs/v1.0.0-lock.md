# v1.0.0 — the lock

> **Status: finalized — the v1.0 release.** Cut autonomously under a delegating
> `/goal` ("implement all PRD till v1.0; you make decisions, including whether
> or not to push"). This is a **milestone/version lock**, not a behavior-change
> feature — so it has no `docs/splits/` survey (the `prd-compare-with-claude`
> rule scopes splits to behavior-change features with a CC analog; a SemVer lock
> is neither).

## Problem

The v0.10.x band closed the two arcs the roadmap set for v1.0: agentic depth
(subagents + a judging Advisor) and the supporting machinery (streaming,
visibility parity, renderer purity, smart compaction). The project has been
pre-1.0, signalling "API/behavior may shift." With the planned surface
delivered and green, it's time to declare the v1.0 contract: a TUI,
Claude-Code-shaped agentic client whose core behaviors are stable.

## Goal

Tag **v1.0.0** on the cumulative surface shipped across v0.10.0–v0.10.5
(on top of the v0.9.x lifecycle/cancellation/heartbeat arc), declaring the
following stable for the 1.x line:

- **Conversation loop** — `ContextManager` state machine; streaming with
  cancel-as-steer (keep-partial-on-cancel); the append-only terminal renderer
  as a pure `ChatStore` reader.
- **Agentic depth** — the `Agent` subagent tool (fresh-context worker, returns
  final text); the Smart Advisor (third actor, turn-end task-list judgment).
- **Context budget** — smart, summary-based `/compact` with an automatic
  threshold trigger and the >1M-token hardening (token-budget cut, reactive
  prompt-too-long retry, in-message tool-result truncation).
- **Visibility parity** — every LLM-bound channel renders (folded, expand by
  re-printing); enforced by `.claude/rules/visibility-parity.md` + the invariant
  test.
- **Task tracking** — the Task tools + always-on panel + the reminder loop.
- **Config / slots** — `fast`/`strong` slots, per-slot `llm_timeout_s` and
  `max_context_tokens`; onboarding TUI.

## Non-goals (post-1.0 / 1.x)

- Web frontend; multi-session UI; parallel / recursive / specialized subagents;
  multi-advisor; user-configurable advisor prompt; per-turn cost tracking;
  live token-by-token rendering / bounded streaming preview; idle-trigger for
  the Advisor; recursive summary-of-summaries; rewind-through-a-summary;
  Esc-cancel of the compaction summary call. Each is recorded in its release's
  PRD/CHANGELOG.
- The legacy `tui.py` Textual app (standing non-goal across the v0.9–v1.0 line).

## Design

No new code. v1.0.0 is the annotated tag on the commit that completes the band,
with a CHANGELOG roll-up and the roadmap's v1.0.0 entry flipped to shipped.
`setuptools_scm` makes the tag the version, so `neutrix --version` reports
`1.0.0` after `pip install -e .`.

## Acceptance

- Full test suite green (**297 passing**) and `ruff` clean across `src/` +
  `tests/` at the tagged commit.
- v0.10.0–v0.10.5 each shipped with PRD + splits (where applicable) + CHANGELOG
  + annotated tag, in ascending numeric order (no backward tag).
- CHANGELOG has a `## [v1.0.0]` roll-up; the roadmap v1.0.0 entry is `shipped`.
- Tagged `v1.0.0` (annotated) and pushed.

## What 1.0 does NOT promise

Pre-1.0 minors carried breaking changes freely. From 1.0, behavior changes to
the stable surface above should bump appropriately and be called out in the
CHANGELOG. Internal refactors (the kind v0.10.3 did) remain free.
