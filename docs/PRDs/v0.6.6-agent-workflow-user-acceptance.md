# v0.6.6 - agent workflow user acceptance gate

## Problem

The current repository agent workflow goes directly from implementation to
CHANGELOG updates. That is too early for user-facing work because the user may
not have accepted the rendered result yet, and acceptance testing may still
need to be confirmed before a release entry is written.

## Goal

Update `AGENTS.md` so every implementation task includes a user-based
acceptance gate before the CHANGELOG step.

## Non-goals

- Change the release requirement to commit, tag, and push.
- Define a project-wide automated acceptance framework.
- Change product code.

## Design

Insert a new workflow step after implementation and before CHANGELOG updates:

1. Write or update the PRD first.
2. Develop the code.
3. Run or present the user-based acceptance test and confirm with the user
   that the result is acceptable.
4. Write or update the CHANGELOG.
5. Commit, tag, and push the changes.

The wording should make clear that CHANGELOG updates wait until the user is
OK with the result and acceptance testing can pass.

## Acceptance

- `AGENTS.md` includes a "user-based acceptance test" step before the
  CHANGELOG step.
- The CHANGELOG step remains after user acceptance.
- The commit/tag/push requirement remains present.
