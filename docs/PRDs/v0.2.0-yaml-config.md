# v0.2.0 — YAML config + fast/strong slots

## Problem

v0.1.0 picked the provider via env vars (`DEEPSEEK_API_KEY`, `GLM_API_KEY`,
`ANTHROPIC_API_KEY`, `NEUTRIX_PROVIDER`, `NEUTRIX_MODEL`) plus `--provider` /
`--model` CLI flags. Three pain points:

1. Env-var config is fiddly across shells, doesn't survive `sudo`, and leaks
   into unrelated processes. There is no single source of truth.
2. `--provider` + `--model` on every invocation is noise. In practice a user
   has one fast/cheap model and one strong/expensive model — re-typing the
   selection isn't useful.
3. The IHEP gateway (`https://aiapi.ihep.ac.cn/apiv2/`) routes to many model
   families through one (base_url, api_key). The v0.1 `Provider` dataclass
   coupled "where do I authenticate" to "what model do I want" — wrong shape.

## Goal

One YAML config at `~/.config/neutrix/config.yaml`. Two named slots — `fast`
and `strong` — bound to (provider, model) pairs. Switch in-TUI with `/fast`
and `/strong`. No env vars, no provider/model CLI flags.

## Non-goals

- More slots than fast/strong (e.g. `medium`, `coding`). Two is enough.
- Live config reload — restart `neutrix` to pick up edits.
- Schema validation library (pydantic) — manual checks suffice.
- Verifying that the model name exists on the provider — let the API error.
- Multi-user config / config layering.

## Design

### Config file (template written on first run)

```yaml
providers:
  ihep:
    base_url: https://aiapi.ihep.ac.cn/apiv2/
    api_key: ""        # paste your IHEP key

  deepseek:
    base_url: https://api.deepseek.com
    api_key: ""

  glm:
    base_url: https://open.bigmodel.cn/api/paas/v4/
    api_key: ""

fast:
  provider: ihep
  model: anthropic/claude-haiku-4-5

strong:
  provider: ihep
  model: anthropic/claude-opus-4-7
```

### Module changes

- **`config.py`** — `Slot` + `Config` dataclasses; `load_config()`;
  `bootstrap_config()` writes template on first run. No env reads.
- **`agent.py`** — `Agent` takes a `Slot`. `switch(slot)` rebuilds
  `AsyncOpenAI`. Drops dependency on the old `Provider` shape.
- **`cli.py`** — drops `--provider`, `--model`. Keeps `--load`,
  `--no-tools`, `--no-markdown`, `--version`.
- **`tui.py`** — `/fast` / `/strong` switch via `Config.slot(name)`;
  `/model` is now informational only; status bar shows slot name + model.
- **`pyproject.toml`** — drops `python-dotenv`, adds `pyyaml`.
- **`.env.example`** — removed.

### First-run behavior

- `~/.config/neutrix/config.yaml` absent → write template, print path,
  exit 0 with a friendly message.
- Active slot's provider has empty `api_key` → exit 1 with the path to edit.

## Acceptance

- `neutrix` on a clean machine (no `~/.config/neutrix/config.yaml`):
  writes the template, prints the path, exits 0.
- `neutrix` after user fills an `api_key`: TUI opens with the `fast` slot
  active; status bar shows `fast · ihep · anthropic/claude-haiku-4-5`.
- `/strong` in the TUI: switches slot, status bar updates, next message
  goes to the strong model.
- `pytest`: smoke tests pass (config + tools + session, no network).
- `ruff check`: clean.
- `neutrix --version`: reports `0.2.0`.
- Editable install picks up the new tag immediately.

## Breaking changes (callouts for the changelog)

- All env-var configuration is removed (`*_API_KEY`, `NEUTRIX_PROVIDER`,
  `NEUTRIX_MODEL`).
- `python-dotenv` is no longer a dependency; `.env` files are not read.
- `--provider` / `--model` CLI flags are removed.
- Direct Anthropic provider entry is dropped from defaults — Claude is
  reached via the IHEP gateway. Users can still add an `anthropic:` entry
  in their YAML manually.

Pre-1.0, breaking changes bump the minor per `.claude/rules/release-workflow.md`.

## Out of scope (deferred)

- Pre-push git hook enforcing CHANGELOG + annotated tag on a `vX.Y.Z`
  release commit.
- Slot names beyond `fast` / `strong`.
- A built-in `init` command — the auto-bootstrap on first run is enough.
