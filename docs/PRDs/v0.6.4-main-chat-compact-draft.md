# v0.6.4 - compact chat blocks and multiline draft

## Problem

The main chat UI still has four rough edges:

- the user draft block does not visually match committed user message blocks;
- the draft input shows an inner focus/background surface, making the draft
  look like a block inside another block;
- the system prompt is over-framed with a border/title, wasting vertical and
  horizontal space for content that only needs low-key visibility;
- vertical spacing between blocks and between text and block edges is too
  loose for a terminal chat;
- some OpenAI-compatible providers can finish with assistant content without
  emitting token deltas, so the conversation state receives the reply but the
  transcript does not show it.

The current single-line draft also leaves users without an obvious way to
write a multi-line prompt.

## Goal

Polish the main chat so the visible model context reads as one compact block
list:

- committed user messages and the editable user draft use the same outer
  block styling;
- the system prompt renders as warning-colored text on the screen background,
  without a border or label;
- the draft has one background surface, not a nested input background;
- block spacing keeps a small separation without changing the saved-session
  format;
- assistant replies are shown even when the provider sends only the final
  assistant message;
- users can insert a newline in the draft while keeping Enter as submit.

## Non-goals

- Add transcript editing, branching, prompt history, or message compaction.
- Redesign onboarding.
- Change provider configuration or saved-session schema.
- Add a complex editor abstraction beyond the current draft widget.

## Design

1. Replace the single-line chat `Input` with a small `DraftInput` wrapper
   around Textual `TextArea`.
2. Keep Enter as submit and support newline insertion with Shift+Enter,
   Alt+Enter, or Ctrl+J.
3. Style the draft `TextArea` as compact and borderless, with the same
   background as the user block and no focused cursor-line background.
4. Remove outer chat container padding, keep a tiny one-row gap between
   adjacent transcript blocks, and remove inner block padding so text sits
   closer to the block edge.
5. Render the system prompt with the system role class but no border title
   and no border/background wrapper.
6. Track whether assistant token deltas were rendered during a turn. If the
   final assistant message contains content but no token delta was shown,
   emit a UI event for that final content and render it as an LLM block.

## Acceptance

- Launching the app shows compact full-width blocks with a tiny visible gap
  between model-visible blocks.
- Text inside message and draft blocks sits directly against the block
  content edge, without an extra padding column.
- The committed user block and the draft block share the same outer role
  class, border title, and background surface.
- The system prompt appears as warning-colored text on the screen background
  with no border title.
- The draft editor has no second inner background when focused.
- Pressing Enter submits the draft.
- Pressing Shift+Enter, Alt+Enter, or Ctrl+J inserts a newline into the draft.
- A provider response that yields only a final assistant message still appears
  as an LLM block in the transcript.
- Existing onboarding input submissions are not treated as main chat
  submissions.
