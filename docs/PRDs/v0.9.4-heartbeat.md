# v0.9.4 — heartbeat liveness indicator

> **Status: Phase-1 re-locked after Phase-2 reopen.** First
> implementation shipped at the Phase-3 gate with a 720 ms cycle
> (~83 BPM); user rejected as "beating too fast." Re-survey
> surfaced three new split points (cycle period, smoothing
> method, randomness) — see
> `docs/splits/v0.9.4-heartbeat.html` for the turn-by-turn record
> and the `## Claude Code reference` section below for the
> embedded summary. **New cycle:** 40-frame truecolor gradient
> at ~100 ms/tick with ±10% jitter = **4.0 s per breath** (~15 BPM,
> middle of the human resting-calm respiratory range). Phase-1
> decisions that diverge from the agent's recommendation or from
> CC: #2 glyph (Steve-Jobs-mode breathing dot, user-directed),
> #9 position (top of message-area, user-directed), #12 cycle
> period (4 s, user-directed), #13 smoothing (truecolor gradient,
> agent recommendation accepted), #14 randomness (±10% jitter,
> agent recommendation accepted). Three other splits diverged from
> CC on purpose: #1 phase set, #7 cancel transition, #4 label
> policy.

## Problem

After v0.9.3:

- Streaming is **off** (`OpenAIChatLLM.stream_response` runs with
  `stream=False`). A 5–30 s LLM round renders nothing visible to
  the user for the entire duration.
- The `ContextManager` state machine has explicit busy states
  (`AWAITING_LLM`, `AWAITING_EXECUTOR`) and a transient
  `CANCELLING` state, but no UI surface reveals them.
- A long `run_shell` (e.g. `sleep 10`) and a slow LLM call are
  indistinguishable from a hung process. The user has no signal
  that the system is alive.

v0.9.6 will re-enable streaming and give the user incremental
output during `AWAITING_LLM`. But:

1. v0.9.6 is a bigger refactor and ships after v0.9.4.
2. Even post-streaming, `AWAITING_EXECUTOR` (slow tool runs) and
   `CANCELLING` (cancel broadcast in progress) remain silent.

The heartbeat is the cheap, universal liveness surface that
covers both gaps.

## Goal

Add a single-line pulse above the input cursor that:

- Pulses with a rotating glyph while the
  `ContextManager` state is `AWAITING_LLM`,
  `AWAITING_EXECUTOR`, or `CANCELLING`.
- Labels the current state:
  - `"✻ LLM"` while `AWAITING_LLM`.
  - `"✻ tool: {name}"` while `AWAITING_EXECUTOR` (tool name
    from the head of `store.pending_tool_calls`).
  - `"✻ cancelling…"` while `CANCELLING`.
- Disappears (renders nothing) on `IDLE`.

Reads `ContextManager.state` and `store.pending_tool_calls`;
adds no new store fields. Pure-renderer change.

## Non-goals

- **No streaming.** v0.9.6 re-enables `stream=True` separately.
- **No new event kinds.** v0.9.0's `AgentEvent` union is enough;
  the heartbeat ticks on a wall clock and reads state, it does
  not subscribe to events.
- **No `patch_stdout`-based rendering.** v0.7.0's lesson stands.
  The heartbeat lives in the same message-area supplier the
  task panel and queue already use.
- **No configurable glyph, cadence, verb list, or
  reduced-motion fallback.** One good default. Defer customization
  to a focused v1.x patch if requested.
- **No latency / token-rate / elapsed-time display.** Liveness
  signal, not a meter. Defer to v1.x.
- **No multi-row rendering for parallel tools.** v0.9.x has no
  parallel tool dispatch; revisit if parallelism ever ships.

## Design

Locked per the 14 splits (11 original + 3 Phase-2 reopen):

- **Glyph set** (split #2, user-directed): a single `●` whose
  brightness breathes via a truecolor gradient. No shape rotation.
  Diverges from CC's ✻-family — Steve-Jobs-mode minimalism.
- **Cadence** (split #3, revised Phase 2): **~100 ms per frame
  with ±10% jitter**. Couples to split #12 (period) and #14
  (jitter).
- **Cycle period** (split #12, user-directed Phase 2): **4.0 s per
  full breath** (~15 BPM, slow-calm end of the human resting band).
  Achieved by 40 frames × ~100 ms tick.
- **Smoothing** (split #13, Phase 2): **truecolor gradient.** 40
  hex gray shades interpolated along a raised-cosine curve from
  trough `rgb(60,60,60)` to peak `rgb(255,255,255)` and back.
  Degrades to ANSI on terminals without truecolor (prompt_toolkit
  color-depth fallback).
- **Randomness** (split #14, Phase 2): **±10% tick jitter** on the
  sleep interval; no phase offset; no brightness jitter. Trivial
  code, desyncs from CPU clocks for an organic feel.
- **Phase set** (split #1): two busy labels
  (`"LLM"` / `"tool: {name}"`) + one transient
  (`"cancelling…"`). Driven by `ContextManager.state`, not by
  per-content-block streaming events.
- **Label policy** (split #4): static phase names; no random
  verb list.
- **Tool-name detail** (split #5): registered tool name only,
  no args.
- **Multi-pending** (split #6): pick head of
  `store.pending_tool_calls`.
- **Cancel transition** (split #7): render `"cancelling…"` while
  in `CANCELLING` (diverges from CC; aligns with neutrix's
  state-machine abstraction).
- **Elapsed / token overlay** (split #8): none.
- **Position** (split #9, user-directed): heartbeat at the
  **top** of the message-area, above the task panel.
- **Idle debounce** (split #10): none.
- **Reduced motion** (split #11): none.

Wiring:

```python
# src/neutrix/terminal_chat.py — module-level constants
HEARTBEAT_GLYPH = "●"
HEARTBEAT_CYCLE_FRAMES = 40              # 40 × ~100 ms ≈ 4.0 s breath
HEARTBEAT_TICK_MS = 100                  # ±10% jitter applied per frame
HEARTBEAT_JITTER_RATIO = 0.10
HEARTBEAT_TROUGH_RGB = (60, 60, 60)      # darkest gray (dim phase)
HEARTBEAT_PEAK_RGB = (255, 255, 255)     # brightest (peak phase)
HEARTBEAT_LABEL_STYLE = "fg:ansiwhite bold"

# Precomputed at module load: 40-entry tuple of "fg:#xxxxxx" strings
# along a (1 - cos(2π · frame / 40)) / 2 brightness curve, so the
# eye reads a smooth breath (peak at frame 20, trough at frames
# 0 and 40 = 0).
HEARTBEAT_BRIGHTNESS_CYCLE: tuple[str, ...] = _build_brightness_cycle()


def format_heartbeat(
    state: State,
    store: ChatStore,
    tick: int,
) -> list[tuple[str, str]]:
    """Return prompt_toolkit StyleAndTextTuples; [] when state == IDLE.

    Glyph: single ● with brightness cycling via HEARTBEAT_BRIGHTNESS_CYCLE
    indexed by (tick % HEARTBEAT_CYCLE_FRAMES). Label is always
    rendered bright; only the glyph fades in/out.
    """
```

- `State` is the enum exported by `src/neutrix/context_manager.py`
  (`IDLE`, `AWAITING_LLM`, `AWAITING_EXECUTOR`, `CANCELLING`). Return
  type matches prompt_toolkit's `StyleAndTextTuples = list[tuple[str, str]]`.
- `TerminalChat._above_input` stack supplier order (per split #9 —
  heartbeat at top):
  1. `format_heartbeat(self.ctx.state, self.store, self._heartbeat_tick)`
  2. `format_task_panel(self.store.tasks)`
  3. queued user messages
- New `async def _heartbeat_ticker(self) -> None` on `TerminalChat`:
  loops while not cancelled; awaits
  `asyncio.sleep(HEARTBEAT_TICK_MS / 1000 * uniform(1 - JITTER_RATIO, 1 + JITTER_RATIO))`
  while `self.ctx.state in {AWAITING_LLM, AWAITING_EXECUTOR, CANCELLING}`;
  increments `self._heartbeat_tick`; calls `self._app.invalidate()`.
  When the CM transitions to `IDLE`, the ticker blocks on
  `await self.store.changes().__anext__()` until the next state change
  wakes it. Cancelled cleanly on `TerminalChat` shutdown alongside the
  existing `_worker_loop` and `_render_watcher` tasks.
- The free helper `heartbeat_loop` takes a `sleep` callable
  (default a jitter-applying wrapper around `asyncio.sleep`) so
  tests can pass a deterministic sleep for stable timing assertions.
- `ContextManager` is NOT modified. It already owns `state`; the
  heartbeat is a pure reader of `(state, store.pending_tool_calls)`.

## Acceptance

- Manual:
  - During an LLM round (`stream=False`, post v0.9.3), the
    heartbeat shows `● LLM` at the top of the message-area; the
    `●` glyph breathes (peak→trough→peak) on a ~4.0 s cycle
    (~15 BPM); each frame's interval varies subtly (±10% jitter)
    so two consecutive breaths look slightly different; the label
    `LLM` stays bright.
  - During a deliberately-slow `run_shell` (e.g. `sleep 5`),
    the heartbeat shows `● tool: run_shell` and breathes.
  - Press Esc during either: the heartbeat briefly shows
    `● cancelling…` (visible for the duration of cancel
    broadcast — measurable ~200 ms for a tree-kill) and then
    disappears.
  - When the CM transitions to `IDLE`, the heartbeat line
    disappears within ~one tick (~100 ms).
  - On a truecolor terminal: the breath is visibly smooth —
    the eye reads continuous fading, not stepwise jumps.
  - On a non-truecolor terminal (e.g. `TERM=xterm`): the breath
    degrades to ANSI shades via prompt_toolkit's color-depth
    fallback. Stepwise but functional.
  - No visible blinking of unrelated regions (transcript, task
    panel, queue) during the pulse.
  - Stack order, top to bottom: heartbeat, task panel, queued
    user messages, input cursor. Heartbeat is the first
    above-input element the eye lands on.
- Automated:
  - `tests/test_heartbeat_format.py`:
    - `test_idle_returns_empty()` — `format_heartbeat(IDLE, …)` returns `[]`.
    - `test_awaiting_llm_label()` — fragment text contains `"LLM"`;
      glyph is `HEARTBEAT_GLYPH` (`●`).
    - `test_awaiting_executor_label_uses_tool_head()` — fixture has
      a `PendingToolCall(name="run_shell", …)` at the head of
      `store.pending_tool_calls`; fragment text contains
      `"tool: run_shell"`.
    - `test_cancelling_label()` — fragment text contains `"cancelling"`.
    - `test_brightness_cycle_is_truecolor_gradient()` — every style
      string in `HEARTBEAT_BRIGHTNESS_CYCLE` matches
      `r"^fg:#[0-9a-f]{6}$"`; the cycle has at least 20 distinct
      values (proof of smooth gradient, not 3-step ANSI).
    - `test_brightness_cycles_by_tick()` — same state, four ticks
      sampled at quarter-points of the cycle (`0`, `N/4`, `N/2`,
      `3N/4`) return four distinct style strings drawn from
      `HEARTBEAT_BRIGHTNESS_CYCLE`. Glyph character identical
      across all ticks.
    - `test_brightness_cycles_modulo_cycle_frames()` — `tick = K`
      and `tick = K + HEARTBEAT_CYCLE_FRAMES` return the same
      style string.
    - `test_label_style_is_always_bright()` — across all ticks,
      the label fragment's style does not change (only the glyph
      fragment's style does).
  - `tests/test_heartbeat_ticker.py`:
    - `test_ticker_increments_while_busy()` — fake CM in
      `AWAITING_LLM`; run `heartbeat_loop` with a no-jitter
      deterministic sleep for 400 ms; assert ≥3 on_tick calls;
      assert tick counter monotonic.
    - `test_ticker_idle_when_state_idle()` — fake CM in `IDLE`;
      run ticker for 400 ms; assert 0 on_tick calls.
    - `test_ticker_wakes_on_store_change()` — start in `IDLE`,
      transition to `AWAITING_LLM` via store mutation; assert
      first on_tick within 200 ms.
    - `test_ticker_clean_shutdown_on_cancel()` — cancel the
      ticker task; assert it exits within one tick (~150 ms)
      without raising.
    - `test_jitter_stays_within_bounds()` — call the
      jitter-applying sleep wrapper 100 times with a fixed
      nominal of 0.1 s; assert every observed nominal multiplier
      is in [0.9, 1.1] and the mean is within ±0.01 of 1.0.
- Human gate before tag.

## Also bundled in this release

A small unrelated user-reported fix rides along in the v0.9.4
tag (per the forward-only versioning rule — see
`memory/feedback_versioning.md`):

**TaskCreate prompt — full upfront planning.** During live use
the LLM tends to call `TaskCreate` for one task at a time rather
than capturing the full multi-step plan upfront. Fix is a
description-only edit to `src/neutrix/tools.py:TaskCreate`:

- Open with *"Use this tool proactively to capture the user's
  full multi-step plan as **separate tasks**, not piece by
  piece."*
- Add *"After receiving new instructions — immediately capture
  every distinct step the request implies as its own task,
  BEFORE starting work on any of them."*
- Keep the V2-aligned lifecycle clauses
  (*"Mark in_progress BEFORE beginning work"*,
  *"Always mark resolved when finished"*,
  *"Call TaskList to find your next task"*) intact.

Acceptance for the bundled fix:

- `tests/test_tools.py` cases for `TaskCreate` still pass
  (signature unchanged).
- Manual: a multi-step user request ("clean up imports, run
  ruff, then add three tests") produces 3 `TaskCreate` calls
  upfront, visible in the task panel, BEFORE the LLM starts on
  task #1.

## Claude Code reference

See `docs/splits/v0.9.4-heartbeat.html` for the full survey.

### Split point: Phase set (#1)
- **CC**: 5 phases driven by streaming content-block events —
  `requesting` / `responding` / `thinking` / `tool-input` /
  `tool-use` (`cc2/src/utils/messages.ts:2985-3020`).
- **Decision**: Alternative — 2-phase + `cancelling…`. Maps 1:1
  to the v0.9.3 `ContextManager` state machine; v0.9.x has no
  streaming or thinking blocks to justify CC's content-block
  granularity.

### Split point: Glyph set (#2)
- **CC**: 6-char `['·', '✢', '✳', '✶', '✻', '✽']` cycled
  forward+backward, 120 ms/frame
  (`cc2/src/components/Spinner/utils.ts:4-10`,
  `SpinnerGlyph.tsx:7`).
- **Decision**: User-directed different — single `●` with
  brightness pulse over 6 frames, no shape rotation. User:
  *"suggest more. what would Steven Jobs do?"* Steve-Jobs-mode
  minimalism — one character, calmer than CC's ornamental
  cycle.

### Split point: Tick cadence (#3, reopened Phase 2)
- **CC**: 120 ms per frame, ~8 Hz
  (`SpinnerAnimationRow.tsx:131`).
- **Decision (Phase 1, since revised)**: Follow CC — 120 ms.
- **Decision (revised Phase 2)**: Alternative — 100 ms nominal,
  ±10% jitter. Coupled to split #12 (cycle period) and split #14
  (randomness): 40 frames × 100 ms = 4.0 s breathing cycle.
  Diverges from CC.

### Split point: Label policy (#4)
- **CC**: random verb from 200+ list, sampled once per mount,
  stable per turn (`constants/spinnerVerbs.ts`,
  `Spinner.tsx:166-171`).
- **Decision**: Alternative — static phase names
  (`"LLM"` / `"tool: {name}"` / `"cancelling…"`). Phase IS the
  information; doubles as a debug affordance for inspecting CM
  state. Random-verb personality is a v1.x polish item.

### Split point: Tool-name detail (#5)
- **CC**: custom `activeForm` if a task sets it (e.g. "Running
  bash…"); args not echoed inline.
- **Decision**: Alternative — registered tool name only. Args
  already appear on the preceding `-> tool_use   {name} {args}`
  transcript line (v0.8.2 colored keyword); the heartbeat
  duplicates nothing and fits narrow terminals.

### Split point: Multi-pending tool (#6)
- **CC**: multiple pending tools render as multi-row in the
  tasks list; spinner labels one phase at a time.
- **Decision**: Alternative — first-only (head of
  `store.pending_tool_calls`). v0.9.x has no parallel tool
  dispatch; multi-row / count splits belong to the release that
  adds parallelism.

### Split point: Cancel transition (#7)
- **CC**: no transition; `resetLoadingState()` hides the
  spinner the moment Esc fires (`REPL.tsx:2106`).
- **Decision**: Alternative — render `cancelling…` while CM is
  in `CANCELLING`. Aligns user-view with the project's core
  state-machine abstraction. neutrix's cancel window is
  measurable (~200 ms tree-kill); CC's instant cancel doesn't
  have the same window to surface.

### Split point: Elapsed / token overlay (#8)
- **CC**: elapsed seconds during thinking (min 2 s display);
  token count after 30 s; both update on a 50 ms clock
  (`SpinnerAnimationRow.tsx:19`).
- **Decision**: Alternative — none. Heartbeat is a liveness
  signal, not a meter. Elapsed / token overlay belongs to the
  v1.x `/status` cost-tracking patch.

### Split point: Position (#9)
- **CC**: two-row footprint above the input prompt
  (`cc2/src/screens/REPL.tsx:4587`).
- **Decision**: User-directed different — heartbeat at the
  **top** of the message-area, above the task panel. Stack:
  heartbeat → task panel → queued messages → input cursor. User
  picked option 2 (above task panel) over agent option 1
  (between task panel and queue); rationale: the heartbeat is
  the most immediate signal and should be the first thing the
  eye lands on when scanning.

### Split point: Idle-hide debounce (#10)
- **CC**: not relevant — streaming fills the gap between
  consecutive busy phases.
- **Decision**: Alternative — no debounce. CM transitions
  `AWAITING_EXECUTOR` → `AWAITING_LLM` directly in the agent
  loop (no intervening `IDLE`), so the blink doesn't happen on
  the common path. If v0.9.6 streaming reveals an edge case,
  the debounce becomes a small follow-up patch.

### Split point: Reduced-motion fallback (#11)
- **CC**: single `●` on a 2 s cycle when reduced motion is
  enabled (OS-level a11y setting).
- **Decision**: No CC analog — defer. The breathing-dot from
  #2 is already calm; the reduced-motion concern that drives
  CC's fallback is less acute. Add a YAML
  `ui.reduced_motion` flag in a v1.x patch if a user requests
  it.

### Split point: Cycle period (#12, Phase-2 reopen)
- **CC**: 12 frames × 120 ms = **1.44 s per cycle** (~42 BPM)
  in normal mode; **2.0 s per cycle** in reduced-motion mode
  (`cc2/src/components/Spinner/SpinnerGlyph.tsx:7,9`). Both
  faster than human resting-calm breathing.
- **Physiological reference**: resting adults breathe at 12–20
  breaths/min (3–5 s per cycle); coherent breathing for HRV is
  5–6 breaths/min (10–12 s).
- **Decision**: User-directed different — **4.0 s per breath**
  (~15 BPM, slow-calm end of resting band). User picked the
  4 s tier over the agent's 3 s middle-tier recommendation,
  after the first implementation (720 ms = 83 BPM) was rejected
  as "beating too fast." Diverges from CC.

### Split point: Smoothing method (#13, Phase-2 reopen)
- **CC**: not applicable in the brightness channel — CC uses
  shape change (✻-family) for smoothness, not brightness
  modulation.
- **Decision**: Alternative — **truecolor gradient** through
  40 hex gray shades along a raised-cosine curve from
  `rgb(60,60,60)` to `rgb(255,255,255)`. Degrades to ANSI on
  terminals without truecolor via prompt_toolkit's color-depth
  fallback. Required by the slow 4-second cycle: with only 3
  ANSI shades the breath would *step* visibly every ~1 s
  instead of flowing.

### Split point: Randomness (#14, Phase-2 reopen)
- **CC**: strict periodic, no jitter
  (`SpinnerAnimationRow.tsx:131` — `Math.floor(time / 120)`).
- **Decision**: Alternative — **±10% jitter on tick interval**,
  no phase offset, no brightness jitter. Each frame sleeps for
  `HEARTBEAT_TICK_MS / 1000 * uniform(0.9, 1.1)`. Cheapest
  organic feel; desyncs the cycle from CPU-clock multiples
  without looking "glitched." Diverges from CC.
