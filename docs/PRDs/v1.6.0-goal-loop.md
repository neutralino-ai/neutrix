# v1.6.0 — native `/goal` autonomous loop

> **Status: proposed (Phase-1 accept gate pending).** Splits:
> [docs/splits/v1.6.0-goal-loop.html](../splits/v1.6.0-goal-loop.html). First of
> the pre-v2 band (v1.6 → v1.7 → v1.8).

## Problem

neutrix runs one turn per user message: after the assistant's final text the loop
goes IDLE and waits for input. There is no way to set a multi-step objective and
let the agent grind it unattended — the core need for an OMILREC speedup run
("optimize this until the benchmark passes"), and the headline practical-autonomy
feature on the v2 path.

## Goal

- **`/goal <text>`** sets an objective and the agent keeps taking turns toward it
  **without re-prompting**, until it declares done or a step cap hits.
- **Deterministic termination**: a distinctive `<<GOAL_COMPLETE>>` sentinel (soft
  exit) **plus** a hard step cap (the real guarantee — an autonomous loop must
  never spin forever).
- **The human always wins**: Esc or any typed message reasserts manual control.
- The ContextManager stays a pure single-turn driver; the loop lives chat-side.

## Non-goals

- Hooks (cut from v2) — neutrix self-contains continue + done-detection.
- A `CompleteGoal` tool, Advisor-judged completion, or cost-based stopping (later
  hardening if the sentinel proves flaky).
- Persisting/resuming a running goal across `--continue` (in-memory only; split #6).
- Nested/parallel goals.

## Claude Code reference

See [docs/splits/v1.6.0-goal-loop.html](../splits/v1.6.0-goal-loop.html) (7 split
points). CC ends a turn when the model emits no `tool_use` (`query.ts:1062`); a
**Stop hook** can block turn-end and re-inject a continue-reason (`stopHooks.ts:268`,
re-entry `query.ts:1278`); bounded by `maxTurns` (`query.ts:1704`); Esc aborts via
the signal. neutrix has no hooks, so it reuses its own pieces.

- **Loop location** — Alternative: the chat `_worker_loop`, not the CM (CM stays
  single-turn; the v3 headless loop reuses the CM, not this policy).
- **Done-detection** — Alternative: `<<GOAL_COMPLETE>>` sentinel on the final
  non-empty line of **committed assistant text only**, + the cap as the hard
  guarantee. (No new tool / no per-turn extra LLM call.)
- **Continuation** — Follow CC (neutrix-shaped): a labelled `<system-reminder>`
  turn (reuses v0.8 reminder injection), not a fake user message.
- **Cap** — Follow CC: `GOAL_MAX_STEPS` (25); graceful stop + notice, not a crash.
- **Interrupt/steer** — Esc cancels the turn AND clears the goal; a typed message
  clears the goal and is handled normally.
- **Resumability** — Alternative: in-memory only (don't auto-resume into an
  unattended loop).
- **Command surface** — No CC analog: `/goal <text>` set+start, `/goal` show,
  `/goal clear` stop.

## Design

### `src/neutrix/context_manager.py`
- New method `continue_goal(goal_text: str)` (mirrors `inject_advisor_message`'s
  single-mutator discipline): append one `{"role": "user", "content":
  "<system-reminder>Continue toward the active goal: {goal}. Work autonomously;
  do not ask the user. When the goal is FULLY achieved, end your reply with a
  final line `<<GOAL_COMPLETE>>`.</system-reminder>"}` record, then run a turn
  (`await self._drive()` via the same path a user turn uses). Returns when the
  turn completes/cancels.
- `goal_completed(text) -> bool` helper (or inline): the committed assistant text's
  final non-empty line, stripped, case-insensitive, equals `<<GOAL_COMPLETE>>`.
  (Reads `store` / the last assistant `MessageRecord`; never tool results.)

### `src/neutrix/terminal_chat.py`
- Fields: `_goal: str | None = None`, `_goal_step: int = 0`; const `GOAL_MAX_STEPS = 25`.
- `_run_command`: `/goal <text>` → set `_goal`, reset `_goal_step`, kick the loop;
  `/goal` (bare) → print active goal + `step N/25`; `/goal clear` → clear + notice.
- `_worker_loop` (and/or after a normal turn): the **goal driver** — while
  `_goal` and `_goal_step < GOAL_MAX_STEPS` and not cancelled:
  - run a continuation turn (`ctx.continue_goal(self._goal)`); `_goal_step += 1`;
  - if the committed assistant text ended with `<<GOAL_COMPLETE>>` → notice
    (`✓ goal complete`), clear, break;
  - if cancelled (Esc) → clear, break;
  - if a user message is waiting in the queue → clear the goal, break (let the
    normal turn run);
  - on cap → notice (`↯ goal paused after 25 steps — /goal <text> to resume`),
    clear.
- Esc path: the existing cancel clears `_goal` too (a real human stop).
- Status line (`format_heartbeat`/`_above_input`): while `_goal`, show
  `◎ goal · step N/25` (reuses the v1.5.0 live region).

### `src/neutrix/cli.py`
- (No goal seeding from the CLI in v1.6 — interactive `/goal` only.)

## Acceptance

- **Automated** (termination paths are the priority — an autonomous loop's failure
  mode is non-termination):
  - sentinel → loop stops + goal cleared after the model emits `<<GOAL_COMPLETE>>`;
  - cap → after `GOAL_MAX_STEPS` continuations with no sentinel, graceful stop +
    clear + notice (drive a fake LLM that never emits the sentinel; assert exactly
    25 steps then stop);
  - Esc mid-loop → cancel + clear (no further continuations);
  - typed message mid-loop → goal cleared, the message handled as a normal turn;
  - sentinel false-positive guard: the token inside a tool result or a non-final
    line does NOT stop the loop;
  - `continue_goal` appends exactly one `<system-reminder>` user record per step
    and the CM remains the only mutator.
  - Full suite green; ruff clean.
- **Manual**: `/goal add a docstring to each public fn in X and run pytest` →
  the agent iterates across turns; `/goal` shows progress; `/goal clear` stops;
  Esc stops; the status line shows `◎ goal · step N/25`.
- Ship `v1.6.0` (built + committed + tagged **locally**; pushed only at the band's
  end push-gate, per the active-goal plan).

## Out of scope

- Goal persistence/resume, `CompleteGoal` tool, Advisor-judged completion, CLI
  `--goal` seeding, nested goals.
