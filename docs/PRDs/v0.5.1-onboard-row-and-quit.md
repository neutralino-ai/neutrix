# v0.5.1 — onboarding polish: row layout, hard Ctrl+C, auto-save, non-blocking verify, inline messages

## Problem

Five issues with v0.5.0 onboarding the user flagged:

1. **api_key row layout overflows.** Textual's `Input` ships with a
   3-line `border: tall $background` default. The v0.5.0 CSS pinned
   `.key-row` to `height: 1`, so the Input's bottom border bleeds onto
   the row below inside the provider card — the visible "overlapped
   bottom line."
2. **Ctrl+C×2 only dismisses the screen.** In v0.5.0 both `q` and
   the second `Ctrl+C` call `self.dismiss(False)`. User expects the
   Ctrl+C escape hatch to kill the whole app (chat included when
   reached via `/onboard`), not return to chat. `q` should remain the
   soft "back to chat" path.
3. **Manual `s` to save is wrong mental model.** Every onboarding
   change is small and reversible (verify → status, f/g → slot, key
   → committed on Enter). The user wants every change persisted to
   YAML the moment it's made — no explicit save step, no risk of
   losing state because they forgot `s`. Keys-in-progress (Input
   value not yet `Enter`-confirmed) should revert on blur, so an
   abandoned half-typed key doesn't accidentally land in YAML or
   stay in the Input as garbage.
4. **Verify blocks the UI.** `action_verify` is `async` but awaited
   directly from the binding handler — the screen freezes for the
   full 8 s timeout. The user can't arrow-around to focus another row,
   can't kick off a second verify, can't even Esc out. Should run in
   the background.
5. **Toast notifications float anywhere on screen.** `self.notify(...)`
   pops a transient toast in the corner. With multiple in flight
   (saved-this, focus-that, verify-failed) it looks chaotic and
   covers content. Want one dedicated message line at the bottom
   that shows the most recent feedback, colour-coded by severity.

## Goal

- Render the api_key row at `height: 1` with the Input flush inside
  the provider card.
- Make `Ctrl+C×2` `app.exit()` the entire process; `q` keeps the
  current `dismiss(True)` semantic.
- Persist on every committing user action — verify, verify-all, f, g,
  Enter on api_key. Remove the `s` "save and launch" binding.
- `q` dismisses with `True` (everything's already saved; the user is
  done).
- An api_key Input that the user typed into but never pressed `Enter`
  on reverts to its last committed value when focus leaves it.
- Verification (single + all) runs as a background worker so focus
  navigation, more verifies, key edits stay responsive while a verify
  is in flight.
- All transient feedback (saved, focus-warning, verify-failed, ctrl+c
  pending) goes to a single `#message` line above the footer,
  colour-coded; no floating toasts.

## Non-goals

- Restyling the rest of the card beyond the row fix.
- A modal "are you sure" dialog instead of the Ctrl+C toast.
- Configurable Ctrl+C scope or auto-save toggle.
- Visual snapshot tests in this patch.
- Live YAML-watch reload (still restart needed for external edits).

## Design

### 1. Borderless single-line api_key Input

```css
ProviderSection > .key-row {
    height: 1;
    margin: 0 0 1 0;
}
ProviderSection > .key-row > .api-key {
    border: none;
    background: $surface;
    padding: 0 1;
    height: 1;
    min-height: 1;
    width: 1fr;
    margin: 0 1;
}
ProviderSection > .key-row > .api-key:focus {
    background: $accent 30%;
}
```

Width `1fr` makes the Input grow with the card. Saved-tag width drops
to `auto`. Label keeps `width: 9`.

### 2. Ctrl+C×2 → `app.exit()`

```python
def action_confirm_quit(self) -> None:
    if self._quit_pending:
        self.app.exit()   # hard-kill the whole process
        return
    ...
```

`q` keeps calling `self.dismiss(True)` (see #3 — everything's already
saved; "back to chat" is the right action).

### 3. Auto-save on every commit

Remove the `s` binding and `action_save_and_launch`. Each path that
modifies state calls `_persist_yaml(...)`:

- `action_verify` (single) — already persists.
- `_verify_all` — already persists.
- `_assign(fast|strong)` — **add** `_persist_yaml(...)` at the end.
- `on_input_submitted` — already persists (Enter committed the key).

`q` now calls `self.dismiss(True)` unconditionally — the YAML is
already authoritative, so "saved" is always the truthful outcome.

For the Input "no-Enter = cancel" behaviour, `KeyInput` remembers its
last-committed value and restores it on blur:

```python
class KeyInput(Input):
    def __init__(self, *args, **kwargs) -> None:
        super().__init__(*args, **kwargs)
        self._committed_value = self.value

    def on_blur(self, event: events.Blur) -> None:
        if self.value != self._committed_value:
            self.value = self._committed_value
```

`OnboardScreen.on_input_submitted` updates `event.input._committed_value
= new_key` so Enter "promotes" the typed-but-not-saved buffer to the
committed baseline.

Footer label for `Ctrl+C` reads "Exit app (Ctrl+C x2)" so the
escalation is visible.

### 4. Background verification

Replace direct `await action_verify(...)` with `self.run_worker(...)`:

```python
def action_verify(self) -> None:
    if isinstance(self.focused, VerifyAllRow):
        self.run_worker(self._verify_all(self.focused.provider), exclusive=False)
        return
    row = self._focused_row()
    if row is None:
        self._notify("focus a model row first", "warning"); return
    self.run_worker(self._verify_one(row), exclusive=False)
```

Each verify owns its single row's status updates; concurrent verifies
on different rows are fine (Textual workers are async tasks, single
event loop). `exclusive=False` so multiple `v` presses pipeline.

### 5. Inline message bar

Single `Static#message` line above the `Footer`, dock-bottom. CSS:

```css
#message {
    dock: bottom;
    height: 1;
    padding: 0 2;
    background: $boost;
}
```

Helper method replaces every `self.notify(...)`:

```python
def _notify(self, text: str, severity: str = "info") -> None:
    color = {"info": "$text-muted", "warning": "$warning",
             "error": "$error", "success": "$success"}[severity]
    self.query_one("#message", Static).update(f"[{color}]{text}[/{color}]")
    if self._msg_timer is not None:
        self._msg_timer.stop()
    self._msg_timer = self.set_timer(4.0, self._clear_message)

def _clear_message(self) -> None:
    self.query_one("#message", Static).update("")
```

Persistent messages (the Ctrl+C confirm) bypass the auto-clear timer
and clear when the state changes (second tap or Esc).

## Acceptance

- The api_key row renders inside the provider card with no overflow.
- `Ctrl+C` once toasts; `Ctrl+C` twice kills the whole process
  (including the chat when launched via `/onboard`); `Esc` cancels.
- `q` returns to chat with a toast that says "onboarding saved"
  (always, since everything is auto-saved).
- Pressing `f` on a verified model writes the slot binding to YAML
  immediately; quitting without `s` keeps the binding.
- Typing a partial key into the Input and then Tab/arrow-out reverts
  the Input to the last-committed value (the YAML never sees the
  half-typed string).
- Starting a verify and immediately pressing `Down` moves focus to
  the next row while the verify is still in flight; status updates
  arrive when the API call finishes.
- All feedback (saved, focus-warning, ctrl+c confirm, verify-failed)
  appears in the single `#message` bar at the bottom — no floating
  toasts. Severity drives colour (`info` dim, `warning` yellow,
  `error` red, `success` green).
- `pytest` + ruff stay green.
- `neutrix --version` reports `0.5.1`.

## Out of scope

- Storage of "draft" key values separate from committed.
- An "Are you sure you want to discard?" prompt on blur with edits.
- Reworking other card elements (base_url, models list, etc.).
