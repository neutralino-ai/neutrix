## Problem

v0.9.3 reverted streaming (`stream=False`) to simplify the cancel rewrite. The
cost: **cancel-as-steer appends `[interrupted by user]` to a history with no
partial assistant text** — at cancel time, a non-streaming round has produced
nothing, so the steer message arrives without the prior assistant intent
visible. The "steer with prior intent in context" pitch is half-realized.
Closing the arc means re-enabling streaming with CC's "keep partial text"
semantic: the bytes that arrived before Esc stay in history.

## Goal

1. Restore `OpenAIChatLLM.stream_response` to `stream=True` (the v0.9.2 shape),
   **merged** with v0.9.3's `_ensure_tool_result_pairing` + `_outbound_prompt`
   (which v0.9.2 predates and which must be preserved).
2. `stream_response` yields `LLMEvent("token", str)` per content delta,
   accumulates `tool_calls` index-keyed, and yields one terminal
   `LLMEvent("assistant", LLMResponse)` with the full message (no speculative
   per-tool_call event — split #4).
3. `OpenAIChatLLM.stop()` closes the underlying `AsyncStream` (v0.9.2 eager
   teardown); the v0.9.3 drive-task-cancel is **kept** and remains the
   mechanism that unwinds `_drive` (split #1).
4. **Keep partial text on cancel without touching the happy path (Lane A,
   split #2):** `_call_llm` stashes the streamed content on `self`; `_do_cancel`
   commits it to `messages` + store *before* the `[interrupted by user]` marker.
   Orphan `tool_use` is repaired by the existing pairing layer — no new code.

## Non-goals

- **Live token-by-token transcript rendering (split #3).** The append-only
  scrollback (v0.6.8) can't grow a line in place; the heartbeat (v0.9.4/v0.9.8)
  remains the during-LLM liveness signal. A bounded preview in `_above_input`
  is a v1.x add. `terminal_chat.py` is unchanged this release.
- **A separate non-streaming path for tool-less providers.** One streaming path;
  plain text streams, tool_calls simply don't fire where unsupported.
- **Partial `tool_use` rendering.** A half-formed tool_call mid-stream renders
  nothing; cancel repairs via the pairing layer.
- **Cost / token-rate display.** v1.x with `/status`.

## Claude Code reference

See [docs/splits/v0.10.1-streaming-steer.html](../splits/v0.10.1-streaming-steer.html)
for the full survey (CC `claude.ts` streaming/abort) and 6 decisions.

### Split point: cancel mechanism
- **CC**: abort signal tears the stream down eagerly, partial kept.
- **Decision**: Follow CC + keep v0.9.3 — both `stop()`-closes-stream and drive-task-cancel; `CancelledError` lands in `_drive`'s existing handler.

### Split point: keep-partial location
- **Decision**: Lane A — stash on `self` during `_call_llm`, commit in `_do_cancel` before the marker; happy-path append untouched (near-zero regression).

### Split point: live token rendering
- **CC**: incremental live region + commit.
- **Decision**: Alternative — deferred to v1.x (append-only scrollback can't grow a line; heartbeat is the liveness signal). Not "infeasible," a conscious cut.

### Split point: event granularity / error handling
- **Decisions**: restore v0.9.2 exactly (`token` + terminal `assistant`, no per-tool_call event); discard partial on hard LLM error (one message), keep partial on user-cancel + timeout.

## Design

### `src/neutrix/llm.py`
Restore the v0.9.2 streaming `stream_response` (recovered from git tag v0.9.2):
`stream=True`; per-chunk `LLMEvent("token", content_delta)`; `_accumulate_tool_calls`
(index-keyed) + `_finalize_tool_calls`; terminal `LLMEvent("assistant", ...)`.
**Preserve** `_ensure_tool_result_pairing` (applied to messages before the call)
and `_outbound_prompt`/`_coerce_message`. Replace the v0.9.3 `_active_task`
mechanism with `_active_stream`; `stop()` closes the stream (idempotent,
best-effort).

### `src/neutrix/context_manager.py`
- New field `_streaming_partial: str = ""` (init=False, repr=False).
- `_call_llm`: reset `self._streaming_partial = ""` at entry; on `"token"`
  events append to it; on `"assistant"` set the return message. Returns the
  final assistant message (unchanged contract) → `_drive` appends it once,
  then clears `self._streaming_partial`.
- `_do_cancel`: before the user marker, if `self._streaming_partial` is
  non-empty, commit `{"role":"assistant","content": self._streaming_partial}`
  via `_append_assistant_message` (messages + store) and clear it. Runs for
  both `user` and `timeout` reasons (keep-partial regardless).
- `_drive`'s `except Exception` (`[LLM error]`): clear `self._streaming_partial`
  first (discard on hard error — one assistant message).

### Unchanged
`executor.py`, `store.py`, `terminal_chat.py`, key bindings. The store's
`pending_assistant_text` stays unused (reserved for the v1.x live preview).

## Acceptance

- Automated (`tests/test_streaming.py` / extend `test_context_manager.py`):
  - Stream N token deltas + a tool_call → assistant message has concatenated
    content and the index-keyed `tool_calls`; one assistant message appended.
  - Cancel mid-stream (after partial tokens) → `messages` ends with the partial
    assistant turn then `[interrupted by user]`; the next outgoing payload passes
    `_ensure_tool_result_pairing`.
  - Streamed **tool-call** round → the committed **store** record carries
    `tool_calls` (renders the `→ tool_use` line) — landmine #1.
  - LLM error after N partial tokens → exactly one assistant message (the
    `[LLM error]`), partial discarded — landmine #2.
  - Cancel with **no** tokens yet → no partial assistant message (existing
    behavior preserved).
  - `stop()` closes the active stream; idempotent when none.
- Full suite stays green; ruff clean. Ship in numeric order (tag after v0.10.0).

## Out of scope

- Live token rendering / bounded preview (v1.x).
- Cost/rate display, partial tool_use UI (v1.x).
