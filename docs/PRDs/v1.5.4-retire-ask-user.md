# v1.5.4 — retire AskUserQuestion (defer to v3)

> **Status: proposed (Phase-1 accept gate pending).** Splits:
> [docs/splits/v1.5.4-retire-ask-user.html](../splits/v1.5.4-retire-ask-user.html).
> Builds on v1.5.3 (which already decoupled permission from the ask channel).

## Problem

The v1.4.8 `AskUserQuestion` tool — the agent pausing to ask the human a
structured question — has been an instability source: the v1.4.8 *park-forever*
regression and the v1.5.1 prompt-hang both lived in its interactive-prompt path,
and it carries a bespoke **bidirectional async-generator** channel
(`Executor.dispatch_all` `yield`/`gen.asend()`) that complicates the tool loop.
After v1.5.3 nothing else uses that channel (permission no longer asks).

User-directed (2026-05-30): *"it's not stable. remove the askuser from all and
defer to v3. v2 shall not take 'askuser'. so remove ask user tool etc."*

## Goal

- **Remove AskUserQuestion and its entire interactive-input apparatus** from v2.
- **`Executor.dispatch_all` reverts to a plain event-emitting async generator** —
  no bidirectional `asend` input path — and the CM tool loop simplifies to a plain
  `async for`.
- **v2 has no human-in-the-loop tool**: the agent always proceeds autonomously
  (the same behaviour headless/subagent runs already had).
- **Defer the concept to v3** (the RL-environment line), where modelling human
  feedback in a seedable loop is in scope.

## Non-goals

- Designing the v3 interactive-feedback mechanism (just record the deferral).
- Touching permission (done in v1.5.3) or any other tool.
- Keeping the bidirectional-generator scaffold "for later" (YAGNI).

## Claude Code reference

See [docs/splits/v1.5.4-retire-ask-user.html](../splits/v1.5.4-retire-ask-user.html).
CC keeps an AskUserQuestion-style interactive tool (`canUseTool` / AskUserQuestion).

### Split point: scope of removal
- **Decision**: User-directed different — remove the **whole apparatus** (tool,
  `prompts.py`, the Executor input branch + `needs_user_input` event, the CM
  `ask_user` port + handler, `terminal_chat._ask_user` + `PROMPT_TIMEOUT_S` + the
  pending-answer machinery + indicator, the `cli.py` wiring).

### Split point: `dispatch_all` shape
- **Decision**: Alternative — revert to a **plain async generator**; the CM drives
  it with `async for` (no `asend`/send/`aclose`-on-park).

### Split point: `prompts.py`
- **Decision**: Alternative — **delete the module** + `tests/test_prompts.py`
  (used only by AskUserQuestion after v1.5.3).

### Split point: subagent tool set
- **Decision**: Follow — `subagent_tool_names()` becomes `BUILTIN_TOOLS − {Agent}`.

### Split point: defer to v3
- **Decision**: No CC analog (deferral) — note it on the roadmap's v3 line; v2
  ships with no ask-the-user tool.

## Design

### Delete
- `src/neutrix/prompts.py` (whole module); `tests/test_prompts.py`;
  `tests/test_ask_user_question.py`.

### `src/neutrix/tools.py`
- Remove the `"AskUserQuestion"` entry from `BUILTIN_TOOLS` and the
  `_ask_user_question` backstop. `subagent_tool_names()` → `… − {"Agent"}`.

### `src/neutrix/executor.py`
- Remove the `if name == "AskUserQuestion": …` branch and the `_needs_input`
  helper / `needs_user_input` `ToolEvent`. `dispatch_all` becomes a plain
  `AsyncIterator[ToolEvent]` that only `yield`s `tool_started`/`tool_finished`.
  Drop the `from neutrix.prompts import …` line.

### `src/neutrix/context_manager.py`
- Remove the `ask_user: AskUserPort | None` field, the `AskUserPort` TYPE_CHECKING
  import, and the `needs_user_input` handling in `_dispatch_tools` — the loop
  reverts to `async for event in gen` over `tool_started`/`tool_finished` (keep the
  cancel-on-`CANCELLING` check). No more `gen.asend`/`aclose`-on-park.

### `src/neutrix/terminal_chat.py`
- Remove `_ask_user`, `PROMPT_TIMEOUT_S`, the `_pending_question`/`_pending_answer`
  Future machinery, the input-loop "pending prompt claims input" branch, and the
  "answer needed" indicator. Keep normal input + Esc-cancel.

### `src/neutrix/cli.py`
- Remove `ctx.ask_user = chat._ask_user`.

### `docs/roadmap.html`
- v3 section: add the interactive-human-feedback deferral note.

### Tests
- Delete the two test files above. Update `test_terminal_chat.py` to drop the
  `_ask_user` round-trip/cancel/timeout tests. The executor/CM layering guard
  `test_executor_has_no_ask_user_field` can stay (it still holds) or move; the
  bidirectional `_drive` helper goes with `test_ask_user_question.py`.

## Acceptance

- **Automated**: full suite green; `ruff` clean. `grep -rn 'AskUserQuestion\|
  ask_user\|needs_user_input\|prompts' src/neutrix` returns nothing (save unrelated
  "prompt" words). `dispatch_all` has no `yield`-as-expression / `asend` consumer.
- **Manual**: a normal turn runs tools and completes; the LLM's tool list no longer
  contains `AskUserQuestion`; nothing prompts the user; Esc still cancels.
- Ship `v1.5.4`.

## Out of scope

- The v3 interactive-feedback design; permission (v1.5.3); other tools.
