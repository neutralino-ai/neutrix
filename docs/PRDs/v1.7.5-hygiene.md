# v1.7.5 — Project hygiene: dead code, doc consistency, README refresh

## Problem

By v1.7.4 the repo had accumulated drift that no longer matched the
shipped surface:

- **Dead config code.** `src/neutrix/config.py` still carried
  `PROVIDER_DEFAULT_MODELS`, `save_config()`, and `_serialize_provider()`
  — the write-back machinery for the **onboarding** flow that v1.7.1
  removed (`onboard.py` deleted, replaced by `bootstrap_config()` +
  "fill your key and re-run"). Nothing in production calls them; only a
  block of `tests/test_smoke.py` exercised them, so the tests kept dead
  code looking live.
- **Stale module docstring.** `store.py` claimed it "imports nothing
  from `tui`, `agent_loop`, `onboard`" — three modules that no longer
  exist in the v0.9+ architecture — while omitting the modules the
  dependency rule actually protects against today.
- **Stale README.** `README.md` still advertised the retired
  `read_file` / `write_file` / `list_dir` / `run_shell` tools and an
  `/onboard` step, predating the Claude-Code-shaped tool set
  (`Read`/`Edit`/`Write`/`Grep`/`Glob`/`Bash` + `Task*`/`Agent`),
  the config-driven `pricing:` block, and the current slash-command set.
- **Roadmap referenced dropped scope.** `docs/roadmap.html` still
  mentioned `ccusage`-compatible JSONL emission as planned scope, which
  the project has decided **not** to pursue (see `v2.0.0-lock`).

None of this is user-visible behavior, but it makes the repo read as if
abandoned scope is still live — exactly the drift the
`release-workflow` "living documents" rule exists to prevent.

## Goal

Bring the repo's dead code and prose back in lock-step with the shipped
v1.7.4 surface, with **zero behavior change**:

1. Remove the onboarding-era dead code from `config.py` and its tests.
2. Fix `store.py`'s dependency-direction docstring to name the real
   modules.
3. Rewrite `README.md` to describe the tools, config, and commands that
   actually ship today.
4. Drop the `ccusage` references from `docs/roadmap.html`.

## Non-goals

- **No behavior change.** This is cleanup; the only removed symbols have
  no production caller. `neutrix --version`, the chat loop, `/cost`,
  rewind, compaction — all unchanged.
- **No touching shipped PRDs or splits.** Historical design records stay
  as written; only living documents (README, roadmap) and dead code are
  corrected. (Per the user's instruction: "don't touch shipped PRD and
  split.")
- **No new features.** Rewind UX (v1.7.6) and the SemVer re-lock +
  GitHub Releases (v2.0.0) are separate, later releases.

## Design

This is a "no CC analog / no behavior change" change, so per
`prd-compare-with-claude.md` it carries **no splits HTML** — there is no
design split point to decide, only drift to correct.

- **`config.py`** — delete `PROVIDER_DEFAULT_MODELS`, `save_config()`,
  `_serialize_provider()` (50 lines). The surviving surface is the
  read-path: `load_config()`, `bootstrap_config()`, `Config`/`Slot`,
  `price_table()`, and the `_read_*` validators.
- **`tests/test_smoke.py`** — delete the 119-line block that exercised
  the removed write-back symbols. The remaining smoke tests cover the
  live read-path.
- **`store.py`** — update the module docstring's dependency list from
  the dead `tui`/`agent_loop`/`onboard` to the real
  `terminal_chat`/`context_manager`/`llm`/`tools`/`executor`.
- **`README.md`** — full rewrite to the v1.7.4 surface: CC-shaped tools,
  IHEP-only config with the `pricing:` block, current CLI flags, the
  slash-command table, and a Documentation section linking
  `architecture.html` + `roadmap.html`.
- **`docs/roadmap.html`** — remove the two `ccusage` mentions; the JSONL
  emission stays neutrix-native with no external-tool-compat promise.
