# v0.5.3 — api_key edit semantics: clear-on-focus, preserve-on-empty-Enter

## Problem

The api_key Input's editing semantics in v0.5.2 don't match the model
a user holds in their head:

1. **Focus shows old masked dots.** When the user Tabs to a key Input
   that already has a saved key, they see masked dots and have to
   manually clear before typing a new one. They want a fresh blank
   field on focus, ready for new input.
2. **Empty Enter erases the saved key.** Focus the Input, type
   nothing, press Enter — the Input visibly drops back to the
   `EMPTY ...` placeholder and (worse) the YAML writes `api_key: ""`.
   The intent of "Enter with nothing typed" is "keep what I had,
   don't change anything," not "erase."
3. **Restoring on Tab/Up/Down already works (v0.5.2),** but only by
   accident: it's the same `_committed_value` revert that breaks #2.

## Goal

The api_key Input behaves like this:

- **Focus** → Input visibly clears (placeholder shows), waiting for
  user to type a new key. The committed value is preserved
  internally.
- **Blur via Tab / Up / Down without Enter** → Input restores to the
  committed value. No YAML write.
- **Enter with the field empty** → Input restores to the committed
  value, focus advances to the next focusable. No YAML write. No
  more accidental erasure.
- **Enter with a non-empty value** → Input value becomes that new
  value, focus advances, YAML written, stale `model_status` cleared.

## Non-goals

- Adding an explicit "erase api_key" action. Users who want to clear
  edit the YAML directly. Rare operation; not worth UI surface.
- A "show original key" affordance while focused. Plain-text reveal
  is a separate decision and a security trade-off.
- Different behavior per provider.

## Design

### `KeyInput` (in `onboard.py`)

Add `on_focus` that clears the visible value. Keep the existing
`on_blur` revert. `_committed_value` continues to be the source of
truth for the saved key.

```python
class KeyInput(Input):
    def __init__(self, *args, **kwargs) -> None:
        super().__init__(*args, **kwargs)
        self._committed_value: str = self.value

    def on_focus(self, event: events.Focus) -> None:
        # Clear visible buffer; committed value is preserved in
        # _committed_value and restored on blur / empty-Enter.
        if self.value:
            self.value = ""

    def on_blur(self, event: events.Blur) -> None:
        if self.value != self._committed_value:
            self.value = self._committed_value

    def on_key(self, event: events.Key) -> None:
        _navigate_focus(self, event)
```

### `OnboardScreen.on_input_submitted`

Short-circuit on empty Enter: restore and advance, leave state and
YAML untouched.

```python
async def on_input_submitted(self, event: Input.Submitted) -> None:
    provider = event.input.id.removeprefix("key-") if event.input.id else ""
    if provider not in self.provider_state:
        return
    new_key = event.value.strip()

    # Empty Enter = "no change". Restore visible value, advance focus.
    if not new_key:
        if isinstance(event.input, KeyInput):
            event.input.value = event.input._committed_value
        self.focus_next()
        return

    # Non-empty: commit baseline first (so any blur from focus-advance
    # doesn't race), then save state + persist YAML.
    if isinstance(event.input, KeyInput):
        event.input._committed_value = new_key
    event.input.value = new_key

    state = self.provider_state[provider]
    if new_key != state.api_key:
        state.model_status.clear()
        for row in self.query(ModelRow):
            if row.provider == provider:
                row.status = UNKNOWN
                row._refresh()
    state.api_key = new_key
    state.key_saved = True
    self._set_saved_indicator(provider, True)
    self._persist_yaml(fast=self.fast_choice, strong=self.strong_choice)
    self._notify(f"saved {provider} api_key", "success")
    self.focus_next()
```

### Test coverage to add

In `tests/test_onboard.py`:

  saved key clears `inp.value` to `""`; `_committed_value` stays.
- `test_empty_enter_preserves_committed_value` — focus, no typing,
  Enter: value restored to `_committed_value`; YAML's api_key

- `test_typed_enter_commits_new_value` — focus, type `"sk-new"`,
  Enter: value becomes `"sk-new"`, YAML updated, focus moved.

Existing tests stay green (they're already correct under the new
semantics — focus clearing the value just adds another step that
doesn't change the assertion outcomes).

## Acceptance

- Tab to an api_key Input with a saved key: visible field becomes
  empty, placeholder shows, `_committed_value` retained.
- Tab/Up/Down out without Enter: visible field restores to saved
  value (masked dots).
- Type nothing + Enter: field restores, focus advances, YAML
  unchanged.
- Type new key + Enter: field shows new key (masked), focus
  advances, YAML updated, prior model_status for the provider
  cleared.
- `pytest -x -q`: all tests pass (new + existing).
- `neutrix --version`: `0.5.3`.

## Out of scope

- "Reveal current key" affordance.
- Modal "are you sure you want to change?" dialog.
- An explicit erase path.
