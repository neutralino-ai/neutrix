# v0.10.5 — smart conversation compaction

> **Status: skeleton.** Detail solidifies after the
> [v0.11.0 smart Advisor](v0.11.0-smart-advisor.md) ships. The
> Advisor adds context cost (per-run LLM calls each consume some
> tokens) and increases the rate at which long sessions approach
> the context-window cliff — compaction is the budget side of the
> same agentic-depth arc.
>
> v0.9.5 already shipped a mechanical `/compact` (manual command,
> drop oldest 50 %, no summarization). v0.11.1 builds on that
> bridge: same slash command name, but now backed by an LLM-summarized
> cut segment and an automatic threshold trigger.
>
> v0.10.0 visibility parity is the precondition: a compacted summary
> is itself a channel sent to the LLM, and must render in the user
> view (folded). Phase 1 splits HTML lands at
> `docs/splits/v0.11.1-compaction.html` when implementation begins.

## Problem

After v0.11.0 a single neutrix session can run indefinitely with:

- the main controller LLM consuming N tokens per round
  (system + messages + tools schema + reminders),
- the Advisor consuming M tokens per fire (system + task list +
  last K turns), at periodic / idle / `/advise` triggers,
- the tool-use side rounds adding messages of variable size,
- the user's own prompts accumulating.

There is currently no mechanism to recover budget. Every LLM
round walks the entire `ContextManager.messages` list. At some
point either:

1. the provider's per-request token limit is hit and the call
   errors out, or
2. tokens become expensive enough that the user feels the cost
   of every additional turn.

CC's analog is `manageConversation` / `/compact`: when the
context budget is approaching its ceiling, summarize older turns
into a single condensed message and continue.

## Goal

1. Detect when `ContextManager.messages` total tokens approaches
   a configurable threshold of the model's context window
   (default proposal: 75 %).
2. Compact older turns by running a separate "summarize this
   conversation" LLM call against the segment to be compacted,
   replacing those turns with one
   `role: "system"` (or `role: "user"`-wrapped
   `<system-summary>...</system-summary>`) block that carries the
   summary.
3. Preserve `messages[0]` (system prompt) and the most recent N
   turns verbatim (default proposal: keep the last 4 user/assistant
   pairs after compaction). The cut point matters; pick by token
   share, not by index.
4. Render the summary in the transcript per v0.10.0 visibility
   parity: folded by default with `[summary]   N turns compacted ·
   M tokens · folded`. Ctrl+E expands.
5. Add a `/compact` slash command for explicit user-driven
   compaction (in addition to the automatic threshold).
6. Persist compaction events in `transcript.save` / `/load` so
   reloaded sessions know which turns were folded into a summary.

## Non-goals

- **Recursive / multi-level summarization** (summary-of-summaries).
  Single-level compaction only for v0.11.1.
- **Compacting tool results separately from messages.** Tool
  results travel inside `role: "tool"` messages; they get
  summarized as part of the segment.
- **Per-turn token tracking on `/status`.** Cost tracking is a
  separate v1.x feature.
- **Streaming the summary call.** Use `stream=False` for
  compaction — single short response, no user-visible incremental
  output. The heartbeat (v0.9.4) shows `AWAITING_LLM` during the
  call.
- **Rewinding through a compacted summary.** Compaction is
  destructive to the message list for the live session; the saved
  transcript can keep both forms if `transcript.save` records the
  pre-compaction state, but that's a v1.x rewind / branch concern.

## Design (sketch — finalize when v0.11.0 lands)

- **Module placement.** New `src/neutrix/compaction.py` rather
  than growing `src/neutrix/context_manager.py` (already 695
  lines as of v0.9.3). Hosts the threshold check, the cut-point
  walker, the summarizer call, and the replacement function.
  `ContextManager` imports + dispatches; compaction logic itself
  is testable in isolation.
- **Slot config field.** Add
  `max_context_tokens: int | None = None` to
  `src/neutrix/config.py:Slot`. Default `None` means "unknown" —
  if unknown, automatic compaction is disabled for that slot
  (the user can still invoke `/compact` manually). Users set
  the value in `~/.config/neutrix/config.yaml` per slot, e.g.
  `fast: { provider: ihep, model: …, max_context_tokens: 200000 }`.
- **Threshold trigger.** `ContextManager` tracks an approximate
  token count (cheap `len(json.dumps(messages)) // 4` heuristic
  for v0.11.1 — revisit only if the heuristic drifts far enough
  to fire spuriously). When the count exceeds the threshold
  (`compaction.threshold` in YAML, default `0.75` of the slot's
  `max_context_tokens`), CM emits a `CompactionEvent` to itself
  before the next `llm_request_start`.
- **Explicit trigger.** `/compact` slash command emits the same
  event regardless of token count.
- **Cut point.** Walk `messages` from the end back, summing
  tokens, until the kept-segment exceeds
  `(1 − compaction.cut_at) * max_context_tokens` (default
  `cut_at = 0.5` — keep the most recent 50 % of budget verbatim).
  Everything earlier is the compaction segment. Tool rounds at
  the boundary are walked forward until the segment edge falls
  between rounds, not mid-round (see Open Question on tool-round
  spanning).
- **Summary call.** `compaction.summarize_segment(segment, slot)`:
  given the segment, call the slot's LLM (default `fast` slot
  per `compaction.slot` YAML key) with a fixed
  `COMPACTION_SYSTEM_PROMPT` ("Summarize the following
  conversation, preserving decisions, open work items, and any
  tool results the agent will need later. Target length: ~200
  tokens.") and the segment messages as input. Returns one short
  paragraph.
- **Replacement.** Atomically swap the segment in
  `ContextManager.messages` with one
  `{"role": "user", "content": "<system-summary>{paragraph}</system-summary>"}`
  message. (See Open Question on `role: "system"` alternative.)
- **Rendering.** v0.10.0 already adds a fold/expand affordance
  for `<system-reminder>` and the system prompt; `<system-summary>`
  reuses the same machinery with its own prefix
  (`[summary]   {N} turns compacted`).
- **Store field.** Add
  `store.compaction_events: tuple[CompactionEvent, ...]` where
  `CompactionEvent` is a frozen dataclass with
  `(turn_count_compacted: int, original_token_count: int,
  summary_token_count: int, timestamp: str)`.
  `transcript.save` / `/load` round-trip the tuple.

## Acceptance (sketch)

- Manual:
  - Drive a session past the 75 % threshold; CM fires a compaction
    before the next LLM round; transcript shows
    `[summary]   N turns compacted` folded; `Ctrl+E` expands.
  - `/compact` works on-demand at any session size; same
    rendering, threshold ignored.
  - Save the session; reload; the summary persists in the
    on-disk transcript and reloads as the folded block.
- Automated, all in `tests/test_compaction.py`:
  - `test_token_count_estimator_matches_fixture()` — fixture of
    known size, assert the heuristic returns within ±15 %.
  - `test_threshold_triggers_at_75_percent()` — populate a
    `Slot` with `max_context_tokens=1000`, fill `messages` past
    750 estimated tokens, assert `should_compact()` returns
    `True`; below 750 → `False`.
  - `test_threshold_disabled_when_slot_has_no_max()` —
    `max_context_tokens=None` → `should_compact()` returns
    `False` regardless of `len(messages)`.
  - `test_cut_point_keeps_recent_half()` — fixture with 20
    turns; assert the cut keeps the most recent ~50 % of token
    budget; assert no orphan `tool_use` blocks survive the cut
    (compaction segment ends between tool rounds).
  - `test_summarize_replaces_segment_with_system_summary()` —
    mock-LLM that returns a fixed summary string; assert the
    output `messages` has the segment replaced by one
    `<system-summary>...</system-summary>` `role: "user"` message
    and the keep-suffix intact.
  - `test_summarize_failure_leaves_messages_unchanged()` —
    mock-LLM raises; assert `messages` unchanged and an error
    event is emitted to the store.
  - Visibility-parity invariant test (the
    `tests/test_visibility_parity.py` scenario set added in
    v0.10.0) gains a `test_summary_visible()` case — the summary
    IS visible (folded by default; expand reveals the paragraph).
- Human gate before tag (the v1.0 release gate).

## Open questions (resolve before implementing)

- **Where does the system summary live in the OpenAI payload —
  `role: "system"` (which means a second system message after
  the configured one) or `role: "user"` wrapped in a
  `<system-summary>` tag (consistent with the v0.8.0
  `<system-reminder>` shape)?** Default proposal: the
  `<system-summary>` wrap. Avoids the "two system messages"
  ambiguity providers handle differently. Confirm at design
  review.
- **What happens when a tool round spans the cut point?**
  The pairing layer (v0.9.3) does NOT span message-replacement
  boundaries; a half-cut tool round would produce orphan
  `tool_use` after the summary. Mitigation: when computing the
  cut point, walk forward past any tool round (the
  assistant message with `tool_calls` plus its paired
  `role: "tool"` results) so the summary boundary always
  falls between rounds.
- **Compaction failure mode.** What if the summarize call
  fails or returns garbage? Default proposal: leave `messages`
  unchanged, emit an error event to the store, surface a
  visible notice in the transcript. The user can `/compact`
  again manually.
- **Multiple compactions in one session.** Walking back from
  the end means later compactions might fold older summaries
  into a new summary. This is single-level compaction by
  definition (a summary is just a message); explicit decision
  needed on whether re-compacting an already-compacted segment
  is allowed. Default proposal: allow it — the summary message
  gets folded into the new summary like any other turn.
- **`compaction.threshold` per-slot vs global.** Different
  slots have different context-window sizes; the threshold is
  meaningful per-slot. Default proposal: per-slot, with a global
  default.
