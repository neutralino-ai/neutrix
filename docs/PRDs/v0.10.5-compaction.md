## Problem

A long session walks the entire `ContextManager.messages` every round; the
Advisor (v0.10.4) adds per-fire tokens. There's no way to recover budget, so
eventually the provider's per-request limit errors out or every turn gets
expensive. v0.9.6 shipped a mechanical `/compact` (drop oldest 50%, no LLM) as a
stopgap. This release makes it smart — summarize the cut segment and continue —
and adds the hardening that guarantees we can get under an arbitrary token limit
even from an already-over-limit state.

## Goal

1. **Token estimate + per-slot threshold.** `estimate_tokens(messages)`
   (`len(json.dumps)//4`); `max_context_tokens: int | None` on `Slot`;
   `should_compact()` true when estimate > 0.75 × max (None ⇒ disabled).
2. **Summary-based compaction.** `smart_compact(messages, *, llm, model, ...)`:
   compute a round-safe cut keeping the recent ~50% of budget, summarize the
   older segment via a `COMPACTION_SYSTEM_PROMPT` LLM call, replace the segment
   with one `<system-summary>…</system-summary>` `role:user` message. Failure ⇒
   messages unchanged + a visible notice.
3. **Auto-trigger** in `ContextManager` before a round when over threshold
   (guarded against re-fire on a no-op); **`/compact`** runs it on demand.
4. **The >1M hardening** (memory): **(a)** `compact_to_token_budget(messages,
   budget)` — drop oldest round-safe until under budget (the under-limit
   guarantee + the pre-summary primitive); **(b)** reactive prompt-too-long
   catch + one retry in `_call_llm`; **(c)** `truncate_large_tool_results(
   messages, cap)` — truncate within an oversized `role:tool` body.
5. **Render + persist.** `<system-summary>` renders folded as `[summary] N turns
   compacted · …` (`/show`-expandable); `CompactionEvent` in the store +
   transcript round-trip.

## Non-goals

- Recursive summary-of-summaries (single-level; a summary is just a message).
- Per-turn token tracking on `/status` (v1.x cost feature).
- Streaming the summary call (`stream=False`; heartbeat covers liveness).
- Rewinding through a summary (compaction is destructive to the live list).

## Claude Code reference

CC analog: `manageConversation` / `/compact` (summarize-and-continue),
`truncateHeadForPTLRetry` (reactive cut+retry), `microCompact.ts` (content
truncation). See [docs/splits/v0.10.5-compaction.html](../splits/v0.10.5-compaction.html)
for the 7 decisions. Highlights:
- **Summary payload shape**: `<system-summary>` `role:user` (avoids two system
  messages; reuses marker machinery).
- **Cut**: between tool rounds only (reuse v0.9.6 forward-snap).
- **/compact**: smart, with the token-budget cut running first so the summarize
  call is always sendable (even from over-limit).
- **Failure**: unchanged + notice; auto-trigger guarded against re-fire.

## Design

### `src/neutrix/config.py`
- `Slot.max_context_tokens: int | None = None` (read from YAML per slot).

### `src/neutrix/compaction.py` (extend)
- `estimate_tokens(messages) -> int` (char//4).
- `should_compact(messages, *, max_context_tokens, threshold=0.75) -> bool`.
- `compact_to_token_budget(messages, *, budget) -> (messages, CompactionOutcome)`
  — round-safe oldest-drop until under budget.
- `truncate_large_tool_results(messages, *, cap) -> (messages, n_truncated)`.
- `COMPACTION_SYSTEM_PROMPT`; `async summarize_segment(segment, *, llm, model) -> str`.
- `async smart_compact(messages, *, llm, model, max_context_tokens, ...) ->
  (messages, CompactionOutcome)` — token-budget cut (if over-limit) → cut by
  share → summarize → replace with `<system-summary>`.
- `SUMMARY_MARKER_OPEN/CLOSE` + `is_summary_marker(content)`.
- `@dataclass(frozen=True) CompactionEvent(turns_compacted, original_tokens, summary_tokens, kind)`.

### `src/neutrix/context_manager.py`
- `compact()` upgraded: if a summarizer llm/model available, run `smart_compact`;
  else fall back to the mechanical `compact_messages` (preserve current return).
- `maybe_auto_compact()` called at round entry; PTL catch+retry in `_call_llm`.
- Store the `CompactionEvent`.

### `src/neutrix/store.py`
- `compaction_events: tuple[CompactionEvent, ...]` + `add_compaction_event()`,
  cleared on `reset()`.

### `src/neutrix/terminal_chat.py`
- `<system-summary>` render branch → `[summary] …` folded notice; `/show summary`
  (or reuse `/compact` notice). Exclude from recall.

### Tests (`tests/test_compaction.py` extend + `test_visibility_parity` case)
- estimator ±, threshold at 75% / disabled-when-None, cut keeps recent half +
  no orphan tool round, summarize replaces segment, summarize-failure leaves
  unchanged, token-budget cut gets under budget, content-truncation caps a huge
  tool body, summary renders folded/excluded-from-recall.
