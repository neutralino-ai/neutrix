# v1.4.7 — live streaming render

> **Status: finalized.** v2-band release #5. Decided autonomously under the
> `build-to-v2.0.0` /goal; **user-asked "streaming: come here"**. Design
> advisor-validated. Splits:
> [docs/splits/v1.4.7-streaming-render.html](../splits/v1.4.7-streaming-render.html).

## Problem

v0.10.1 restored *transport* streaming + keep-partial-on-cancel but deferred the
*rendering* — the user never sees tokens arrive (the full message appears at
once, same as non-streaming). For a Claude-Code-feel coding agent (and the
v0.9.x liveness arc), tokens should render live.

## Goal

Render assistant tokens live, without disturbing the append-only scrollback:
- `_call_llm` routes `token` deltas to `store.start/extend_assistant_stream`.
- `_above_input` shows a **bounded** preview (last `STREAM_PREVIEW_LINES`=8 lines)
  of `store.pending_assistant_text` — in the live region only.
- On finalize (commit / cancel / error), clear pending **in the same
  synchronous beat** as the committed append; the full message commits to
  scrollback via the existing `_render_record` path.
- **Unify on `store.pending_assistant_text`** as the single in-flight holder —
  retire v0.10.1's `ContextManager._streaming_partial` (a second mirror would
  risk divergence across the four exit paths).

## Non-goals

- Reworking append-only scrollback (the preview lives in `_above_input`).
- Per-token render-rate tuning / tick-decoupling (coalesced invalidate is fine).
- Streaming tool-call fragments visibly (still finalize-then-render).

## Claude Code reference

CC renders streamed blocks in a live region, commits on finish. neutrix matches
the *semantics* via a bounded live-region preview + scrollback commit. See
[docs/splits/v1.4.7-streaming-render.html](../splits/v1.4.7-streaming-render.html).
Decisions (advisor-validated): unify the mirror on the store; strict
preview/scrollback channel separation; clear-pending in the commit beat.

## Design

### `src/neutrix/store.py`
- `clear_pending_assistant_text()` — drop pending WITHOUT appending a record
  (the full message is appended separately with tool_calls).

### `src/neutrix/context_manager.py`
- Retire `_streaming_partial`. `_call_llm`: `start_assistant_stream()` at entry,
  `extend_assistant_stream(delta)` per token. Normal commit: append then
  `clear_pending_assistant_text()` (no await between). Error: clear (discard).
  `_do_cancel`: commit from `store.pending_assistant_text` then clear
  (keep-partial-on-cancel, now store-sourced).

### `src/neutrix/terminal_chat.py`
- `_stream_preview(pending)` (last-N-lines, `…` elision); `_above_input` renders
  it (dim italic) in the live region only.

## Acceptance

- Automated: tokens populate `store.pending` mid-round and clear on commit;
  keep-partial-on-cancel still holds (now via `store.pending`); `_stream_preview`
  bounds correctly. **The v0.10.1 cancel-keep + v0.10.5 shrink-cursor tests stay
  green** (proof the unification preserved behavior). Full suite green (347);
  ruff clean.
- Manual: a long reply streams a bounded preview above the input; commits to
  scrollback on finish; Esc mid-stream keeps the partial.
- Ship `v1.4.7`.

## Out of scope

- Streaming tool-call fragments, render-rate tuning, scrollback rework.
