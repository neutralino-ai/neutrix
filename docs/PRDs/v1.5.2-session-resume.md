## Problem

A neutrix session is ephemeral: kill it (or the recurring proxy hang) and the
conversation is gone — `/save`·`/load` is manual and easy to forget. And, as the
user flagged, **per-turn LLM/tool durations aren't recorded** anywhere (the
v1.5.0 status-bar elapsed is live-only). Both are gaps a Claude-Code-replacement
must close.

## Goal

- **Auto-persist** every turn to a **Claude-Code-compatible** session log so
  `ccusage` / CC `--resume` can read it, and a killed session is resumable.
- **`--continue`** resumes the most recent session in the cwd; **`/resume`**
  picks a past one.
- **Record per-turn timing** (`timestamp` + `llm_ms`/`tool_ms`) in the log.

## Non-goals

- The neutrix-native **`/goal` loop** (separate later release).
- Cross-project resume, session rename/tag/summary, fork-on-resume (CC has these;
  later if wanted).
- Replacing `/save`·`/load` (kept as explicit single-file export).
- A full-screen picker (numbered in-flow list instead — split #5).

## Claude Code reference

See [docs/splits/v1.5.1-session-resume.html](../splits/v1.5.1-session-resume.html).
CC: `~/.claude/projects/<sanitized-cwd>/<uuid>.jsonl`, append-per-message, each
entry `{type, timestamp, cwd, sessionId, …}`; `--continue` = most-recent-by-mtime
(`conversationRecovery.ts`); `--resume` picker (`listSessionsImpl.ts` head/tail
lite reads); restore = full message array + tasks (`sessionRestore.ts`).

- **Format/location**: **User-directed different** — CC's JSONL *shape*, but under
  `~/.cache/neutrix/sessions/<cwd>/<uuid>.jsonl`, **NOT `~/.claude`** (user: "don't
  write to ~/.claude folder"; `$NEUTRIX_SESSION_HOME` overrides). Drops ccusage-compat.
- **Append-as-you-go**: Follow CC — hook `ChatStore.changes()`.
- **Timing**: Follow CC `timestamp` + Alternative add `llm_ms`/`tool_ms`.
- **`--continue`**: Follow CC — most-recent-by-mtime → `ReplaceHistoryEvent`.
- **`/resume`**: Alternative — numbered in-flow list (reuse v1.4.8 channel).
- **Restore**: Follow CC — messages + records + tasks; slot stays launch slot.
- **Identity**: append-to-same-file on resume (no fork); UUID at launch.

## Design

### `src/neutrix/session_store.py` (new)
- `session_dir(cwd, home)` → `~/.cache/neutrix/sessions/<sanitized-cwd>/` (base:
  explicit `home` > `$NEUTRIX_SESSION_HOME` > real home; NOT `~/.claude`). cwd
  sanitized path-safe, hashed if >200 chars.
- `SessionWriter(path)`: `append(record: MessageRecord, *, llm_ms=None, tool_ms=None)`
  → one JSONL line `{type, role, content, tool_call_id?, tool_name?, timestamp,
  llm_ms?, tool_ms?}` (CC-shaped + namespaced extras). Best-effort (swallow OSError).
- `list_sessions(cwd, home)` → `[SessionInfo(path, mtime, first_prompt, n_msgs)]`
  sorted mtime-desc, via head/tail-lite reads. `most_recent(cwd)` → first or None.
- `load_session(path)` → `(raw_messages, records, tasks)` (the `ReplaceHistoryEvent`
  triple) — reuse `transcript`'s record→dict mappers where shared.

### `src/neutrix/terminal_chat.py`
- A session writer subscribed alongside the renderer: on each new `MessageRecord`,
  append it; for assistant rounds attach `llm_ms` (from `ctx.phase_started_at` at
  commit), for tool entries `tool_ms`.
- `_cmd_resume(args)`: no arg → numbered list of `list_sessions(cwd)`, await a pick
  via the v1.4.8 answer channel, then `ReplaceHistoryEvent`; `<id/path>` → load directly.
- Resume reuses the live session file (append continues).

### `src/neutrix/cli.py`
- `--continue`/`-c`: seed from `most_recent(cwd)` (like `--load` seeds from a file).
  `--resume <id>`: seed from that session. Wire the live `SessionWriter`.

## Acceptance

- **Automated**: `session_store` round-trips (append N records → `load_session`
  rebuilds the same messages+tasks); `list_sessions` sorts mtime-desc with
  first-prompt/n-msgs; `most_recent` picks newest; timing fields written + parsed;
  sanitize handles long cwd. Resume drives `ReplaceHistoryEvent` and the suite's
  existing load-path tests stay green. Full suite green; ruff clean.
- **Manual**: a turn auto-writes a JSONL line under `~/.cache/neutrix/sessions/<cwd>/`;
  kill neutrix mid-session, relaunch with `--continue` → the conversation is back;
  `/resume` lists past sessions, picking one restores it; the JSONL shows
  `llm_ms`/`tool_ms` per turn; `ccusage` can read the dir.
- Ship `v1.5.1`.

## Out of scope

- `/goal` loop, cross-project resume, rename/tag/summary, fork-on-resume.
