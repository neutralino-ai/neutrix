# v0.5.0 — onboarding: persisted status, verify-all, key preservation, scoped arrows, UI rework

## Problem

Five concrete issues, all in onboarding:

1. **Verification status is session-only.** Quitting `/onboard` loses every
   `✓` and `✗`. Reopening shows everything as `?` again, even though the
   user just verified them ten seconds ago.
2. **No way to verify a full provider at once.** Each model requires a
   separate `v` keystroke. For a provider with 8 models that's painful.
3. **API key visually disappears after `Enter`.** With `password=True` the
   pre-save rendering and the post-save rendering both look like empty
   field (or differently-rendered dots), and there's no "saved" affordance
   on the row.
4. **Arrows escape the focusable scope and trap focus.** Pressing `Up`
   past the first focusable lands on the focused `VerticalScroll`, which
   handles `up`/`down` as scroll commands instead of focus navigation;
   `Down` no longer moves focus until the user `Tab`s out.
5. **The UI is flat and ugly.** Plain `Static` rows, no color-coded
   status, no visual provider grouping, no "saved key" indicator.

## Goal

Ship a polished onboarding flow that:

- Persists model verification status to `~/.config/neutrix/config.yaml`
  per provider, so reopening `/onboard` resumes where the user left off.
- Adds a special `(all)` row per provider that verifies every model in
  parallel via one `asyncio.gather`.
- Keeps the api_key visible (masked) after `Enter`, with a clear "saved"
  affordance next to the field.
- Constrains arrow-key focus navigation so it never lands on a widget
  that hijacks the arrows.
- Looks like something a 2026 TUI agent would ship — bordered provider
  cards, colored status, accent-coloured slot tags, generous padding.

## Non-goals

- A second config-format version negotiation. The new `model_status` key
  is additive and silently ignored by older loads (it's only consumed by
  onboarding).
- Encrypted storage of api_keys or model_status. Same plain YAML.
- Auto-verify on `/onboard` open. User still presses `v` (or `v` on
  the `(all)` row).
- Background re-verify when api_keys change in other tools.
- A full chat-TUI restyle. This release only touches `OnboardScreen`.
- A migration that rewrites old configs. Reads still work; on first
  save the new fields appear.

## Design

### 1. Persisted model status — YAML schema

Additive change inside each provider entry:

```yaml
providers:
  ihep:
    base_url: https://aiapi.ihep.ac.cn/apiv2/
    api_key: sk-xxxxx
    model_status:
      "anthropic/claude-haiku-4-5": verified
      "anthropic/claude-opus-4-7": failed
```

- `model_status` is optional. Missing → all models start as `unknown`.
- Three values: `verified`, `failed`, or absent (= `unknown`). No other
  values are written.
- **When the api_key changes, `model_status` is cleared** in memory and
  on the next persist — old verifications no longer mean anything.
- `Config.slot()` is unchanged: slot resolution doesn't depend on
  `model_status`.

`save_config()` serialises `model_status` only when non-empty, so
existing configs without it stay clean.

### 2. Verify-all in parallel

Add a new focusable widget `VerifyAllRow` rendered as the first row
under each provider's "Models" subheader:

```
   ▶ (all) — verify all in parallel
```

When the row is focused and `v` is pressed:

```python
async def _verify_all(self, provider: str) -> None:
    state = self.provider_state[provider]
    if not state.api_key:
        self.notify(f"set {provider}'s api_key first", severity="warning")
        return
    rows = [r for r in self.query(ModelRow) if r.provider == provider]
    for r in rows:
        r.status = "verifying"; r._refresh()
    client = AsyncOpenAI(base_url=state.base_url, api_key=state.api_key)
    tasks = [_check_one(client, r.model) for r in rows]
    results = await asyncio.gather(*tasks)
    for r, ok in zip(rows, results, strict=True):
        r.status = "verified" if ok else "failed"
        state.model_status[r.model] = r.status
        r._refresh()
    self._persist_yaml(...)
```

Single shared `AsyncOpenAI` client per parallel batch — cheaper than
constructing 8 clients.

### 3. Key preservation after `Enter`

Two defensive changes inside `on_input_submitted`:

- Explicitly re-assign `event.input.value = new_key` after reading it,
  so the Input keeps showing the masked dots even if Textual ever
  changes its post-submit behaviour.
- Add a sibling `Static` ("saved") next to the api_key Input that lights
  up green after a successful submit, and stays. Removes ambiguity for
  the user: they can see the key is committed without trusting the dots.

The Input keeps `password=True` (masked while typing).

### 4. Arrow scope — don't trap focus on `VerticalScroll`

Root cause: `VerticalScroll(can_focus=True)` (Textual default) made it
a tab/arrow target. Once focus lands there, the widget's own
`up`/`down` BINDINGS hijack the keys for scrolling and the
ModelRow/KeyInput `on_key` navigation never fires.

Fix: subclass with `can_focus = False` (and let Textual auto-scroll
when a child gains focus, which it already does).

```python
class FocusScroll(VerticalScroll):
    can_focus = False
```

`ModelRow.focus_previous` from the top row still wraps to the last
focusable — that's Textual's default `Screen.focus_previous` behaviour,
which works once `VerticalScroll` is out of the chain.

### 5. UI rework

Visual structure per provider, with rounded borders and accent colours:

```
╭ ihep ────────────────────────────────────────────────────────╮
│ https://aiapi.ihep.ac.cn/apiv2/                              │
│                                                              │
│  api key  [••••••••••]              saved                    │
│                                                              │
│  Models                                                      │
│   ▶  (all)                          verify all in parallel   │
│   ✓  anthropic/claude-haiku-4-5     verified  ▸ fast         │
│   ✗  anthropic/claude-opus-4-7      failed                   │
│   ○  anthropic/claude-sonnet-4-6                             │
╰──────────────────────────────────────────────────────────────╯
```

CSS strategy:

- Each `ProviderSection` gets `border: round $primary` and a
  `border_title` set to the provider name.
- Status icon column uses Rich-markup colour:
  `✓` `$success`, `✗` `$error`, `○` `$text-muted`, `…` `$warning`.
- Slot tags rendered as `▸ fast` in `$accent` bold, right-aligned.
- Focus highlight on rows: `background: $accent 25%` (clear but not
  glaring).
- Header bar: `neutrix · onboarding` left, config-path right.
- Bottom intro replaced with a `Footer` plus a separate help bar
  showing the current bindings only.

Padding and margins generous; no widget touches the border.

All colour tokens reference Textual's design system (`$primary`,
`$success`, `$error`, `$accent`, `$text-muted`, `$boost`) so the theme
follows the user's terminal palette.

### File touch list

### Out of scope (deferred)

- Per-model timestamps ("last verified 3 days ago").
- A `model_status: stale` state that triggers re-verify automatically.
- Verify-all-providers (one keystroke to verify every model across
  every provider). Probably overkill.
- An "edit api_key" affordance that pre-clears the field on focus.

## Acceptance

- After `v` on any model row and quit + reopen `/onboard`, the row
  retains its `✓` / `✗` status from the YAML.
- `v` on the `(all)` row of a provider with N models fires N
  `chat.completions.create` calls in parallel (single `AsyncOpenAI`
  client), then writes the N statuses to YAML in one save.
- Typing an api_key + `Enter`: the dots remain visible, and a green
  "saved" tag appears next to the Input until the key is edited again.
- Pressing `Up` from the first focusable and `Down` from the last
  wrap, and the focus never lands on `VerticalScroll` (so arrows never
  switch to scroll mode).
- Visual: each provider sits inside a rounded bordered card with a
  titled header; status icons are colour-coded; assigned slots show as
  accent-coloured `▸ fast` / `▸ strong` tags right-aligned on the row.
- `pytest` + ruff stay green.
- `neutrix --version` reports `0.5.0`.
