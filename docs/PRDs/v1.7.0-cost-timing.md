# v1.7.0 — usage, cost & timing

## Problem

neutrix reports nothing about what a session *costs*. For real OMILREC work —
especially long autonomous `/goal` runs — there is no way to see token usage or
dollar spend, and the per-turn `llm_ms`/`tool_ms` timing captured since v1.5.2 is
persisted but never surfaced. The stream already carries usage (OpenAI in a final
chunk; the anthropic gateway in `message_start`/`message_delta`) but neutrix drops
it. This is the last capability gap before the v2.0.0 lock.

## Goal

1. Capture token usage from **both** stream protocols and a normalized
   per-message record persists to the session JSONL.
2. Turn usage into a **dollar cost** via a curated price table, gracefully
   degrading to "(cost unknown)" for unpriced models.
3. Surface per-session **cumulative** usage, cost, and timing via a **`/cost`**
   command and a terse **status-line** readout; survive `--continue`/`/resume`.

## Non-goals

- **Parallel tools and background Bash — DEFERRED** (Split #10, user-delegated
  architect call). Concurrent `tool_use` dispatch is "an unused optimization"
  (`executor.py` v0.9.3) — models rarely batch calls; background Bash is a whole
  subsystem against "keep it simple." Cost/timing **is** v1.7.0, the last PRD
  before v2. The roadmap reshape (dropping the old v1.7.0=parallel / v1.8.0=cost
  entries) lands with this PRD's acceptance. Reopen post-v2 only if real use
  demands it.
- **Per-subagent cost attribution** — subagent usage rolls into the session total;
  splitting it out is YAGNI (Split #9).
- **A network price fetch at runtime** — the curated table is vendored data
  (Split #4).
- **A session-exit summary** — neutrix has no exit hook; `/cost` is the surface
  (Split #7).
- **Changing the streaming/cancel/watchdog control flow** beyond the one-shot
  `include_usage` probe (Split #2).

## Claude Code reference

See `docs/splits/v1.7.0-cost-timing.html` for the full survey (10 split points,
agent-decided under the user's "you make decision yourself" delegation, 2026-05-31;
vetoable via that file). CC reads usage off the stream in `updateUsage()`
(`services/api/claude.ts:2924-2987`, cumulative-per-message, `>0` guards), sums in
`accumulateUsage()` (`:2993-3038`, cache tokens separate), prices from a hardcoded
per-million table with 4 token classes (`utils/modelCost.ts:26-142`), has **no
`/cost`** (exit summary + status line), tracks API-vs-wall timing
(`logging.ts:686-688`), and keeps a single mutable global accumulator
(`state.ts`). neutrix mirrors the *semantics*.

### Split point: usage capture — both stream protocols
- **CC**: native `updateUsage` off the Anthropic stream.
- **Decision**: Follow CC semantics — extend the v1.6.1 inbound parser to read the
  OpenAI final-chunk `usage` **and** the anthropic `message_start`/`message_delta`
  usage; `LLMResponse` gains a normalized `usage`.

### Split point: `stream_options.include_usage` 400-risk
- **Decision**: Alternative — **OpenAI-path-only, probe-once-per-slot,
  send-and-catch**. Gate on `_uses_anthropic_messages_gateway()` false. Treat **any
  400 while `include_usage` was set** as unsupported (do not parse the opaque
  gateway body), retry once without it, cache the capability on the slot. Tests: a
  genuine non-`include_usage` 400 still propagates; the retry never double-yields
  tokens.

### Split point: token-class normalization (the silent-corruption risk)
- **Decision**: Follow CC — **store raw per-provider counts**; normalize to 4
  fields (`input`/`output`/`cache_read`/`cache_write`). **Cache accounting verified
  EMPIRICALLY per provider before pricing** (capture one real `usage` payload each):
  OpenAI `prompt_tokens` typically *includes* `cached_tokens` → `input = prompt −
  cached`; Anthropic reports `cache_read_input_tokens` *separately* → no subtract.

### Split point: cost source
- **Decision**: Alternative — **LiteLLM's `model_prices…json` as DATA, not the
  dep** ([[project_v2_v3_direction]]); a small curated subset, per-million, 4
  classes; unknown model → "(cost unknown)".

### Split point: cumulative accounting (v3-RL headless constraint)
- **Decision**: No-CC-analog variant — a **session-scoped `CostLedger`** that
  *observes* completed `LLMResponse`s (does not drive the loop, keeping the CM
  headless/pure), **persists the raw token counts** (not the computed dollar) to
  JSONL, computes $ lazily on read, and rebuilds from JSONL on resume. The ledger
  is a **live accumulator fed once per assistant turn — never a sum over the live
  `store.messages`** — because compaction/rewind shrink the store while session
  cost must stay cumulative; the append-only JSONL backs `from_jsonl` on resume.

### Split point: usage persistence — line shape (decided mid-implementation)
- **CC**: no analog — CC persists a single global usage accumulator blob per
  session (`state.ts`), not per-message JSONL.
- **Decision**: Alternative — a **dedicated `{"type":"usage", …}` JSONL line per
  completed turn** (mirrors the existing `append_tasks` typed-line precedent),
  *not* the message-line extras the cumulative-accounting draft implied. A turn's
  `tool_ms` is not known when the assistant line flushes, and the message-flush
  cursor lives in `terminal_chat` while timing/usage live in the CM; a bundled
  per-turn line lets `CostLedger.from_jsonl` rebuild one `LedgerEntry` from one
  line without correlating assistant + tool lines or widening `MessageRecord`.
  `terminal_chat` persists new entries via its own cursor **and flushes once
  explicitly at turn completion** (not only on the `store.changes()` tick) so a
  future `await` reordering between the assistant append and `ledger.record()`
  cannot silently drop the final turn's usage. See
  `docs/splits/v1.7.0-cost-timing.html` Split #11.

### Split point: timing
- **Decision**: Follow CC — reuse v1.5.2 `llm_ms`/`tool_ms`; ledger rolls up
  API/tool/wall.

### Split point: `/cost` command + status line
- **Decision**: Alternative / No-CC-analog — **add `/cost`** (CC has none): session
  totals (4 token classes, $, API/tool/wall, per-model) + a terse status-line
  readout (`$x · Nk↑ Nk↓`), quiet per [[feedback_ui_hints]].

### Split point: subagent attribution
- **Decision**: Alternative — subagent usage **rolls into the session ledger**
  (store is the source of truth); per-subagent split deferred.

### Split point: parallel tools + background Bash
- **Decision**: Alternative — **deferred** (see Non-goals). The vetoable split.

## Design

1. **`Usage` dataclass** (`llm.py`): `input`, `output`, `cache_read`,
   `cache_write` (ints) + `raw: dict` (the untouched provider payload, source of
   truth). A `+` that sums fields. `LLMResponse` gains `usage: Usage | None`.

2. **Inbound capture** (extend the v1.6.1 seam): OpenAI path reads the final
   chunk's `usage` (`prompt_tokens`/`completion_tokens`/`prompt_tokens_details.
   cached_tokens`); `_apply_anthropic_event` reads `message_start.message.usage`
   (input + `cache_creation`/`cache_read`) and accumulates `message_delta.usage`
   (output). Normalization applies the empirically-verified per-provider cache rule.

3. **`include_usage` probe** (`OpenAIChatLLM`): only when not the anthropic
   gateway. A per-slot capability cache (`_include_usage_supported: bool | None`):
   `None` → attempt with `stream_options={"include_usage": True}`; on **any** 400
   while set, mark `False`, retry once without; once known, skip the probe. Isolated
   from the token-yield path so a retry cannot double-emit.

4. **Pricing** (`pricing.py`, new): a vendored dict seeded from LiteLLM's
   `model_prices_and_context_window.json` (per-million, 4 classes) for the models
   neutrix's slots use; `cost(usage, model) -> float | None` (None → unknown).

5. **`CostLedger`** (`cost_ledger.py`, new, session-scoped): a pure in-memory
   accumulator (no I/O of its own — v3-headless). `record(model, usage, llm_ms,
   tool_ms)` appends a frozen `LedgerEntry`; cumulative getters sum on demand;
   `cost()` computes lazily via `pricing`. `from_jsonl()` rebuilds the entries from
   the dedicated `{"type":"usage", …}` lines on resume. The CM holds an optional
   `ledger` reference (default `None` ⇒ loop untouched) and calls `record(...)`
   once per completed assistant turn — it **enriches**, never branches on the
   ledger. `SessionWriter.append_usage(...)` (new) writes the per-turn line;
   `terminal_chat` owns the writer + ledger, injects the ledger into the CM, and
   persists new entries via a dedicated cursor in the `store.changes()` loop **plus
   an explicit flush at turn completion** (Split #11). Per-turn raw token counts
   (the source of truth) + normalized 4 classes + `llm_ms`/`tool_ms` land on that
   line; dollars are never stored.

6. **Surface**: a `/cost` command (`terminal_chat.py`) renders the ledger
   (tokens by class, $ or "(cost unknown)", API/tool/wall, per-model); the v1.5.0
   status bar gains a terse cumulative readout.

## Acceptance

1. **Usage capture (unit, recorded events):** an OpenAI final-usage chunk and an
   anthropic `message_start`+`message_delta` sequence each produce the correct
   normalized `Usage` on `LLMResponse`; the per-provider cache rule is applied.
2. **`include_usage` (unit):** first OpenAI turn probes with `include_usage`; a 400
   while set flips the slot to unsupported and retries once without (no double-yield
   of tokens, asserted); a genuine non-`include_usage` 400 still raises; anthropic
   slots never send it.
3. **Cache accounting (live):** one real `usage` payload captured from an
   OpenAI-compat slot and an anthropic slot confirms whether `input` includes
   `cache_read`; the normalization matches.
   - **Result (2026-05-31, verified live against the IHEP gateway):** OpenAI path
     (`deepseek-ai/deepseek-v4-flash`) → `prompt_tokens=11, completion_tokens=23,
     prompt_tokens_details.cached_tokens=0` → `input=11 output=23` ✓. Anthropic path
     (`anthropic/claude-haiku-4-5`) → top-level `input_tokens=14, output_tokens=4,
     cache_*=0` ✓. **The capture caught a real bug:** the gateway carries the
     Anthropic counts as `model_extra` *on* `chunk.usage` (with the OpenAI fields
     `null`), so the original "route usage by presence" logic sent it to
     `_usage_from_openai` and normalized to **all-zeros**. Fixed: usage is routed by
     **protocol** (the slot's anthropic-gateway flag), the full payload kept as
     `raw`; a regression test (`test_anthropic_gateway_usage_on_chunk_usage_not_misread_as_openai`)
     encodes the real shape. A non-zero `cache_read` live exercise needs explicit
     prompt-cache priming (neutrix sends no `cache_control` yet) — the field
     *placement* asymmetry is confirmed; live cache-hit counts are deferred to real
     cached traffic.
4. **Pricing (unit):** a known model yields the expected $ (per-million, 4 classes,
   cache priced distinctly); an unknown model yields `None` → "(cost unknown)".
5. **Ledger (unit):** records persist raw tokens (not $) to JSONL; cumulative sums
   are correct; `from_jsonl()` rebuilds totals on resume; changing a table price
   reprices an existing session (dollars computed on read).
6. **Surface:** `/cost` prints the totals + per-model breakdown; the status line
   shows the terse readout when usage exists and hides it on "(cost unknown)".
7. `ruff` clean; full `pytest` green.
