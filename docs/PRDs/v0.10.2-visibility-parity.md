# v0.10.2 — Visibility parity: show what the LLM sees

> **Status: finalized.** Third release on the march to v1.0, fleshed from the
> skeleton autonomously under a delegating `/goal`. Split-point decisions in
> [docs/splits/v0.10.2-visibility-parity.html](../splits/v0.10.2-visibility-parity.html).
>
> **Scope correction (advisor-flagged):** the skeleton's premises were stale.
> Ground-truthing the current renderer shows **3 of 4 channels already satisfy
> parity** — only **tool schemas** are genuinely hidden. This release is scoped
> to the real gap, not the skeleton's full 4-channel list. CC has no direct
> analog (parity is a neutrix invariant), so this is "No CC analog" work.

## Problem

The invariant we want: **every byte the LLM receives in a round appears in the
user-visible transcript, differing only by fold/expand — never by omission.**
Current state, verified against `src/neutrix/terminal_chat.py`:

| Channel | Renders today? |
|---|---|
| system prompt | **Yes** — full, inline (`_render_record` → `print_system`) |
| tool schemas (`tools=`) | **No** — never rendered (the gap) |
| `<system-reminder>` | **Yes** — a distinct dim notice (`format_reminder_notice`) |
| tool results | **Yes** — folded summary + `/tool N` expand |

So the genuine gap is **tool schemas**: the user never sees what tool
definitions the LLM is offered, so a wrong/stale/missing schema is invisible
until tool-calling breaks. Secondary: a long system prompt dominates session
start un-folded.

## Goal

1. Render the **tool schemas** as a folded block (`[tools] M schemas · K B`) at
   session start when tools are enabled. (The real gap.)
2. **Fold the system prompt** when it exceeds a threshold (short prompts — incl.
   the default — stay inline, preserving current behavior).
3. Provide **expand-by-append** via a `/show <what>` command (`system`,
   `tools`) that re-prints the full content below (the `/tool N` idiom
   generalized). **No `Ctrl+E` binding** (split #1).
4. A normative **`.claude/rules/visibility-parity.md`** so future PRDs don't
   reintroduce hidden channels — worded for the append-only expand-by-append
   reality.
5. An **invariant test** asserting that for a recorded round, every populated
   channel of the `LLMRoundBundle` produced ≥1 render call (dynamic field
   enumeration, not a hardcoded tuple).
6. Discharge v0.10.0's deferred subagent rendering with a **`[subagent]` label**
   (split #5) — the result already folds/expands via the tool-result path.

## Non-goals

- **In-place fold/expand toggle / `Ctrl+E`** — infeasible in append-only
  scrollback (split #1); expansion is by re-printing below.
- **Editing** system prompt or schemas from the UI (read-only renders).
- **Re-working the reminder or tool-result rendering** — already parity-compliant.
- **Per-block fold state across restarts.**
- **Truncating what's sent to the LLM** (the opposite asymmetry).
- **`tui.py`** (standing non-goal).

## Claude Code reference

No direct CC analog — visibility parity is a neutrix-specific invariant (CC's
React/Ink transcript doesn't enforce LLM-view == user-view). See
[docs/splits/v0.10.2-visibility-parity.html](../splits/v0.10.2-visibility-parity.html)
for all 5 decisions. Key ones:

### Split point: fold model
- **Decision**: No CC analog / Alternative — expand-by-append (no in-place toggle), forced by the append-only renderer; generalizes the `/tool N` idiom into `/show`.

### Split point: bundle + AgentEvent
- **Decision**: No event (v0.9.3 removed AgentEvents). Render static system/tools once at session start; `LLMRoundBundle` is a snapshot for the invariant test.

### Split point: subagent fold (v0.10.0 deferral)
- **Decision**: a `[subagent]` label on the existing tool-result fold path — no new fold machinery.

## Design

### `.claude/rules/visibility-parity.md`
The normative rule, worded for append-only: every LLM-bound channel renders a
folded summary; full content is reachable by an expand command that re-prints
below. Single source of truth; one render hook per channel; label injected turns;
invariant test.

### `src/neutrix/context_manager.py`
```python
@dataclass(frozen=True)
class LLMRoundBundle:
    system: str
    messages: tuple[dict[str, Any], ...]
    tools: tuple[dict[str, Any], ...] | None
```
Plus `ContextManager.round_bundle() -> LLMRoundBundle` building the snapshot
from `messages[0]`, `messages`, and `get_schemas(tool_names)` if
`effective_tools_enabled()`. Used by the invariant test.

### `src/neutrix/terminal_chat.py`
- `SYSTEM_FOLD_THRESHOLD = 200` (bytes). `_render_record` system branch: if
  `len(content) > threshold`, print `[system] · N B` folded summary; else inline
  (current behavior). Stash the full text for `/show system`.
- A `[tools] M schemas · K B` folded block printed once at session start (after
  the initial transcript) when tools are enabled; full schema list stashed for
  `/show tools`.
- `_cmd_show(args)`: `/show system` and `/show tools` re-print the full content.
  Add to help; not busy-guarded (read-only).
- Tool-result summary: when the tool is `Agent`, render the keyword as
  `subagent` instead of `tool_result` (`[subagent]` label).

### Invariant test (`tests/test_visibility_parity.py`)
Spy on `TerminalView.print_*` across session-start AND the `store.changes()`
walk; build the `LLMRoundBundle`; assert every populated field produced ≥1
render call, iterating fields dynamically.

## Acceptance

- Automated: `tests/test_visibility_parity.py` passes (tools block renders;
  system folds when long, inline when short; reminder labelled; `[subagent]`
  label; the dynamic no-hidden-channel assertion). Full suite green; ruff clean.
  `.claude/rules/visibility-parity.md` exists.
- Manual: launch → `[tools] M schemas` folded block at start (tools on); `/show
  tools` prints the schema list; a long system prompt shows `[system] · N B`,
  `/show system` expands; a subagent result reads `[subagent]`.
- Ship in numeric order (tag after v0.10.1).

## Out of scope

- In-place toggle / `Ctrl+E`; UI editing; `/dump` whole-bundle; persisted fold
  state; retrieval/cache-snippet parity (rule covers them; impl when they exist).
