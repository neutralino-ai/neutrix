## Problem

Every tool the LLM calls runs unconditionally — including a destructive
`Bash` like `rm -rf`. A coding agent needs a safety layer: a way to block
dangerous operations and honor per-project allow/deny rules, `.claude/`-compatibly.

## Goal

1. **`PermissionPolicy`** (allow / deny / ask rules) loaded from
   `~/.claude/settings.json` + project `.claude/settings.json` +
   `.claude/settings.local.json` (CC's `permissions.{allow,deny,ask}` shape;
   rules `ToolName` or `ToolName(glob)` matched against the tool's primary arg).
2. **Two modes, default `auto`** — `auto` allows normal ops but blocks
   clearly-destructive Bash (deterministic heuristic: `rm -rf`, force-push,
   `dd`, `mkfs`, fork-bombs, `curl|sh`, `git reset --hard`, …) with a
   block-with-notice; `allow-all` (`/allow`) disables all checks.
3. **Gate in `Executor.dispatch_all`** before dispatch; a non-allow verdict
   yields a blocked `tool_finished` (the model adapts). Precedence: deny >
   explicit-allow (overrides auto danger) > auto-danger > ask > allow.

## Non-goals

- **Plan mode** — removed (user-directed).
- **Interactive approve-dialog** — deferred to the AskUserQuestion release
  (which builds the cross-thread input channel); for now `ask` → block-with-notice.
- **LLM-classifier `auto`** (CC's YOLO) — the deterministic danger heuristic is
  the v1.4.0 auto; a classifier is a later refinement.
- **A filesystem/network sandbox** — v1.14.0.

## Design

### `src/neutrix/permissions.py` (new — pure)
- `PermissionPolicy(allow, deny, ask)`; `load_policy(cwd, home)` merges settings.
- `_DANGEROUS_BASH_RE` + `_is_dangerous`; `decide(tool, args, *, mode="auto",
  policy)`; `block_reason(tool, verdict)`.

### `src/neutrix/executor.py`
- `policy: PermissionPolicy` + `permission_mode: str = "auto"`; the gate in
  `dispatch_all` (yield a blocked `tool_finished` on non-allow). Defaults keep
  existing tests unaffected (non-dangerous tools allowed).

### `src/neutrix/cli.py` + `terminal_chat.py`
- `cli`: `executor.policy = load_policy(os.getcwd())`.
- `_cmd_allow` toggles `auto` ↔ `allow-all`; `/allow` in help; status line
  shows `allow-all` when active.

## Acceptance

- Automated (`tests/test_permissions.py` + `test_terminal_chat.py`): auto allows
  normal ops + blocks the dangerous Bash set; explicit allow overrides danger;
  deny wins; ask rule → ask; allow-all disables checks; `load_policy` merges;
  the Executor gate blocks denied/dangerous and runs the rest; `/allow` toggles.
  Full suite green (343); ruff clean.
- Ship `v1.4.0`.

## Out of scope

- Plan mode (removed), interactive approval (→ AskUserQuestion), LLM-classifier
  auto, sandbox (v1.14.0).
