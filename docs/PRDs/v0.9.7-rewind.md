## Problem

The user has no way to:

- Re-issue a prior prompt with a small edit (e.g., fix a typo,
  change one constraint) without retyping it from scratch.
- Roll back the last N turns when the LLM went down a wrong
  path, and re-prompt from a known-good point.
- Inspect / step through history while keeping the option to
  return to the live tail.

CC has rewind: up-arrow on an empty buffer recalls prior user
turns; `Esc Esc` (or similar) lets you branch from any point.
neutrix has neither.

## Goal

1. **Up-arrow recall.** When the draft buffer is empty,
   `Up-arrow` loads the most recent user turn into the buffer
   for editing; successive `Up-arrow` presses walk further back;
   `Down-arrow` walks forward; `Esc` clears the recalled text and
   returns to a fresh draft.
2. **Rewind on submit.** When the user submits a recalled (or
   edited) prior turn, `ContextManager.messages` is rolled back
   to the index of that turn — i.e., everything after the recalled
   turn is dropped from history. The submitted text becomes the
   new last user turn; the LLM runs against the rewound history.
3. **`/rewind N`** slash command — drops the last N user turn
   pairs (user + assistant + tool-rounds) without recalling text.
   Useful when the user wants to type fresh from an earlier point.
4. **Rewind notice (revised — scope change, see Claude Code reference).**
   neutrix's renderer is append-only and cannot retroactively restyle or
   un-print the dropped turns, so the original `[rewound]` strikethrough
   is infeasible. Instead a forward notice — `↶ rewound N turns (back to
   turn K)` — is printed at the cursor on rewind (the v0.9.6 `/compact`
   pattern). The dropped turns stay in scrollback as history; the LLM's
   view (`ContextManager.messages`) excludes them.

## Non-goals

- **Multi-branch tree UI.** One linear history; no `git`-style
  branches preserved in the live session. Saving to disk
  preserves only the active branch.
- **Cross-session rewind state.** Each session starts at the live
  tail; rewinds within a session do not survive `/save` + reload
  unless the user re-rewinds after loading.
- **Edit-in-place of prior assistant turns.** Rewind drops the
  assistant turn; it cannot be edited.

## Claude Code reference

See [`docs/splits/v0.9.7-rewind.html`](../splits/v0.9.7-rewind.html) for the
full survey (CC behavior + alternatives + per-split decision). Summary:

- **History shape (#1)** — Follow CC: **destructive trim**
  (`messages.slice(0, i)`, `REPL.tsx:3671`). CC's *rewind* is destructive
  (distinct from CC's *cancellation* append-marker — the v0.9.2/3 incident).
  *Flagged for confirmation; implemented as destructive in `rewind_to`.*
- **Recall ↔ rewind coupling (#2)** — recommend **decoupled** (Follow CC):
  Up-arrow only fills the buffer; rewind is explicit `/rewind N`. Skeleton
  Goal #2 wanted submit-recalled-turn to truncate; that makes a normal
  Enter destructive. *Flagged — needs user.*
- **`rewind_to` mechanism (#3)** — No CC API analog: new
  `ContextManager.rewind_to(index)` mirroring `compact()` (trim + rebuild
  store, preserve tasks). *Implemented.*
- **Recall source (#4)** — walk `role:user` messages in `cm.messages`
  (diverges from CC's history file; neutrix has none).
- **`/rewind N` (#5)** — Alternative: count, not CC's interactive selector.
- **Tool-round boundary (#6)** — Follow CC: drop whole rounds atomically;
  `_safe_rewind_index` snaps back so the kept head never ends on an
  unanswered `tool_calls` or dangling `role:tool`. *Implemented.*
- **Transcript after rewind (#7, SCOPE CHANGE)** — CC removes turns by
  re-rendering a virtual list; neutrix's append-only renderer cannot. Use a
  **forward notice** (see Goal #4). *Flagged — needs user.*
- **Esc (#8)** — recommend Esc clears recall when IDLE, stays cancel when
  busy. *Flagged — needs user.*
- **State gate (#9)** — IDLE-only, mirror `/compact`. **Up/Down bindings
  (#10)** — explicit, override the built-in `InMemoryHistory`.

## Design (sketch)

- Add `ContextManager.rewind_to(message_index: int) -> int`:
  trims `self.messages[message_index:]`, returns the number of
  messages dropped. Mirrors v0.9.2's earlier `rollback_to` that
  v0.9.3 removed — now reintroduced for user-driven rewind.
- New `terminal_chat` key binding: `Up` on empty buffer reads
  from a `history_cursor: int` walking backwards through
  user-role messages in `cm.messages`. The buffer fills with the
  recalled text; `Esc` resets cursor and clears buffer.
- New `/rewind N` command: `cm.rewind_to(len(cm.messages) - 2*N)`
  (roughly; tool-round-boundary aware), prints
  `rewound N turns`.
- Renderer: visibility-parity (v0.10.0) labels for rewound
  turns. Reuse the `<system-reminder>` / `<system-summary>`
  folding pattern with a new `[rewound]` prefix.

## Acceptance

- Manual:
  - Send 3 prompts. Press Up. Latest user turn fills the buffer.
    Press Up. Second-latest fills. Press Esc. Buffer clears.
    Press Up + edit + Enter. Messages list rolls back to that
    point; new user turn replaces; LLM responds.
  - `/rewind 2` drops the last 2 user turn pairs;
    `cm.messages` is shorter; transcript renders the dropped
    turns with `[rewound]` styling.
  - `/save` + reload: the saved transcript contains only the
    post-rewind state.
- Automated:
  - **[done]** `tests/test_rewind.py::test_rewind_to_drops_suffix` —
    `cm.rewind_to(5)` against a 10-message fixture leaves 5 messages.
  - **[done]** `test_rewind_respects_tool_round_boundary` — a rewind that
    would split a tool round drops the whole round (snaps to a clean
    boundary). Plus `test_rewind_at_turn_boundary_keeps_whole_round`,
    `test_rewind_noop_at_or_beyond_len`,
    `test_rewind_clamps_into_system_prefix`,
    `test_rewind_rebuilds_store_and_preserves_tasks`. All green (suite 219).
  - **[pending UI]** `test_up_arrow_walks_user_turns` — pressing Up N times
    yields the Nth-most-recent user turn.
  - **[pending UI]** `test_recall_then_esc_clears` — recall + Esc leaves the
    buffer empty and the cursor reset.
- Human gate before tag.

## Out of scope

- Branch-tree UI (post-v1.0).
- Edit-in-place of assistant turns.
- Cross-session rewind history.
