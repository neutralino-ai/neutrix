# v1.7.1 — config-driven pricing · 3-number cache display · remove onboard

## Problem

Live use of the v1.7.0 cost feature surfaced three issues:

1. **Prices were hand-estimated and the provenance was overclaimed.** `pricing.py`
   shipped a hardcoded USD table whose docstring claimed it was "seeded from
   LiteLLM's `model_prices…json`" — but the values were guesses, and materially
   wrong (e.g. `claude-opus-4-7` is really **$5 / $25** per Mtok, not the guessed
   **$15 / $75**). Baking *any* price data into neutrix's code is the root problem:
   it staleness-rots, it's USD-only, and it never matches what the user is actually
   billed (CNY, via the IHEP gateway).
2. **Cache tokens aren't visible where the user looks.** The status line and
   per-model line fold cache into a single `↑` (input + cache); a user "only sees
   up and down".
3. **Onboarding is broken.** `/onboard` (manage keys / slots) doesn't work.

## Goal

1. **Prices live in the config YAML — no price data in neutrix code** (user
   direction: *"write the price in yaml directly, no fixed code in neutrix"*).
   neutrix becomes pure mechanism: read a model's price from config, multiply by
   tokens. No vendored table, no LiteLLM coupling, no model-name mapping (prices
   are keyed by the **exact** model string the slot already uses). Any currency —
   the user types their real rates and a display symbol.
2. **3-number cache display — `hit · miss · output`** — in one merged readout
   (status line, `/cost`, per-model), capturing cache hits robustly across the
   provider field-name variants.
3. **Replace the broken interactive `onboard`** with a static, deterministic
   first-run bootstrap: ship an IHEP-only example config (empty key, frozen
   prices); copy it when config is missing; when the key is empty, tell the user
   to fill it and exit. No interactivity, no model needed.

## Non-goals

- **No price data or `litellm` (data *or* dep) in neutrix code.** Config is the
  single source of price truth. (We still *used* LiteLLM's public JSON once, as a
  reference, to pre-fill the user's config with real USD numbers — but nothing
  ships in the package.)
- **Not a fixed currency.** The config carries a display symbol; default `$`,
  trivially switched to `¥` by editing. (Supersedes the earlier CNY-vs-USD
  debate — currency is now just config.)
- **Not enabling prompt caching** (`cache_control`). The IHEP gateway already
  auto-caches (verified live); this is correct *capture + display*.

## Claude Code reference

No turn-by-turn split discussion: the user delegated ("you make decision
yourself"), rejected the clarifying AskUserQuestion, and steered by direct
instruction, including this final simplification. Decisions are recorded here and
in `docs/splits/v1.7.1-cost-fix.html`. CC prices from a **hardcoded** USD
per-million table (`utils/modelCost.ts`); neutrix deliberately **diverges** —
price data is user config, not code — because neutrix targets a CNY-billing
gateway with models whose real rates CC's table would never carry.

### Split: where prices live
- **Decision**: Alternative (user-directed) — **in the config YAML, keyed by the
  exact model string**; **zero price data in neutrix code**. A `pricing:` block:
  a `currency` symbol + a `models:` map of `model → {input, output, cache_read,
  cache_write}` (per **million** tokens). No vendored JSON, no `pricing_model`
  name-mapping, no `provider/`-strip — the key *is* the slot's model string.
  Missing model → `(cost unknown)`, tokens still shown. Reprice = edit the YAML.

### Split: currency
- **Decision**: User-directed — a **config display symbol** (default `$`). The
  numbers are whatever the user writes; pre-filled with real **USD** rates now,
  switchable to `¥` by editing the symbol + numbers. No currency logic in code.

### Split: cache token model — 3 numbers
- **Decision**: Alternative (learned from `dr.sai-nano/notebooks/deepseek_cache_explained.ipynb`)
  — display **`hit · miss · output`**: `hit = cache_read`; `miss = fresh input +
  cache_write`; `output`. `hit + miss = total input`, matching DeepSeek's native
  `prompt_cache_hit_tokens + prompt_cache_miss_tokens = prompt_tokens`. The 4
  normalized fields + `raw` stay internal (accurate pricing — `cache_write` still
  priced at its config rate — and **no JSONL/ledger churn**); only the *display*
  collapses to 3.

### Split: cache-capture robustness (the real v1.7.0 gap)
- **Decision**: Follow the **live-verified** field shapes. `_usage_from_openai`
  reads the hit from the first present of `prompt_tokens_details.cached_tokens` /
  top-level `cache_read_tokens` (IHEP gateway) / `prompt_cache_hit_tokens` (direct
  DeepSeek), and the miss from `prompt_cache_miss_tokens` else `prompt − hit`.
  Anthropic unchanged. (Verified live: the gateway populates `cached_tokens` for
  DeepSeek on warm calls and auto-caches a large prefix for `anthropic/*`.)

### Split: first-run bootstrap (replaces the broken interactive `onboard`)
- **Decision**: User-directed — **a static example config + a fill-the-key
  message, no interactivity** (the interactive `onboard` is what broke; copying a
  file + printing a message has nothing to break, and needs no LLM/model, so the
  init chicken-and-egg dissolves). `cli.py` already does the *missing-config* case
  (`bootstrap_config()` writes the template, prints "add an api_key, then re-run",
  exits). This PRD: (1) **the empty-key / no-usable-slot case** (currently the
  broken `run_onboarding` fallback) prints the **same** "fill your key at
  `<config path>` and re-run" message and exits — no onboarding; (2) the shipped
  `DEFAULT_CONFIG` template is **trimmed to IHEP-only** and carries the **frozen
  `pricing:` block** (USD); (3) **`onboard` is deleted** (`onboard.py`,
  `/onboard`, its help line + busy-gate entry). *Researched opencode's "free model
  without registration" (it runs its own funded `zen` gateway with anonymous/free
  allowlisted models — not a client-side key trick, not replicable, and
  free-riding it violates their ToS); the deterministic example-config path is the
  correct neutrix answer.*

## Design

1. **`config.py`** — parse an optional top-level `pricing:` section into a
   `PriceTable` (`{model: Price}` + `currency: str`), exposed via `Config`. `Price`
   = `input/output/cache_read/cache_write` per **million** tokens (missing → 0).
   Pre-fill the **default config template** *and* the user's
   `~/.config/neutrix/config.yaml` with the real USD numbers (below).
2. **`pricing.py`** — shrink to pure mechanism: a `Price` dataclass and
   `cost(usage, price) -> float` (Σ over the 4 classes / 1e6). No table, no data,
   no name resolution. The old hardcoded `_PRICES` and the misleading docstring
   are deleted.
3. **`llm.py`** — extend `_usage_from_openai` per the capture-robustness split
   (field-variant hit/miss) + a regression test for the direct-DeepSeek shape.
4. **`cost_ledger.py` + `terminal_chat.py`** — the `CostLedger` holds the injected
   `PriceTable` (from config) + currency; `cost()` / `by_model` look up each
   model's `Price` (None → unknown); a `hit/miss/output` view (`miss = input +
   cache_write`) drives the merged status-line / `/cost` / per-model readout with
   the config currency symbol. `from_jsonl` unchanged; the table is injected after
   rebuild (reprice-on-read = the next session reads the edited YAML).
5. **First-run bootstrap + remove onboard** —
   - `config.py`: trim `DEFAULT_CONFIG` to **IHEP-only** (one provider, `fast`/
     `strong` slots on `anthropic/claude-haiku-4-5` / `claude-opus-4-7`, `api_key:
     ""`) and embed the **frozen `pricing:` block** (the USD numbers above; the
     map covers the gateway-served models so any slot model is priced). This is
     the "example config" copied on first run.
   - `cli.py`: keep the existing *missing-config* path (`bootstrap_config()` →
     "add an api_key, re-run" → exit 0); **replace the `run_onboarding` fallback**
     (empty-key / no-usable-slot) with a clear "your api_key is empty — fill it at
     `<config.path>` and re-run" message + exit (non-zero). No interactive flow.
   - Delete `onboard.py`, `_cmd_onboard`, the `/onboard` help line + busy-gate
     entry, and the `from neutrix.onboard import run_onboarding` calls. Remove
     onboard tests; add a test that the empty-key path prints the fill-key message
     and exits without launching anything.

**Real USD numbers to pre-fill (per Mtok, from LiteLLM's public JSON, 2026):**

## Acceptance

1. A model priced in config yields the expected cost over the 4 classes; a model
   **absent** from the config `pricing.models` → `None` → `(cost unknown)` (tokens
   still shown). **No price data remains in neutrix code** (grep: `pricing.py` has
   no model rates).
2. Editing a number in the config YAML reprices that model on the next session
   (dollars computed on read from the injected table).
3. The cache hit is captured from **both** the direct-DeepSeek
   `prompt_cache_hit_tokens` shape (regression test) and the gateway
   `cached_tokens` shape; `hit + miss = total input`.
4. The status line + `/cost` + per-model show 3 numbers (`hit · miss · output`)
   with the config currency symbol; `/cost` retains the full 4-class detail.
5. **Bootstrap + onboard removal:** the shipped `DEFAULT_CONFIG` is IHEP-only with
   `api_key: ""` and the frozen `pricing:` block; a missing config is written +
   the user told to add a key (existing behavior); an **empty-key** start prints
   "fill your key at `<path>` and re-run" + exits (no `run_onboarding`); `onboard.py`
   / `/onboard` / its help + busy-gate are gone; onboard tests removed, empty-key
   message tested; full `pytest` green; `ruff` clean.
6. The default config template + the user's config carry the `pricing:` block with
   the real USD numbers + `currency: "$"`.
