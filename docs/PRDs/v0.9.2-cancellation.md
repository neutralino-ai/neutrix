# v0.9.2 — Esc / Ctrl+C cancellation + Executor / LLM / Controller decomposition

## Problem

After v0.9.1, the keyboard responds to Ctrl+C (two-tap exit) and
Ctrl+Z (suspend), but there is no way to **interrupt** an in-flight
LLM round or tool dispatch and return cleanly to the REPL's idle
state. Today the user can either:

- Wait for the assistant to finish (could be minutes for a long
  tool run, hours for a runaway `sleep`);
- Double-tap Ctrl+C to exit the whole process (loses session state,
  loses queued user messages, may leave subprocesses orphaned).

Codex (Rust, ratatui) treats `Esc` as the universal "stop and go
back to idle" key when a task is running. Claude Code (TypeScript)
threads an `AbortController` through every tool's `ToolUseContext`;
its `BashTool` registers an `abort` listener on the signal and
calls `tree-kill` with `SIGKILL` on cancel — the subprocess tree
dies, the tool returns with `interrupted: true`, conversation
history stays valid.

v0.9.0 already prepared the generator-side cancellation path
(`Agent.stream_reply` is PEP 525-safe — `.aclose()` does not raise
`RuntimeError`). What is missing is:

1. A **controller-side cancel handle** wired to a key (Esc / first
   Ctrl+C while busy).
2. A **task pool** that knows what is in flight (current LLM round,
   any cancellable subprocess tools) so it can clean up atomically.
3. **History integrity** — if cancel fires after the LLM emits
   `tool_calls` but before all tool results arrive, the next OpenAI
   request will 400 with "messages with tool_calls must be followed
   by tool messages." Today's design has no guard.
4. **A clean architecture to do this on.** Today `Agent` builds its
   own LLM, owns the message list, runs the loop, dispatches tools,
   and is reached into by the view. Adding cancellation on top of
   that monolith would be either ugly (state on Agent) or fragile
   (back-reference cycles). The decomposition is part of v0.9.2,
   not a follow-on.

## Architectural model (in-process services)

The mental model is **service-oriented**: each component has one
job, a narrow command interface, and is reachable only through that
interface. Physically everything is in one Python process — async
generators and method calls, not RPC. The boundaries are real even
though the wire is implementation detail; if/when a component
(notably `Advisor` in v0.11.0) needs to be a hosted service, only
the call site changes, not the contract.

```
            ChatStore                ← canonical state surface
                ↑                       (UI reads; controllers write
                │                        through events)
                │ reads
                │
            TerminalChat (UI)
                ↓ commands only
            Controller
                │  cancel() broadcasts to ↓
        ┌───────┼────────────┬───────────┐
        ▼       ▼            ▼           ▼
       LLM   Executor    (Advisor       (future
      .stop() .cancel()    .cancel()     services)
                │           v0.11.0)
                ▼
              Agent (message router; messages + stream_reply +
                     rollback_to). Agent is stateless w.r.t.
                     in-flight execution; its one state is the
                     conversation history.
```

**Data-flow contract**:

- **Commands** flow UI → Controller → {LLM, Executor, …}. Always
  method calls; always idempotent; never read state back.
- **State** flows {LLM, Executor, Agent} → ChatStore → UI. Always
  via the v0.9.0 `store.apply(event)` reducer; never polled
  directly.

The view does NOT call `executor.is_busy()`, `llm.is_streaming()`,
or any predicate on a service. If it needs to know what's in
flight, it reads `store.llm_active` / `store.pending_tool_calls`
(per v0.9.0). For cancellation, it checks its OWN field —
`self._current_stream_task` — which it created.

## Goal

1. **`LLM.stop()`** added to the `ChatLLM` Protocol and the
   `OpenAIChatLLM` implementation. `stop()` aborts any in-flight
   `stream_response` cleanly: closes the underlying OpenAI
   `AsyncStream`, so the next `__anext__` raises `StopAsyncIteration`
   and the wrapping `async for` exits. Idempotent — no-op when no
   stream is in flight.
2. **`Executor`** — new class in `src/neutrix/executor.py`. Owns
   the in-flight turn:
   - `_pool: list[subprocess.Popen]` — cancellable processes
     registered by tools that own one.
   - `_turn_snapshot_len: int | None` — `len(agent.messages)` at
     the top of the turn, for rollback.
   - `_cancel_requested: bool` — flag tools may consult.

   API (commands only, no predicates):
   - `stream_turn(user_text) -> AsyncIterator[AgentEvent]` —
     manages the turn lifecycle (snapshot, forward to
     `agent.stream_reply`, post-turn cleanup).
   - `cancel() -> None` — idempotent. No-op if no turn is in
     flight. When in flight: tree-kills every registered Popen,
     sets the cancel flag, rolls `agent.messages` back to the
     snapshot.
   - `register_cancellable(proc)` / `unregister_cancellable(proc)`
     — tool-facing.
3. **`Controller`** — new class in `src/neutrix/controller.py`.
   The single command surface the view drives. Owns:
   - `agent`, `executor`, `llm`, `store` (all injected),
   - `_current_stream_task: asyncio.Task | None`.

   API:
   - `send(user_text) -> None` — creates the task that iterates
     `executor.stream_turn(user_text)`, forwards each event
     through `store.apply(event)` AND through a view-injected
     event sink for rendering, and clears `_current_stream_task`
     on completion.
   - `cancel() -> bool` — broadcasts to subordinates and reports
     whether anything was actually cancelled:
     ```python
     def cancel(self) -> bool:
         task = self._current_stream_task
         if task is None or task.done():
             return False
         self.llm.stop()        # close HTTP stream
         self.executor.cancel() # kill pool + rollback messages
         task.cancel()          # close async generator
         return True
     ```
     Each subordinate's `stop()` / `cancel()` is independently
     idempotent — controller doesn't ask "are you busy?", just
     broadcasts. Designed so v0.11.0's `Advisor` plugs in as a
     fourth broadcast target by adding one line.
4. **`Agent` gains exactly one new method**: `rollback_to(n: int)`.
   Trims `self.messages[n:]`. That's the only seam `Executor`
   uses; no other state moves to Agent.
5. **`Agent` stops building its own LLM.** Its constructor
   accepts `llm: ChatLLM` (injected). `TerminalChat` (or the
   bootstrap glue) constructs the LLM once and passes the same
   instance to both `Agent` and `Controller`. The `Agent.switch`
   path stays (it forwards to `llm.switch(slot)`).
6. **`Esc` binding**: unconditional, `eager=True`. Calls
   `chat.controller.cancel()` directly. No filter. No state
   read.
   - **Trade-off accepted: all `Alt-*` combos are lost.** `eager`
     swallows the meta-prefix before prompt_toolkit can compose
     `Alt+B` (back-word), `Alt+F` (forward-word), `Alt+D`
     (delete-word), `Alt+Enter` (v0.9.1's newline). Replacements
     for the load-bearing one: `Ctrl+J` for newline (v0.9.1
     binding, unchanged). Word-motion via Alt is gone.
     v0.9.1's `escape, enter` binding is deleted as part of
     v0.9.2 (not silently left as dead code).
7. **First `Ctrl+C` while a turn is in flight**: same handler
   path. Calls `controller.cancel()`; if it returns True (something
   was cancelled), done. If False (idle), falls through to
   v0.9.1's arm-or-exit logic. Does NOT arm the quit shortcut
   while cancelling.
8. **`run_shell` registers its Popen with the executor.** Rewrite
   from `subprocess.run` to `subprocess.Popen(start_new_session=
   True)` + `communicate`, with `register_cancellable(proc)`
   before the wait and `unregister_cancellable(proc)` in the
   `finally`. On cancel, the Popen's process group gets SIGTERM
   then SIGKILL (Python analog of Claude Code's `tree-kill`).
9. **Idle-state contract after cancellation**: atomically
   - `executor._pool` is empty,
   - `agent.messages` rolled back to the pre-turn snapshot
     (drops user_turn + any partial assistant turn + any orphan
     tool_call),
   - `store._llm_active = False`,
   - `store.clear_pending_tool_calls()`,
   - the **queued user messages survive** (per v0.9.1's queue)
     and are sent next when the worker loop resumes,
   - a dim `interrupted` notice is printed in the transcript,
   - the worker loop returns to `await queue.get()`.
10. **Two-tap `Ctrl+D`**: unchanged from v0.9.1 (empty-buffer
    only; non-empty buffer keeps forward-delete-character).

## Non-goals

- **Multi-process / RPC implementation.** The service-oriented
  *interfaces* land in v0.9.2; the *transport* stays in-process.
  Moving any component to a subprocess or HTTP service is a
  v1.x story (web frontend) and not earned by today's
  single-user-single-process reality. The interfaces are designed
  so that swap is mechanical, not a rewrite.
- **`Advisor` itself.** v0.11.0. The decomposition leaves a
  one-line broadcast slot for it in `Controller.cancel()`.
- **Cooperative cancellation of arbitrary tools.** Pure-compute
  tools (FileRead, ListDir, TaskCreate, …) run via
  `asyncio.to_thread`; cancelling the outer task does not kill
  the running thread. Accepted: they complete in the background;
  their results are dropped silently (the rollback covers
  history). Only tools that **register a Popen with the
  executor's pool** (today: `run_shell`) actually stop. A
  generic `CancellableTool` protocol with a polling
  `cancel_event` is later (likely v0.10.x).
- **Backgrounding subprocesses on "interrupt" reason** (Claude
  Code's two-mode abort). v0.9.2 has one cancel mode —
  Esc / first-Ctrl+C — and it always means **kill**.
- **Saving the partial assistant turn for steering.** Codex saves
  the partial text. For v0.9.2 we rollback the whole turn; the
  user re-prompts.
- **Cancelling the v0.8.0 task-reminder injection.** The reminder
  is a message append at the start of `stream_reply`, covered by
  the per-turn snapshot rollback.
- **Cancelling slash commands (`/save`, `/load`, `/switch`).**
  These don't run through `controller.send` and don't register
  with the pool.
- **Renaming `agent_loop.py` / `Agent`.** Same call as v0.9.0.
  The new `Controller` lives alongside; rename can ride v0.10.0
  if still warranted.
- **The legacy `tui.py` Textual app.** Same non-goal as v0.9.1.

## Design

### `LLM.stop()` (`src/neutrix/llm.py`)

`ChatLLM` Protocol gains:

```python
class ChatLLM(Protocol):
    def switch(self, slot: Slot) -> None: ...
    def stream_response(self, *, ...) -> AsyncIterator[LLMEvent]: ...
    def stop(self) -> None:
        """Abort any in-flight stream_response.

        Closes the underlying transport so the next ``__anext__``
        on the iterator surfaces ``StopAsyncIteration`` and the
        wrapping ``async for`` exits cleanly. Idempotent — no-op
        when no stream is in flight."""
```

`OpenAIChatLLM` holds a handle to the active `AsyncStream`
returned by the OpenAI SDK during `stream_response`:

```python
class OpenAIChatLLM:
    def __init__(self, slot: Slot) -> None:
        ...
        self._active_stream: AsyncStream | None = None

    async def stream_response(self, *, ...) -> AsyncIterator[LLMEvent]:
        stream = await self._client.chat.completions.create(
            stream=True, ...
        )
        self._active_stream = stream
        try:
            async for chunk in stream:
                yield LLMEvent(...)
        finally:
            # PEP 525 safe — no yield in finally.
            self._active_stream = None

    def stop(self) -> None:
        stream = self._active_stream
        if stream is None:
            return
        try:
            stream.close()  # OpenAI SDK AsyncStream.close
        except Exception:
            pass  # best-effort; the broadcast must not raise
```

`stop()` may be called from any task (including the asyncio task
that's currently iterating the stream — that's why it's
synchronous). `stream.close()` is the OpenAI SDK's
explicit-teardown method; it interrupts the HTTP read so the
iterator's next `__anext__` raises immediately rather than waiting
for the next chunk.

### `Agent.rollback_to` (`src/neutrix/agent_loop.py`)

```python
def rollback_to(self, snapshot_len: int) -> None:
    """Trim self.messages to snapshot_len.

    Used by Executor on cancellation to drop the current turn so
    the conversation stays valid (e.g. no orphan tool_call in
    history that would 400 the next OpenAI request)."""
    if 0 <= snapshot_len < len(self.messages):
        del self.messages[snapshot_len:]
```

`Agent.__init__` is changed so `llm` is required, not auto-built:

```python
@dataclass
class Agent:
    slot: Slot
    llm: ChatLLM                     # now required, was optional
    system_prompt: str = DEFAULT_SYSTEM_PROMPT
    ...

    def __post_init__(self) -> None:
        if not self.messages:
            self.messages = [{"role": "system", "content": self.system_prompt}]
        # no more `if self.llm is None: self.llm = OpenAIChatLLM(self.slot)`
```

All construction sites pass an LLM. The bootstrap (in
`terminal_chat.py` or wherever Agent is created) becomes:

```python
llm = OpenAIChatLLM(slot)
agent = Agent(slot=slot, llm=llm, ...)
executor = Executor(agent=agent)
controller = Controller(agent=agent, executor=executor, llm=llm, store=store)
```

### `Executor` class (`src/neutrix/executor.py`, new file)

```python
@dataclass
class Executor:
    """Owns the in-flight turn: pool of cancellable processes,
    per-turn snapshot for history rollback, single cancel entry.

    Wraps an Agent. Agent is the message router; Executor is the
    stateful surface the controller drives for one turn."""

    agent: Agent
    _pool: list[subprocess.Popen] = field(default_factory=list, repr=False)
    _turn_snapshot_len: int | None = field(default=None, repr=False)
    _cancel_requested: bool = field(default=False, repr=False)

    def register_cancellable(self, proc: subprocess.Popen) -> None:
        self._pool.append(proc)

    def unregister_cancellable(self, proc: subprocess.Popen) -> None:
        try:
            self._pool.remove(proc)
        except ValueError:
            pass

    def cancel(self) -> None:
        """Idempotent. No-op if no turn is in flight."""
        if self._turn_snapshot_len is None:
            return
        self._cancel_requested = True
        for proc in list(self._pool):
            _tree_kill(proc)
        self._pool.clear()
        self.agent.rollback_to(self._turn_snapshot_len)

    async def stream_turn(
        self, user_text: str
    ) -> AsyncIterator[AgentEvent]:
        self._turn_snapshot_len = len(self.agent.messages)
        self._cancel_requested = False
        try:
            async for event in self.agent.stream_reply(
                user_text, executor=self
            ):
                yield event
        finally:
            # PEP 525: no yield in finally — pure assignment only.
            self._turn_snapshot_len = None
            self._cancel_requested = False
            self._pool.clear()  # defensive
```

### `_tree_kill` helper (`src/neutrix/executor.py`)

POSIX-only. Python analog of Claude Code's `tree-kill`.

```python
def _tree_kill(proc: subprocess.Popen, grace_seconds: float = 0.2) -> None:
    if proc.poll() is not None:
        return
    try:
        os.killpg(os.getpgid(proc.pid), signal.SIGTERM)
    except (ProcessLookupError, PermissionError):
        return
    try:
        proc.wait(timeout=grace_seconds)
        return
    except subprocess.TimeoutExpired:
        pass
    try:
        os.killpg(os.getpgid(proc.pid), signal.SIGKILL)
    except ProcessLookupError:
        pass
```

### `Controller` class (`src/neutrix/controller.py`, new file)

```python
@dataclass
class Controller:
    """Single command surface the view drives. Broadcasts cancel
    to its subordinates (llm, executor, future advisor); never
    polls their state."""

    agent: Agent
    executor: Executor
    llm: ChatLLM
    store: ChatStore
    event_sink: Callable[[AgentEvent], Awaitable[None]] | None = None
    _current_stream_task: asyncio.Task | None = field(
        default=None, init=False, repr=False
    )

    async def send(self, user_text: str) -> None:
        async def _drive() -> None:
            agen = self.executor.stream_turn(user_text)
            try:
                async for event in agen:
                    self.store.apply(event)
                    if self.event_sink is not None:
                        await self.event_sink(event)
            finally:
                await agen.aclose()

        self._current_stream_task = asyncio.create_task(_drive())
        try:
            await self._current_stream_task
        finally:
            self._current_stream_task = None

    def cancel(self) -> bool:
        """Broadcast cancel. Returns True iff anything was in flight."""
        task = self._current_stream_task
        if task is None or task.done():
            return False
        self.llm.stop()
        self.executor.cancel()
        task.cancel()
        # Future: self.advisor.cancel()   # v0.11.0
        return True
```

`event_sink` is the view-injected hook for rendering events
(`_handle_event` in `TerminalChat`). Controller doesn't know
what rendering means; it just calls back for each event.

### `run_shell` registers with the executor (`src/neutrix/tools.py`)

```python
def _run_shell(
    command: str,
    timeout: int = 30,
    *,
    executor: Executor | None = None,
) -> str:
    logger.info("shell tool: {!r}", command)
    proc = subprocess.Popen(
        command,
        shell=True,
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
        text=True,
        cwd=os.getcwd(),
        start_new_session=True,  # so killpg targets the whole tree
    )
    if executor is not None:
        executor.register_cancellable(proc)
    try:
        out, err = proc.communicate(timeout=timeout)
    except subprocess.TimeoutExpired:
        proc.kill()
        out, err = proc.communicate()
        return f"ERROR: command timed out after {timeout}s"
    finally:
        if executor is not None:
            executor.unregister_cancellable(proc)
    if executor is not None and executor._cancel_requested and proc.returncode < 0:
        return "[cancelled by user]"
    parts = [f"exit_code: {proc.returncode}"]
    if out:
        parts.append(f"stdout:\n{out}")
    if err:
        parts.append(f"stderr:\n{err}")
    return "\n".join(parts)
```

`get_schemas`'s JSON schema for `run_shell` stays `(command,
timeout)` — `executor` is dispatch-injected, not part of the LLM
contract.

### Dispatch wiring

`dispatch(name, arguments, store=..., executor=...)` grows an
`executor=` kwarg, forwarded to the tool function only if its
signature declares it. The existing store-injection logic
generalizes from "inject store" to "inject any declared kwarg
from this set."

`Agent.stream_reply(user_text, *, executor=None)` accepts an
optional `executor`. `Executor.stream_turn` passes `self`. The
tool dispatch shim threads `executor` to `dispatch(...)`. No
back-references; Agent never holds an Executor.

### `TerminalChat` slim-down

```python
class TerminalChat:
    def __init__(self, slot, store, view):
        self.slot = slot
        self.store = store
        self.view = view
        self.llm = OpenAIChatLLM(slot)
        self.agent = Agent(slot=slot, llm=self.llm, store=store)
        self.executor = Executor(agent=self.agent)
        self.controller = Controller(
            agent=self.agent,
            executor=self.executor,
            llm=self.llm,
            store=store,
            event_sink=self._handle_event_async,
        )
        self._current_stream_task = None  # mirrors controller's for the cancel handler
        ...

    async def _send_message(self, text: str) -> None:
        self.store.clear_pending_tool_calls()
        self._streaming_assistant = False
        # Track the controller's task so the key bindings can use
        # self._current_stream_task as their is-busy check.
        send_coro = self.controller.send(text)
        self._current_stream_task = asyncio.create_task(send_coro)
        try:
            await self._current_stream_task
        except asyncio.CancelledError:
            await self._on_stream_cancelled()
        finally:
            self._current_stream_task = None

    def try_cancel_current_stream(self) -> bool:
        """Called from key bindings. Returns True iff cancel fired."""
        return self.controller.cancel()

    async def _on_stream_cancelled(self) -> None:
        self.store.clear_pending_tool_calls()
        if self.store.llm_active:
            from neutrix.agent_loop import AgentEvent
            self.store.apply(AgentEvent("llm_request_end"))
        if self._streaming_assistant:
            await self.view.write_raw("\n")
            self._streaming_assistant = False
        await self.view.print_notice("interrupted", style="dim yellow")
```

Note the small redundancy: the view tracks `_current_stream_task`
in addition to the controller. That's because `controller.send`
awaits its inner task to completion (the controller-side task is
gone by the time `cancel()` looks at it). The view's outer task
is the cancellation lever. This separation is intentional — the
controller owns "what's in flight" for its own send-loop; the
view owns the asyncio handle that the OS-event (Esc) cancels.

### Esc binding (`src/neutrix/terminal_chat.py`)

```python
def build_draft_key_bindings(quit_state, chat):
    # ... existing v0.9.1 bindings ...

    @bindings.add("escape", eager=True)
    def _(event) -> None:
        chat.try_cancel_current_stream()

    return bindings
```

No filter. The handler always fires; `try_cancel_current_stream`
no-ops when nothing's in flight. `eager=True` cuts the
meta-prefix wait at the cost of losing all `Alt-*` combos.

Delete v0.9.1's `escape, enter` Alt+Enter binding — it cannot
compose under eager Esc; keeping the dead binding around would
mislead future readers.

### `Ctrl+C` precedence

The v0.9.1 `c-c` handler grows one branch up top:

```python
@bindings.add("c-c")
def _(event) -> None:
    if chat.try_cancel_current_stream():
        return
    _arm_or_exit(event, "c-c", KeyboardInterrupt)
```

### Queued user messages

Untouched by cancellation. The v0.9.1 queue lives on the worker
side of `await queue.get()`; cancelling the view's
`_current_stream_task` does not drain it. After
`_on_stream_cancelled` returns, the worker loop's next
`await queue.get()` pops the next queued message and starts a
fresh `_send_message`.

## Acceptance

### Automated

`tests/test_llm.py` gains:

- **`OpenAIChatLLM.stop()` on idle is a no-op.** No exception.
- **`OpenAIChatLLM.stop()` on an active stream closes it.** Mock
  the OpenAI SDK's AsyncStream. Begin a `stream_response`
  iteration on a task; call `.stop()`; assert the mock's
  `.close()` was called and the iterator's next `__anext__`
  surfaces `StopAsyncIteration` (or the wrapping `async for`
  exits) within a short budget.

`tests/test_executor.py` (new):

- **In-flight snapshot is set during a turn.** Construct an
  `Agent` with a fake LLM. Start `executor.stream_turn("hi")`
  and pump one yield. Assert `executor._turn_snapshot_len is
  not None` (internal — this test is the one place we assert
  the invariant). After normal unwind, assert it's None.
- **Snapshot/rollback unit test.** Fake LLM that emits two
  tokens. Iterate `stream_turn` until just before completion;
  call `executor.cancel()`. Assert `agent.messages` is back to
  the pre-call length, pool empty.
- **Orphan-tool_call regression gate.** Fake LLM emits one
  `tool_call` (forcing `assistant_msg` with `tool_calls` to be
  appended). Cancel mid-dispatch. After unwind, assert
  `agent.messages` contains no entry with `role == "assistant"`
  and `tool_calls` — the rollback removed the orphan. Asserts
  the OpenAI 400 cannot recur.
- **`cancel` is idempotent when idle.** No-op on fresh Executor.
- **`run_shell` Popen registration + tree-kill.** Spawn
  `_run_shell("sleep 30", executor=executor)` on a thread.
  Poll-assert `len(executor._pool) == 1` within a short budget.
  Call `executor.cancel()`. Assert the process is killed
  within `_tree_kill`'s grace + a margin (~1 s). Pool empty.
  Skipped on Windows.
- **Non-cancellable tool leaks but doesn't poison history.**
  Fake tool with `time.sleep(2)`, no executor registration.
  Cancel during dispatch. Assert tool thread keeps running
  (sentinel that flips after sleep) but `agent.messages` is
  rolled back.

`tests/test_controller.py` (new):

- **`Controller.cancel()` broadcasts in order.** Mock LLM and
  Executor. Set up a fake in-flight task. Call `cancel()`.
  Assert `llm.stop()`, `executor.cancel()`, `task.cancel()`
  were each called exactly once, in that order.
- **`Controller.cancel()` on idle returns False, broadcasts
  nothing.** No `llm.stop`, no `executor.cancel`, no
  `task.cancel`. Returns False.
- **`Controller.send` end-to-end.** Fake LLM + fake Executor.
  Events from `stream_turn` are applied to the store in order
  AND passed to `event_sink` in order. `_current_stream_task`
  is cleared after completion.
- **Future-extension proof.** Add a synthetic `extra_target`
  with a `stop()` method to the controller for one test; assert
  it's called on `cancel()` when it exists. (Documents the
  v0.11.0 Advisor extension point.)

`tests/test_agent_loop.py` gains:

- **`rollback_to` unit test.** Append messages, call
