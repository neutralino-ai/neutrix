# v1.1.0 — coding tools (CC-shaped)

> **Status: finalized — release #1 of the v2 band** (full Claude-Code parity →
> `v2.0.0` lock). Split decisions confirmed by the user ("looks good"). Splits:
> [docs/splits/v1.1.0-coding-tools.html](../splits/v1.1.0-coding-tools.html).

## Problem

neutrix's tool set is too crude for real coding work: `read_file` (no ranges),
`write_file` (blind overwrite), `list_dir`, `run_shell`. There is no surgical
edit, no code search, no glob — the things a coding agent uses every turn. To be
"almost like Claude Code" (user-directed), the LLM-facing tool surface must
become CC-shaped: `Read` / `Edit` / `Write` / `Grep` / `Glob` / `Bash`.

## Goal

Replace the four crude tools with six CC-shaped ones (user chose **Replace**, not
add):

- **`Read`** — `cat -n` line-numbered output; `offset`/`limit`; a default line
  cap; records the path as read (for `Edit`/`Write` gating).
- **`Edit`** — `file_path`, `old_string`, `new_string`, `replace_all`; exact +
  unique match (else `replace_all`), `new_string` must differ; **requires a
  prior `Read`**; emits a one-line `± file: −N/+M` notice.
- **`Write`** — overwrite; **requires a prior `Read` if the file exists**.
- **`Grep`** — ripgrep-backed when `rg` is on PATH, else a pure-Python fallback;
  `pattern`, `path`, `glob`, `output_mode` (content/files_with_matches/count),
  `-i`, `-n`, context lines.
- **`Glob`** — pure-Python (`pathlib`), mtime-sorted, 100-file cap; `pattern`,
  `path`.
- **`Bash`** — the existing `run_shell` engine (tree-kill cancel + `timeout`)
  renamed; description nudges the model toward the dedicated tools.

## Non-goals

- **Multimodal `Read`** (images/PDF/notebooks) — v2.x stretch.
- **Hard-blocking `sed`/`grep`/etc. in `Bash`** — that's a permissions concern →
  **v1.4.0** (permissions/plan-mode); v1.1.0 only nudges via the description.
- **`run_in_background` / Monitor** — folds into the parallel-tools release.
- **Per-tool approval / diff-approval UI** — v1.4.0.
- **LSP / code intelligence** — v2.x stretch.

## Claude Code reference

See [docs/splits/v1.1.0-coding-tools.html](../splits/v1.1.0-coding-tools.html)
for the full survey + 7 split points. Decisions (all confirmed):

### Split point: read-before-edit + read-state home
- **CC**: Edit/Write require a prior Read (`readFileState`); `tools/FileEditTool/prompt.ts:4`.
- **Decision**: Follow CC — **enforce**; read-state lives on the per-session
  `Executor` (`Executor.read_paths: set[str]`), already forwarded to tools and
  per-subagent (correct isolation). Unread Edit/Write → `ERROR: read it first`.

### Split point: Grep backing
- **CC**: ripgrep (`tools/GrepTool/GrepTool.ts:33`).
- **Decision**: Alternative — `rg` if on PATH, else a Python `os.walk`+`re`
  fallback covering the common subset (pattern, glob, `-i`, `-n`,
  content/files modes).

### Split point: Bash banned commands
- **CC**: pushes off `sed/grep/cat/find`.
- **Decision**: Alternative — description-only nudge for v1.1.0; hard block →
  v1.4.0. Keep tree-kill cancel + `timeout`.

### Split point: Read / Edit / Glob shapes
- **Decision**: Follow CC — `Read` `cat -n` + `offset`/`limit` + line cap
  (multimodal deferred); `Edit` exact-unique + `replace_all` + must-differ +
  one-line `±` notice (no live in-place diff — append-only); `Glob` pure-Python,
  mtime-sorted, 100-cap.

### Split point: migration
- **Decision**: clean replace, no aliases; ship as a **minor** (`v1.1.0`) — the
  tool names are LLM-internal, not a public API, so renaming isn't a v1.0
  "stable surface" break. Update `subagent_tool_names()` + tests.

## Design

### `src/neutrix/executor.py`
- Add `read_paths: set[str]` (per-session). A `mark_read(path)` setter; tools
  check membership. Subagents get a fresh `Executor` → fresh read-state.

### `src/neutrix/tools.py`
- New tool impls + registry entries: `Read`, `Edit`, `Write`, `Grep`, `Glob`,
  `Bash`. Remove `read_file`/`write_file`/`list_dir`/`run_shell`.
- `Read` calls `executor.mark_read(abspath)`; `Edit`/`Write` require it (forward
  `executor`). `Bash` reuses `_run_shell`'s Popen/tree-kill, renamed.
- `Grep`: `rg` subprocess when available (parse `--json` or `-n` output), else
  Python fallback. `Glob`: `pathlib` + sort + cap.
- `subagent_tool_names()` updates to the new set minus `Agent`.

### `src/neutrix/terminal_chat.py`
- `Edit`/`Write`/tool-result rendering: a one-line `±` edit notice via the
  existing tool-result path (no new fold machinery).

### Tests (`tests/test_tools.py` / extend)
- Read ranges + line cap + marks-read; Edit unique/replace_all/must-differ +
  read-before-edit error; Write overwrite + read-before-overwrite; Grep
  rg-path + Python-fallback parity on a fixture tree; Glob mtime+cap; Bash
  rename keeps timeout/tree-kill. Subagent allowlist excludes `Agent`, includes
  the new tools. Full suite green; ruff clean.

## Acceptance

- Automated: the test set above; full suite green; ruff clean.
- Manual: the model can Read a file, Edit it surgically (and is refused if it
  didn't Read first), Grep the tree, Glob for files, run Bash with a timeout.
- Ship as `v1.1.0` (numeric order, first of the v2 band).

## Out of scope

- Multimodal, background bash, permissions/approval, LSP — later band releases.
