# v0.5.5 - onboarding: keep api_key masked after Enter

## Problem

In onboarding, pressing Enter in an `api_key` field already saves the
typed key and moves focus to the next row. The remaining user-visible
bug is that the field can appear empty after that focus move, even
though the key was written to YAML.

That is confusing because onboarding deliberately clears the visible
buffer on focus for fresh editing, but after a successful Enter the
field should no longer look like an empty draft. It should show a
masked committed value.

## Goal

After typing a key and pressing Enter:

- the provider's `api_key` is updated in memory and YAML;
- focus moves to the next focusable widget;
- the input renders a password mask for the committed key;
- the field does not show the `EMPTY ...` placeholder or a blank draft.

## Non-goals

- Show the raw API key after commit.
- Remove clear-on-focus editing behavior.
- Change empty Enter semantics: empty Enter still means "no change".
- Change verification or slot-assignment behavior.

## Design

Make `KeyInput` treat the committed value as the authoritative display
value whenever the widget is not focused:

1. Keep promoting the typed buffer to `_committed_value` in
   `action_submit()` before the Submitted message is posted.
2. Add a small `restore_committed_display()` helper that copies
   `_committed_value` back into `value` when the widget is blurred.
3. Call the helper from `on_blur()`.
4. After `OnboardScreen.on_input_submitted()` saves the key and calls
   `focus_next()`, call the helper immediately and once after refresh.
   This absorbs any late focus/editing event that temporarily empties
   the visible buffer after the commit path has completed.
5. Render password masks as `*` characters, matching the terminal
   convention users expect when they say the field should show `****`.

This keeps the existing editing model intact: while the field has
focus it can be empty for replacement, but once Enter commits and focus
leaves, the display returns to a masked saved value.

## Acceptance

- Typing an api_key and pressing Enter saves YAML, moves focus, and the
  input's rendered line contains a password mask.
- A simulated late empty buffer after submit is restored to the
  committed value before the user sees an empty field.
- Empty Enter with an existing key still restores the committed key and
  leaves YAML unchanged.
- Tab/Up/Down away from an uncommitted draft still reverts to the last
  committed key.
- Focused fields still clear for editing.
- `neutrix --version` resolves from the new `v0.5.5` tag after release.
