# v0.4.2 — refresh provider model catalogs (deepseek, glm, ihep)

## Problem

`PROVIDER_DEFAULT_MODELS` shipped names that no longer exist on the
upstream providers, so every onboarding `v`erify call would 404:

| Provider | Old (broken) | Status |
|---|---|---|
| deepseek | `deepseek-chat`, `deepseek-reasoner` | retired v3 names |
| glm | `glm-4.6`, `glm-4-plus`, `glm-4-flash` | superseded by 5.x line |
| ihep | `anthropic/claude-haiku-4-5`, `anthropic/claude-opus-4-7`, `anthropic/claude-sonnet-4-6`, `deepseek/deepseek-v3` | claudes still valid; deepseek path renamed; missing openai/zhipu/kimi entries the gateway routes |

## Goal

Replace each provider's catalog with the names the providers / gateway
actually serve today.

## Non-goals

- Schema changes (still `dict[str, list[str]]`).
- Auto-discovery via `/v1/models`.
- Backwards-compat: pre-1.0, old names just go away. Any user YAML that
  binds an old name on a slot keeps working as-is (we don't touch user
  config) but onboarding's curated catalog moves forward.
- Cost/quality ordering — keep entries in vendor-grouped order so users
  see related models adjacent.

## Design

```python
PROVIDER_DEFAULT_MODELS: dict[str, list[str]] = {
    "ihep": [
        "anthropic/claude-haiku-4-5",
        "anthropic/claude-opus-4-7",
        "anthropic/claude-sonnet-4-6",
        "openai/gpt-5.5",
        "deepseek-ai/deepseek-v4-pro",
        "deepseek-ai/deepseek-v4-flash",
        "zhipu/glm-5.1",
        "kimi/kimi-k2.6",
    ],
    "deepseek": ["deepseek-v4-flash", "deepseek-v4-pro"],
    "glm": ["glm-5.1", "glm-5.1-highspeed"],
}
```

Notes captured during review:
- `openai/gpt-image-2` was on the candidate list but excluded — it's an
  image-gen endpoint and would always fail `/v1/chat/completions`
  verification, polluting the onboarding screen with `✗` rows.
- Kimi is namespaced `kimi/kimi-k2.6` (matches the gateway's
  vendor/model pattern alongside `anthropic/`, `openai/`, `zhipu/`).
- The default-slot bootstrap (`fast` → `anthropic/claude-haiku-4-5`,
  `strong` → `anthropic/claude-opus-4-7`) stays — both are still in
  the catalog.

Sync the deepseek model name in the smoke-test fixture YAML
(`tests/test_smoke.py`) from `deepseek-chat` to `deepseek-v4-flash`.

## Acceptance

- `PROVIDER_DEFAULT_MODELS["deepseek"] == ["deepseek-v4-flash", "deepseek-v4-pro"]`.
- `PROVIDER_DEFAULT_MODELS["glm"] == ["glm-5.1", "glm-5.1-highspeed"]`.
- `PROVIDER_DEFAULT_MODELS["ihep"]` lists the 8 entries above, in order.
- `pytest` + ruff stay green.
- `neutrix --version` reports `0.4.2`.

## Out of scope

- A version-pinned catalog or "model is deprecated" warnings.
- Pulling the lists from a remote registry / discovery API.
- A migration that rewrites existing user YAML.
