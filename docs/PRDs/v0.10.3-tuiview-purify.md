## Problem

`terminal_chat.py` is mostly a reader of `ChatStore`, but one piece of state
still lives view-private: the folded-tool-result tray (`self._tool_records:
list[ToolRecord]`). As long as a view holds canonical-ish state, the v0.10.4
Advisor's "I only mutate the store" model has a hole ŌĆö a second actor can't see
or coordinate with view-private writes. This is the last bit of state on the
wrong side of the boundary.

## Goal

1. **Move the folded-tray state into the store.** `ToolRecord` becomes pure
   data in `src/neutrix/store.py`; the tray becomes
   `ChatStore.folded_tool_results` with an `add_folded_tool_result()` mutator
   (the store assigns the index). The view holds no tray list.
2. **Strip `ToolRecord` of rendering logic** ŌĆö its summary methods move to
   module-level functions in the view (`tool_record_summary[_parts]`), keeping
   `store` free of render dependencies.
3. **Prove the view is swappable** with `tests/fake_view.py`: a `FakeView` that
   renders the whole user surface from `ChatStore` alone, plus
   `tests/test_fake_view.py` scenarios.
4. **Enforce the boundary** where it's meaningful: `FakeView` imports only
   `neutrix.store` (asserted by AST parse) ŌĆö not a module-level grep on
   `terminal_chat.py` (infeasible; it hosts the orchestrator ŌĆö split #1).

## Non-goals

- **Web frontend / multi-session UI / transcript browser** (post-v1.0).
- **Splitting `TerminalView` into its own module** ŌĆö the FakeView proof makes it
  unnecessary; the orchestrator legitimately drives `ContextManager`.
- **Refactoring `onboard.py`, the heartbeat ticker, the TaskPanel.**
- **Behavior change** ŌĆö the transcript renders identically; this is hygiene.

## Claude Code reference

No CC analog ŌĆö CC's React/Ink view is coupled by design; this is a neutrix
store-purity contract. See
[docs/splits/v0.10.3-tuiview-purify.html](../splits/v0.10.3-tuiview-purify.html)
for the 4 decisions:
- **Boundary check**: Alternative ŌĆö scoped to a store-only `FakeView` (module
  grep on `terminal_chat.py` infeasible).
- **ToolRecord**: pure data in `store`; rendering moves to view functions.
- **Folded tray**: `store.folded_tool_results`, store-assigned index, wiped on
  `reset()`.
- **Swap-test assertions**: structured substrings, not exact bytes.

## Design

### `src/neutrix/store.py`
- `@dataclass(frozen=True) ToolRecord(index, name, arguments, result)` ŌĆö pure data.
- `ChatStore._folded_tool_results` + `folded_tool_results` property +
  `add_folded_tool_result(name, arguments, result) -> ToolRecord` (assigns
  1-based index) + cleared in `reset()`.

### `src/neutrix/terminal_chat.py`
- Import `ToolRecord` from `store`; delete the local class.
- `tool_record_keyword`/`tool_record_summary_body`/`tool_record_summary`/
  `tool_record_summary_parts` module functions (the `[subagent]` label lives in
  `tool_record_keyword`).
- `_store_and_print_tool_result` calls `store.add_folded_tool_result(...)`; the
  view-private `_tool_records` list is removed.
- `/tool [N]` reads `store.folded_tool_results`; `/clear` + `/load` no longer
  clear a view list (`store.reset()` does it).

### `tests/fake_view.py` + `tests/test_fake_view.py`
- `FakeView.render(store) -> str` reading only the store.
- Scenarios: empty, user+assistant, streaming pending text, tool round + folded
  tray, subagent label, reminder + cancel, tasks, reset wipes tray.
- `test_fake_view_imports_only_store`: AST-asserts the `neutrix` imports of
  `fake_view.py` are exactly `{neutrix.store}`.
