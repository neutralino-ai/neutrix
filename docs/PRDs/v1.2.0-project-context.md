# v1.2.0 — project context (CLAUDE.md + @-mentions + /init)

> **Status: finalized.** v2-band release #2. Decided autonomously under the
> `build-to-v2.0.0` /goal. Splits:
> [docs/splits/v1.2.0-project-context.html](../splits/v1.2.0-project-context.html).

## Problem

neutrix's agent has **no project memory**. The system prompt is a fixed
`DEFAULT_SYSTEM_PROMPT`; it never reads `CLAUDE.md`, has no way to pull a file
into context by reference, and can't bootstrap a project's conventions. This is
CC's killer coding feature and the largest single gap toward replacement.

## Goal

1. **Auto-load memory files** into the system prompt: user `~/.claude/CLAUDE.md`,
   project `CLAUDE.md` / `.claude/CLAUDE.md` / `AGENTS.md` (walking cwd → root,
   cwd-most highest priority), and `CLAUDE.local.md`. De-dup by resolved path.
   Support `@path` imports inside them (recursive, depth ≤ 5, skip code blocks,
   cycle-safe). Wrap with a "these instructions OVERRIDE default behavior" header.
2. **`@`-mentions** in a typed message inline the referenced file(s) into that
   user turn (`<file path="…">…</file>`), with `@path#L10-20` line ranges; a dir
   → a listing; missing → a note.
3. **`/init`** enqueues a fixed prompt that drives the agent (Read/Grep/Glob/Bash
   + Write) to survey the repo and write a concise `CLAUDE.md`.

`.claude/`-compatible (discovers CC's own files; drops into a CC project).

## Non-goals

- `.claude/rules/*.md` conditional (path-frontmatter) loading — needs the edit-event
  surface; v1.6.0 (hooks).
- The full 7-phase `/init` wizard (skills/hooks generation) — v1.3.0+.
- Multimodal `@`-mentions (images/PDF) — v1.11.0.
- Attaching `@`-files as separate multimodal blocks — we inline (OpenAI-compat).

## Design

### `src/neutrix/context_files.py` (new — pure, testable)
- `MEMORY_INSTRUCTION_PROMPT` (OVERRIDE header).
- `discover_memory_files(cwd, home) -> list[MemoryFile]` — priority-ordered.
- `_expand_imports(content, base_dir, *, depth=0, seen) -> str` — `@import`.
- `compose_memory_block(files) -> str`; `compose_system_prompt(base, cwd) -> str`.
- `expand_at_mentions(text, cwd) -> str` — inline `@path`(`#L..`)/dir/missing.

### Wiring
- `cli.py`: `system_prompt = compose_system_prompt(DEFAULT_SYSTEM_PROMPT, os.getcwd())`
  for the CM `system_prompt=` and the fresh seed message (when not `--load`).
- `terminal_chat.py`: `_worker_loop` expands `@`-mentions
  (`text = expand_at_mentions(text, cwd)`) after dequeue, before
  `handle_event(UserMessageEvent(text))`; new `_cmd_init` enqueues the init prompt;
  `/init` added to help.

## Acceptance

- Automated (`tests/test_context_files.py`): discovery order + de-dup on fixture
  dirs; `@import` recursion + depth-5 cap + code-block skip + cycle-safe; compose
  wrapper present; `expand_at_mentions` for file / `#L` range / dir / missing;
  `compose_system_prompt` no-ops cleanly when no memory files. `_cmd_init`
  enqueues a non-empty prompt. Full suite green; ruff clean.
- Manual: drop a `CLAUDE.md` in cwd → it shows folded in the system block
  (`/show system` expands); `@file.py` inlines the file; `/init` writes a CLAUDE.md.
- Ship `v1.2.0`.

## Out of scope

- Conditional rules, full /init wizard, multimodal mentions (later releases).
