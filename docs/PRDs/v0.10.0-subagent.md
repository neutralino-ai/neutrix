## Problem

The user can trim history (`/compact`, `/rewind`) but cannot delegate a
focused sub-task to a fresh-context worker. Reading ten files to produce a
one-paragraph summary inflates the main controller's context with all ten
files' contents; exploratory work that needn't live in the main history
still does. CC's `Agent` tool spawns a subagent with its own system prompt,
message list, and tool set, runs it to completion, and returns only the
final assistant text — the parent's context grows by one `tool_result`
instead of by the whole sub-task. This is the structural answer to
context-explode that mechanical/smart compaction only palliate.

## Goal

1. New `src/neutrix/subagent.py` with `run_subagent(...)`:
   - Builds a fresh `ContextManager` (its own `ChatStore`, its own
     `Executor`, the subagent system prompt, a scoped tool allowlist, and
     an `llm` built from the parent's slot).
   - Drives the existing CM round loop to completion (IDLE, final
     assistant text, no pending tool calls) — **reuses** the CM loop, does
     not reimplement it.
   - Returns `SubagentResult(final_text, turn_count, cancelled, error)`;
     `final_text` is capped at `SUBAGENT_MAX_RESULT_CHARS = 100_000`.
   - Bounded by `SUBAGENT_MAX_TURNS = 25` LLM rounds (runaway guard).
   - Cancellable via a `threading.Event` (cross-loop safe — see Design).
2. New `Agent` tool in `src/neutrix/tools.py`:
   - Schema: `Agent(description: str, prompt: str, subagent_type: str = "general-purpose")`
     (CC-shaped minimal core; `subagent_type` accepts only `general-purpose`
     in v0.10.0, others return an `ERROR:` string).
   - Sync dispatch (runs in the executor's worker thread): drives
     `asyncio.run(run_subagent(...))` in that thread's own event loop;
     builds a fresh `OpenAIChatLLM` from the parent slot; returns the
     subagent's final text as the `tool_result` content.
   - Recursion blocked two ways: the subagent's tool allowlist omits
     `Agent` (schema-scoping), plus a `threading.local` "inside-subagent"
     backstop that makes a nested `Agent` call return an `ERROR:` string.
3. **Cancellable.** Esc during a subagent run stops the subagent's
   token/subprocess burn (via the `threading.Event`); the parent receives
   `[cancelled by user]` through the *existing* v0.9.3 cancel path (marker
   + `_ensure_tool_result_pairing`), not through the Agent return.
4. **Visible, minimally (Goal #3 downgraded — see splits #5/#12).** The
   subagent's final result renders through the *existing* tool-result path
   (foldable like any tool result). v0.10.0 adds **zero** `terminal_chat.py`
   changes. Rich subagent rendering — a dedicated folded block with turn
   count and (eventually) per-round transcript + `Ctrl+E` — is **v0.10.2**
   (visibility parity), which owns the fold machinery and will extend
   `run_subagent`'s return shape if it needs turn-by-turn data.

## Non-goals

- **Parallel subagents.** One at a time. Parallel spawning is v1.x.
- **Subagent → subagent recursion.** Single-level only (enforced two ways).
- **Persistent subagent state across calls.** Each `Agent` call is fresh.
- **Custom subagent types** beyond `general-purpose`. v1.x.
- **Streaming subagent output to the parent.** Runs to completion; only the
  final string returns. (The parent's v0.9.8 heartbeat shows liveness.)
- **Per-round subagent transcript retention.** The subagent's `ChatStore`
  is local and discarded (split #12). v1.x may persist it.
- **Per-call model override** (CC's `model` field). v1.x; v0.10.0 reuses the
  parent slot.

## Claude Code reference

See [docs/splits/v0.10.0-subagent.html](../splits/v0.10.0-subagent.html) for
the full survey (CC `Agent`/`Task` dispatch) and all 12 split-point decisions.
All decisions were made autonomously under the delegating `/goal`. Highlights:

### Split point: tool schema — how wide
- **CC**: 10-field schema incl. background/teammate/worktree (`AgentTool.tsx:82-138`).
- **Decision**: Alternative — minimal CC-shaped core `Agent(description, prompt, subagent_type="general-purpose")`; the rest are out-of-scope (YAGNI).

### Split point: execution model
- **CC**: forked context drives the same query engine (`forkedAgent.ts`).
- **Decision**: Follow CC (semantics) — `run_subagent` reuses `ContextManager`'s round loop; no bespoke loop.

### Split point: tool allowlist + recursion
- **CC**: inherit minus a disallow-set; `Agent` blocked (`agentToolUtils.ts:70`, `AgentTool.tsx:332`).
- **Decision**: Follow CC — CM `tool_names` allowlist = builtins minus `Agent`; plus a `threading.local` runtime backstop.

### Split point: return packaging + size cap
- **CC**: final text only, capped 100k (`agentToolUtils.ts:229,276`).
- **Decision**: Follow CC — `tool_result` is `final_text` truncated to `SUBAGENT_MAX_RESULT_CHARS = 100_000`.

### Split point: rendering before v0.10.2's fold (#5) + transcript retention (#12)
- **CC**: folds subagent messages inline (`AgentTool.tsx:1084`).
- **Decision**: Alternative — defer ALL bespoke subagent rendering to v0.10.2; v0.10.0 renders the result as a normal tool result (zero `terminal_chat.py` changes); retain nothing per-round (v1.x).

### Split point: cancellation
- **CC**: sync agents share the parent AbortController (`AgentTool.tsx:1127`).
- **Decision**: Follow CC + reuse v0.9.3 — `threading.Event` (cross-loop safe) stops the subagent's burn; the `[cancelled by user]` marker reaches the parent through the existing v0.9.3 pairing path, not the Agent return.

### Split point: runaway guard (#8) + errors (#9) + system prompt (#10) + slot (#11)
- **Decisions**: `SUBAGENT_MAX_TURNS = 25` (Follow CC `maxTurns`); errors caught → `SubagentResult(error=...)` surfaced as `[subagent error: ...]`; dedicated `SUBAGENT_SYSTEM_PROMPT`; reuse parent slot (build fresh `OpenAIChatLLM` in-thread to avoid cross-loop client sharing).

## Design

### Core `ContextManager` additions (sequenced first, TDD, 234 stays green)
- `tool_names: frozenset[str] | None = None` — `None` = all builtins
  (unchanged behavior). When set, `_call_llm` advertises only those schemas.
- `max_rounds: int | None = None` — `None` = unbounded (unchanged). When set,
  `_drive` stops after that many LLM rounds.
- `__post_init__` sets `executor.slot = self.slot` so the dispatch layer can
  forward the slot to tools that declare it.

### `tools.py`
- `get_schemas(names: Collection[str] | None = None)` — scoped schema list.
- `Agent` tool (sync `func`): validates `subagent_type`; checks the
  `threading.local` recursion flag; builds a fresh `OpenAIChatLLM(slot)` and
  registers a `threading.Event` with the parent `executor`; runs
  `asyncio.run(run_subagent(...))`; returns `final_text` (or
  `[subagent error: ...]` / `[cancelled by user]` for the non-Esc paths).
- `dispatch(...)` forwards `slot` (and `executor`) to tools that declare them
  (same signature-inspection mechanism as `store`).
- `subagent_tool_names()` — builtins minus `Agent`.

### `executor.py`
- `slot: Slot | None = None` field (set by CM wiring), forwarded in
  `dispatch_all`.
- `register_cancel_event` / `unregister_cancel_event`; `cancel()` sets every
  registered `threading.Event` (alongside tree-killing the Popen pool).

### `subagent.py` (new)
- `SUBAGENT_SYSTEM_PROMPT`, `SUBAGENT_MAX_TURNS`, `SUBAGENT_MAX_RESULT_CHARS`.
- `@dataclass(frozen=True) SubagentResult(final_text, turn_count, cancelled, error)`.
- `async def run_subagent(*, user_prompt, slot, llm, tool_names, max_turns, cancel_event=None, system_prompt=SUBAGENT_SYSTEM_PROMPT) -> SubagentResult`:
  fresh `ContextManager`; a cancel-watcher task polling `cancel_event` →
  `cm.cancel()`; `await cm.handle_event(UserMessageEvent(user_prompt))`;
  extract last assistant text; cap; build the result.

## Acceptance

- Automated (`tests/test_subagent.py`):
  - `test_subagent_runs_to_completion` — stub LLM one-shot answer →
    `SubagentResult(final_text=..., turn_count=1, cancelled=False)`.

  - `test_max_rounds_caps_drive` — stops after N rounds.
