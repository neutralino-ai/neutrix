# v0.10.0 — subagent framework (Agent tool)

> **Status: skeleton.** User-reported as urgent for v1.0. The
> structural fix for context-explode that v0.9.5 (mechanical
> /compact) and v0.11.1 (smart compaction) only partially
> address: spawning a fresh-context worker that handles a focused
> sub-task and returns the result, leaving the main controller's
> context budget untouched. CC analog: the `Agent` tool that
> dispatches subagents. Slotted after v0.11.1 (smart compaction)
> because subagents reuse the LLM round + ContextManager
> machinery — best built on top of a stable core.
> Phase 1 splits HTML at `docs/splits/v0.12.0-subagent.html` when
> implementation begins.

## Problem

After v0.9.5 (mechanical /compact) and v0.11.1 (smart
compaction), the user can trim history but cannot:

- Spawn a focused sub-task ("read these 10 files and summarize
  them; return the summary") without inflating the main
  controller's context with the full file contents.
- Delegate exploratory work that doesn't need to live in the
  main conversation history.
- Parallelize independent sub-tasks (eventual scope; v0.12.0
  ships serial subagents only).

CC's `Agent` tool spawns a subagent with its own system prompt,
own message list, own tool set, runs it to completion, and
returns the final assistant text to the parent. The parent's
context grows by one tool-result message containing the final
text — orders of magnitude less than if the parent did the work
itself.

## Goal

1. New `neutrix.subagent` module with a `Subagent` class:
   - Owns its own `ContextManager` instance.
   - Receives a system prompt + initial user prompt + tool
     allowlist on spawn.
   - Runs the LLM/tool loop synchronously (within an outer
     async task) until the inner CM returns to IDLE with a final
     assistant message and no pending tool calls.
   - Returns the final assistant text as a string.
2. New `Agent` tool registered in `src/neutrix/tools.py`:
   - Schema: `Agent(description: str, prompt: str,
     subagent_type: str = "general-purpose")` (CC-shaped).
   - Dispatch: spawn a `Subagent`, run, return final text as the
     `tool_result` content.
3. **Visible to the user.** The subagent's full transcript —
   every LLM round, every tool call, every tool result — renders
   in the main transcript as a folded block (per visibility
   parity from v0.10.0). Default-folded with a one-line summary
   `↳ subagent: {description} — N turns, returned {K} chars`.
   `Ctrl+E` expands.
4. **Cancellable.** Esc during a subagent run cancels the
   subagent (broadcasts to its `ContextManager.cancel()`) and
   returns `[cancelled by user]` as the tool result. The parent
   controller proceeds with the cancel-as-steer semantic from
   v0.9.3.

## Non-goals

- **Parallel subagents.** v0.12.0 dispatches one subagent at a
  time. Parallel spawning is v1.x.
- **Subagent → subagent recursion.** A subagent cannot itself
  call the `Agent` tool. Single-level only.
- **Persistent subagent state across calls.** Each `Agent`
  invocation is fresh-context; no memory carries between
  invocations.
- **Custom subagent types** beyond `general-purpose` for
  v0.12.0. Specialized subagents (`code-review`,
  `documentation-writer`, etc.) are v1.x.
- **Streaming subagent output to the parent.** The subagent runs
  to completion; only the final string is returned.

## Design (sketch)

- `src/neutrix/subagent.py`:
  ```python
  @dataclass(frozen=True)
  class SubagentResult:
      final_text: str
      turn_count: int
      cancelled: bool
      error: str | None

  async def run_subagent(
      *,
      system_prompt: str,
      user_prompt: str,
      tools: tuple[ToolSpec, ...],
      llm: ChatLLM,
      cancel_token: asyncio.Event,
  ) -> SubagentResult:
      ...
  ```
- Reuses `ContextManager`, `Executor`, `LLM` from the parent's
  module set. The Subagent gets its own instance of each.
- `tools.py`: register `Agent` tool. Its dispatch path is the
  only place that calls `run_subagent`; pass the parent's `llm`
  and a fresh cancel token.
- Renderer: a new `[subagent]` block in `terminal_chat.py`,
  collapsed by default per visibility-parity rules.

## Acceptance

- Manual:
  - Send a prompt that triggers the LLM to call
    `Agent(prompt="read foo.py, bar.py, baz.py and summarize").
    The subagent runs (visible as a folded block), produces a
    summary, returns it; the parent LLM sees only the summary in
    its `tool_result` and continues.
  - Esc during the subagent run cancels it; parent receives
    `[cancelled by user]`; can be steered.
- Automated:
  - `tests/test_subagent.py::test_subagent_runs_to_completion` —
    mock-LLM that returns a one-shot answer; assert
    `run_subagent` returns `(final_text=..., turn_count=1)`.
  - `test_subagent_dispatches_tools` — subagent calls a tool,
    receives result, finalizes.
  - `test_subagent_cancel_returns_cancelled_marker` — fire the
    cancel token mid-run; assert `SubagentResult(cancelled=True)`.
  - `test_agent_tool_returns_subagent_output` — full path
    through the `Agent` tool dispatch.
- Human gate before tag.

## Out of scope

- Parallel subagents (v1.x).
- Recursive subagent dispatch (v1.x — needs design).
- Specialized subagent types (v1.x).
- Cross-subagent / cross-session subagent state (post-v1.0).
