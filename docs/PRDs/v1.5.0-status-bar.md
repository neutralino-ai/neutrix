# v1.5.0 — LLM / Exec / Advisor status bar

> **Status: finalized.** Decided autonomously under the `build-to-v2.0.0` /goal;
> **user-requested** after the v1.4.9 hang/stall debug ("maybe need a status bar
> showing if LLM/executor/advisor is busy/queuing"), design approved ("build it
> as designed"). Splits: [docs/splits/v1.5.0-status-bar.html](../splits/v1.5.0-status-bar.html).

## Problem

During the v1.4.9 hang the user couldn't tell whether the agent was alive,
stuck, or done — "I don't know if it's dead or running." The `_above_input`
heartbeat shows a winking dot + a bare phase label (`LLM` / `tool: X`), but no
elapsed time, no streamed-size, and — critically — no **progress age** (when did
a token last arrive). A long-but-live response and a dead connection look
identical.

## Goal

Extend the live region into a one-line status bar that shows, for the **active
actor**:
- **which actor** holds the turn — `LLM` / `Exec: <tool>` / `Advisor`,
- **elapsed** for the current phase (`0:47`),
- in-flight **token/size** count (LLM only),
- **progress age** — `last token Ns ago`, escalating to `⚠ Ns no tokens` as it
  nears `llm_timeout_s` (the anti-"is it dead" signal),
with the stall flag **suppressed during tool execution** (CC parity) and the
v0.9.8 presence-blink kept (no brightness fade — 256-color ceiling).

## Non-goals

- Three always-on lamps (one active-actor line instead — split #1).
- Per-second token rate, cost/$ (separate cost release).
- Truecolor glimmer / smooth red fade (256-color bands — split #7).
- Scrollback entry (live region only; no visibility-parity obligation — split #8).

## Claude Code reference

See [docs/splits/v1.5.0-status-bar.html](../splits/v1.5.0-status-bar.html).
CC: `components/Spinner/SpinnerAnimationRow.tsx` (one line: glyph + verb +
`· elapsed · N tokens`), `useStalledAnimation.ts`
(`isStalled = timeSinceLastToken > 3000 && !hasActiveTools` — stall OFF during
tools), mutually-exclusive modes.

- **Active-actor line, not 3 lamps** — Follow CC (one line). The CM is single-actor.
- **Progress-age as text** — No CC analog (CC red-shifts a glyph); explicit
  `last token Ns ago` text, derived from v1.4.9 `last_progress_at`.
- **Per-phase elapsed** — from a new `phase_started_at`.
- **Token count** — Follow CC; `approximate_token_count(store.pending_assistant_text)`.
- **Stall suppressed during tools** — Follow CC; the ⚠ only renders in AWAITING_LLM.
- **Advisor while IDLE** — No CC analog; a chat-side `_advisor_busy` flag.
- **Presence-blink kept** — Alternative; discrete white→red swap, no fade.

## Design

### `src/neutrix/context_manager.py`
- Add `phase_started_at: float | None` (init False). Set `= time.monotonic()`
  at each busy-state entry in `_drive` (AWAITING_LLM next to `last_progress_at`;
  AWAITING_EXECUTOR before `_dispatch_tools`); cleared to `None` in the `finally`
  (alongside the IDLE reset). `last_progress_at` already bumps per token (v1.4.9).

### `src/neutrix/terminal_chat.py`
- `format_heartbeat(...)` gains `phase_started_at` + `now` (injectable for tests).
  Build the line as `glyph + actor + " · " + elapsed [+ " · " + tokens] [+ " · " +
  age]`. AWAITING_LLM → actor `LLM`, tokens from pending text, age from
  `last_progress_at` (show `last token Ns ago` past a 3s floor, `⚠ Ns no tokens`
  past `stall_threshold_s`, glyph red when stalled). AWAITING_EXECUTOR → actor
  `Exec: <tool>`, elapsed only, **no age/stall**. `format_duration_short(s)` →
  `M:SS` / `Ns`.
- `_above_input`: if CM IDLE but `_advisor_busy`, render `Advisor · <elapsed>`
  (from `_advisor_started_at`). Set `_advisor_busy`/`_advisor_started_at` around
  `_maybe_run_advisor()` in `_worker_loop`.
- Pass `phase_started_at=self.ctx.phase_started_at` from `_above_input`.

### `src/neutrix/cli.py` (diagnosability)
- At startup, `logger.info("neutrix {} starting — src={} (loaded {})", …)` with the
  package path + `context_manager.py` mtime — so a running process's actual code
  + freshness is greppable (the setuptools_scm `__version__` is install-frozen and
  can lag the live editable source; the recurring stale-process confusion).

## Acceptance

- **Automated**: `format_heartbeat` shows actor + `M:SS` elapsed; LLM phase shows
  token count + `last token Ns ago`; past `stall_threshold_s` shows `⚠ … no tokens`
  + red glyph; **Exec phase shows NO stall/age even when progress age is large**
  (suppression); IDLE returns empty unless `_advisor_busy`. Existing
  `test_heartbeat_format` / `test_heartbeat_ticker` updated. Full suite green; ruff clean.
- **Manual**: a slow LLM reply shows a rising token count + `last token Ns ago`
  ticking; a stalled call shows `⚠ … no tokens` + red dot; a long Bash shows
  `Exec: Bash 0:12` with no false stall; the turn-end advisor shows `◇ Advisor`.
- Ship `v1.5.0`.

## Out of scope

- Cost/$ tracking, token rate, 3-lamp layout, truecolor animation.
