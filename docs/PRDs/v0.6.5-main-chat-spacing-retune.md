# v0.6.5 - main chat spacing retune

## Problem

The v0.6.4 chat polish still looks wrong in the rendered terminal:

- the system prompt starts at the left screen edge, making it feel clipped;
- the one-line draft editor is too tall because the `TextArea` has a fixed
  three-row content height inside the user block;
- the draft therefore has too much vertical text-to-edge space even though
  horizontal block padding was removed.

## Goal

Retune spacing based on the rendered TUI, not just CSS inspection:

- give the unframed system prompt a small left/right inset;
- keep message blocks compact with the existing tiny inter-block gap;
- make the draft editor height follow the number of draft lines, so a
  one-line draft has one content row and multi-line drafts expand up to a
  small cap.

## Non-goals

- Change chat behavior, provider calls, saved-session format, or onboarding.
- Redesign colors, role labels, or block borders.
- Add prompt history or a larger editor feature set.

## Design

1. Keep the system prompt unframed, but add a one-column horizontal padding
   on the system role class.
2. Change `DraftInput` from a fixed-height `TextArea` to a compact auto-sized
   editor:
   - minimum content height: one row;
   - maximum content height: six rows;
   - height recalculated after text loads and after TextArea change events.
3. Add regression coverage for system prompt inset and draft height behavior.

## Acceptance

- The system prompt no longer starts in terminal column 0.
- A one-line draft renders with one content row between the user block border
  lines.
- A multi-line draft grows beyond one row without making the composer
  unnecessarily tall.
- Existing Enter-submit and newline shortcuts still work.
