# v0.6.0 - main chat polish

## Problem

The main chat screen feels like a raw terminal form instead of a chat
application:

- the input is docked to the bottom edge and spans the full screen;
- the first screen has no clear prompt/composer focus;
- the active system prompt is not visible;
- while the model is responding there is no obvious activity state;
- the default system prompt is still tuned for a coding assistant,
  not the simple chatbot behavior expected from the main app.

## Goal

Make the main app feel closer to a simple Codex/Claude-style chat:

- show a centered floating composer before the first user message;
- after the first message, place the composer directly under the
  message list instead of docking it to the bottom edge;
- reduce the input's visual footprint and keep it inside a compact
  block;
- display the active system prompt on screen;
- show an animated responding indicator and prevent accidental extra
  submissions while the model is streaming;
- keep basic chatbot behavior: user submits a message, the language
  model streams a response.

## Non-goals

- Rebuild the app as a browser UI.
- Add multi-turn editing, branches, attachments, avatars, or prompt
  management.
- Change provider configuration, onboarding, tools, or session formats.
- Add complex theme settings.

## Design

Use the existing Textual app and keep the implementation small:

1. Change the default `Agent` system prompt to:
   `You are a helpful assistant. Keep it simple.`
2. Replace the docked input/status layout with a `#chat` container:
   - initial state: the composer is centered;
   - active state: a scrollable message list gets `1fr` height and the
     composer follows it in normal layout.
3. Put the system prompt, responding indicator, input, and status line
   inside the composer block.
4. Style messages as compact role-labelled blocks with different
   surfaces for user, assistant, system, tool, and error messages.
5. While streaming:
   - mark the app busy;
   - disable the input;
   - animate a small "assistant is responding" line;
   - restore the input and status after the response or error.

## Acceptance

- Launching the main app shows the system prompt and a centered compact
  composer before any user message is sent.
- The input is not docked to the screen bottom.
- Submitting a user message mounts the log, posts the user message, and
  keeps the composer immediately under the message list.
- Assistant tokens stream into an assistant message.
- While a response is in progress, the UI displays an animated
  responding indicator and the input cannot submit another message.
- The default system prompt is exactly:
  `You are a helpful assistant. Keep it simple.`
- Existing onboarding behavior remains unchanged.
