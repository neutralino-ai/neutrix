> **Status: finalized.** Fifth release on the march to v1.0, the big rock —
> a new runtime actor. Done autonomously under a delegating `/goal`. Split
> decisions in [docs/splits/v0.10.4-smart-advisor.html](../splits/v0.10.4-smart-advisor.html).
>
> **Version-drift reconciliation:** the skeleton calls itself "v0.11.0",
> tuiview-purify "v0.10.1", and compaction "v0.11.1". Translated to current
> numbers: this is **v0.10.4**; it depends on the shipped **v0.10.3** (store
> purity) + **v0.10.2** (reminder rendering); compaction is **v0.10.5**.

## Problem

The only "advice" the LLM gets between turns is mechanical: the v0.8.1 in-band
tool-result nudge (same string regardless of state) and the v0.8.0 10-turn
`<system-reminder>` (fires on a counter, body templated with no judgment).
Neither can notice a task is effectively done, spot a stale/drifted task, or
add a task the user just implied. The user asked for the upgrade in v0.8.1
review: *"the advisor will be upgraded to be smart => not just mechanically
record task update etc. but revise."*

## Goal

A third actor, `Advisor`, alongside the controller:
- New `neutrix.advisor` module with `Advisor` + `AdvisorOutcome`.
- `Advisor.run_once(*, tasks, recent_turns, llm, model) -> AdvisorOutcome` —
  **side-effect-free**: builds an advisor-role prompt (advisor system prompt +
  task list + last K=6 turn pairs + Task-tool schemas), makes one LLM call,
  returns parsed `(task_calls, suggestion)`.
- Trigger: **turn-end hook** in `_worker_loop` — periodic (every N=5 completed
  turns) + explicit `/advise`; run-lock against re-entry; session cap (20).
- Two output channels, applied by the orchestrator (not by `run_once`):
  1. **Task mutations** via `tools.dispatch(name, args, store=…)` — rendered as
     a folded `↳ advisor: …` audit notice.
  2. **A judged suggestion** injected via a new
     `ContextManager.inject_advisor_message(text)` as a `<advisor>…</advisor>`
     pseudo-user turn — rendered expanded as `↳ advisor: <body>`; the next
     controller turn sees it in `messages`.

## Non-goals

- Replacing the v0.8.1 nudge or v0.8.0 reminder (the Advisor augments them).
- A general subagent framework (that's v0.10.0, shipped).
- Cross-session learning; writing to anything but the store (Task tools +
  message injection only).
- Streaming the Advisor's call (small, background, final-only — heartbeat covers
  liveness).
- **Idle-timer trigger** — deferred for scope (needs a background timer
  coordinating with the run-lock + IDLE requirement); periodic + `/advise` are
  the deterministic, testable core. Not infeasible.

## Claude Code reference

See [docs/splits/v0.10.4-smart-advisor.html](../splits/v0.10.4-smart-advisor.html)
for the full survey (CC's turn-end forked actors) and 7 decisions. Key ones:

### Split point: trigger + awaited
- **CC**: turn-end stop hook, fire-and-forget, cheap gates.
- **Decision**: Follow CC for the turn-end hook; **Alternative — awaited** (race-free in neutrix's single async loop; the decoupled input loop keeps the user typing), completed-turns-only, failure-isolated.

### Split point: injection path
- **Decision**: route through CM. `run_once` is side-effect-free; the orchestrator applies task mutations via `dispatch(store=…)` and injects the suggestion via `ContextManager.inject_advisor_message` (writes messages + store in lockstep) — preserving the single-mutator invariant (v0.10.3).

### Split point: marker handling
- **Decision**: `is_advisor_message` → excluded from Up-arrow recall (`_is_real_user_prompt`) + a distinct `_render_record` branch (`↳ advisor:`).

### Split point: LLM / slot
- **Decision**: a **fresh** `OpenAIChatLLM` on the `fast` slot (never switch the shared client); injected for tests.

## Design

### `src/neutrix/advisor.py`
- `ADVISOR_SYSTEM_PROMPT` (review-and-suggest; guardrails: don't execute, don't
  touch yourself/your scheduling, only Task tools), `ADVISOR_CADENCE_TURNS=5`,
  `ADVISOR_MAX_RUNS=20`, `ADVISOR_RECENT_TURNS=6`.
- `@dataclass(frozen=True) AdvisorOutcome(task_calls: tuple[tuple[str,str],...], suggestion: str | None)`.
- `Advisor`: `should_run()` (periodic counter + cap + run-lock), `note_turn()`,
  `run_once(...)` (LLM call → parse tool_calls + text). Reuses
  `get_schemas({"TaskCreate","TaskUpdate","TaskList"})` for the advisor's tools.

### `src/neutrix/context_manager.py`
- `ADVISOR_TAG_OPEN/CLOSE` + `is_advisor_message(content)` (module-level,
  alongside `is_task_reminder`).
- `inject_advisor_message(text)`: appends `{"role":"user","content":
  "<advisor>…</advisor>"}` to `messages` + store record (mirrors
  `_append_user_message`).

### `src/neutrix/terminal_chat.py`
- `_is_real_user_prompt`: also exclude `is_advisor_message`.
- `_render_record`: `<advisor>` branch → expanded `↳ advisor: <body>` notice.
- `TerminalChat`: own an `Advisor` + a fresh advisor `OpenAIChatLLM` (fast slot
  if configured, else active). `_maybe_run_advisor(forced=False)` called at
  turn-end in `_worker_loop` (only when the last message is an assistant turn);
  applies the outcome (dispatch task calls + audit notice; inject suggestion).
- `/advise` command → `_maybe_run_advisor(forced=True)`.

### Tests (`tests/test_advisor.py`)
- Trigger policy: periodic counter fires at N, run-lock drops re-entry, cap stops
  auto-trigger.
- `run_once` with a mock LLM: tool_calls → `task_calls`; text → `suggestion`;
  both.
- Output handling: task calls dispatched mutate the store; suggestion routed via
  `inject_advisor_message` (messages + store, wrapped in `<advisor>`).
- Recall exclusion + render: `is_advisor_message` true for injected turns; an
  `<advisor>` record renders as `↳ advisor:` not a raw `>` user turn.

## Acceptance

- Automated: `tests/test_advisor.py` + the marker/render tests pass; full suite
  green; ruff clean. The v0.8.0 reminder still renders folded (no regression).
- Ship in numeric order (tag after v0.10.3).

## Out of scope

- Idle trigger, multi-advisor, user-configurable advisor prompt, advisor
  streaming UI (all later).
