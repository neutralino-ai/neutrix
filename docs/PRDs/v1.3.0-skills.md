## Problem

neutrix's slash commands are all hardcoded `_cmd_*` methods — there's no way to
add a reusable prompt/automation without editing the source, and no
interop with Claude Code's `.claude/skills` / `.claude/commands` / `.claude/agents`.
A CC replacement must discover and run those.

## Goal

1. **Markdown skills/commands** from `.claude/skills/<name>/SKILL.md` and
   `.claude/commands/*.md` (user `~/.claude` then project; project overrides),
   with frontmatter (`name`/`description`/`argument-hint`). `/<name>` enqueues
   the body as a user turn, substituting `$ARGUMENTS` and `$1`…`$9`.
2. **Hot-reload** — re-discover on add/edit/remove without restart.
3. **Custom subagent types** from `.claude/agents/*.md` — `subagent_type` values
   the `Agent` tool can spawn (body = system prompt), extending v0.10.0.

`.claude/`-compatible throughout.

## Non-goals

- A model-invocable `Skill` tool (deferred; slash form is the core).
- `paths:`-frontmatter conditional skills (needs the edit-event surface — v1.6.0 hooks).
- The full `/init` skill/hook generation wizard.

## Design

### `src/neutrix/skills.py` (new — pure)
- `SkillDef`, `AgentDef`; `_parse_frontmatter`.
- `discover_skills(cwd, home)` (skills dirs + command files, user→project).
- `discover_agents(cwd, home)`; `render_skill(skill, args)`;
  `skills_signature(cwd, home)` (mtime fingerprint for hot-reload).

### `src/neutrix/terminal_chat.py`
- Discover skills at init (`_skills`, `_skills_sig`); `_run_command` falls
  through to the skill registry (`/<name>` → enqueue `render_skill`); a
  `_skills_poller` background task re-discovers on signature change.

### `src/neutrix/tools.py`
- `_agent`: `subagent_type != "general-purpose"` resolves against
  `discover_agents(cwd)`; pass the agent's system prompt to `run_subagent`;
  unknown → error listing available types.

## Acceptance

- Automated (`tests/test_skills.py` + `test_terminal_chat.py`): discover
  skill-dir + command-file; project-overrides-user; frontmatter name override;
  `$ARGUMENTS`/`$1` substitution; signature changes on edit; discover agents;
  `Agent` uses a custom agent's system prompt + errors on unknown type; `/<name>`
  dispatch enqueues the rendered body; unknown command still errors. Full suite
  green (331); ruff clean.
- Ship `v1.3.0`.

## Out of scope

- Model-invocable Skill tool, conditional skills, /init wizard (later).
