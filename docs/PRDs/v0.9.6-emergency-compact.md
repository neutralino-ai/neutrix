# v0.9.6 — emergency mechanical /compact

> **Status: skeleton.** Slotted between v0.9.4 (heartbeat) and
> v0.9.6 (streaming-with-steer) to give users immediate manual
> relief from context-explode. Mechanical only — no
> summarization, no automatic threshold. The smart version
> (with LLM-summarization + threshold trigger + slot
> configuration) ships as v0.11.1.
> Phase 1 splits HTML at `docs/splits/v0.9.5-emergency-compact.html`
> when implementation begins.

## Problem

User-reported during live use: long agentic sessions hit the
provider's per-request token limit and the session dies. The
only existing escape hatch is `/clear`, which wipes the entire
conversation — too destructive for "I just want to trim the
oldest half."

Smart compaction (LLM-summarized, automatic threshold) is
v0.11.1 in the agentic-depth arc — 5 releases away. That's too
long to wait for any user actively hitting the cliff.

## Goal

Ship a `/compact` slash command that:

1. Identifies the oldest 50 % of `ContextManager.messages` by
   index (preserving `messages[0]` — the system prompt — and the
   tool-round-boundary alignment from v0.9.3 pairing rules).
2. Replaces the cut slice with a single placeholder user message:
   `<system-compact>{N} earlier turns removed by /compact (no
   summary)</system-compact>`.
3. Surfaces a dim transcript notice: `compacted N turns (~K
   tokens dropped)`.
4. Saves / loads the compacted state unchanged through
   `transcript.save` / `/load`.

No LLM call. No summarization. Mechanical only.

## Non-goals

- **Smart summarization.** Ships in v0.11.1.
- **Automatic threshold trigger.** Ships in v0.11.1.
- **Configurable cut ratio.** Hardcoded at 50 %; users who want
  a different ratio wait for v0.11.1.
- **Recovery / undo.** Compaction is destructive to the live
  message list. Users who want recovery use `/save` before
  `/compact`. Rewind from a compacted point is a separate
  question for v0.10.2.

## Design (sketch)

- New `compact_messages(messages, system_index=0, keep_ratio=0.5)
  -> tuple[list[dict], int, int]` pure function (new module
  `src/neutrix/compaction.py`). Returns
  `(new_messages, turns_dropped, approx_tokens_dropped)`.
- Tool-round-boundary awareness: when computing the cut index,
  walk forward past any in-progress tool round so the cut
  boundary always falls between rounds, not mid-`tool_use` /
  `tool_result` pair (mirrors v0.9.3 pairing rules).
- `/compact` handler in `terminal_chat.py` calls
  `compact_messages` against `cm.messages`, replaces the slice,
  emits a `ChatStore` mutation event for the renderer.
- Visibility parity (v0.10.0) hasn't shipped yet — the
  `<system-compact>` block renders as a plain user turn for now;
  v0.10.0's labelling work will pick it up.

## Acceptance

- Manual:
  - Run a session past ~30 turns. `/compact`. Transcript shows
    `compacted 15 turns (~3.2k tokens dropped)`. Subsequent LLM
    rounds succeed; payload is half-size.
  - Cancel + steer still works post-compaction (v0.9.3 pairing
    layer handles the synthetic compact message correctly).
  - `/save` + reload preserves the compacted state.
- Automated:
  - `tests/test_compaction.py::test_mechanical_compact_drops_oldest_half`
    — fixture with 20 turns; assert 10 dropped, system prompt
    intact, latest 10 intact.
  - `test_mechanical_compact_respects_tool_round_boundary` —
    fixture where the naive cut would split a tool round; assert
    the actual cut falls between rounds.
  - `test_mechanical_compact_idempotent` — calling /compact
    twice in a row is safe (second call may be a no-op or further
    halve; either is acceptable, just no crash).
- Human gate before tag.

## Out of scope

- Smart compaction (v0.11.1) — automatic threshold + LLM
  summarization. v0.9.5 is the mechanical bridge that lets
  users keep working until v0.11.1 lands.
