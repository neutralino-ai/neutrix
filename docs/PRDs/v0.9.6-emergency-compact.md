# v0.9.6 — emergency mechanical /compact

> **Status: accepted (autonomous draft).** A no-LLM `/compact` slash
> command that drops the oldest ~50 % of `ContextManager.messages`
> mechanically, so a user hitting the provider's per-request token
> limit has an escape hatch that isn't the all-or-nothing `/clear`.
> The *smart* version — automatic threshold trigger + LLM-summarized
> cut segment — is **v0.10.5**, five releases out. This is the bridge.
>
> Drafted under a `/goal` (autonomous mode): the Phase-1 split-point
> discussion could not happen turn-by-turn, so all 12 decisions in the
> Claude Code reference section were made by the agent acting as
> architect and are open to redirection at the Phase-3 accept gate.
> Splits survey: [`docs/splits/v0.9.6-emergency-compact.html`](../splits/v0.9.6-emergency-compact.html).

## Problem

User-reported during live use: long agentic sessions accumulate
context until the provider's per-request token limit is hit and the
session stalls. The only existing escape hatch is `/clear`, which
wipes the *entire* conversation — far too destructive for "I just
want to trim the oldest half and keep going."

Smart compaction (LLM-summarized cut, automatic threshold) is
**v0.10.5** in the P1 tier — too far out to help a user actively
hitting the cliff today. v0.9.6 ships the mechanical bridge now.

## Goal

Ship a `/compact` slash command that:

1. Preserves the leading `system` message(s), then drops the oldest
   ~50 % of the remaining (non-system) messages **by index**, snapping
   the cut forward to a tool-round boundary so the kept tail never
   begins on an orphan `tool_result`.
2. Inserts a single `role:user` placeholder in place of the cut slice:
   `<system-compact>{N} earlier turns removed by /compact (no
   summary)</system-compact>`.
3. Compacts **both** `cm.messages` (the LLM payload) and the typed
   `ChatStore` (the render/save source), while **preserving tasks**.
4. Prints one dim transcript notice — `compacted N turns (~K tokens
   dropped)` — and suppresses re-printing the already-visible tail.
5. Round-trips through `/save` + `/load` unchanged (the placeholder
   rides as an ordinary user message; no transcript schema change).

No LLM call. No summarization. No automatic trigger. Mechanical only.

## Non-goals

- **Smart summarization.** The LLM-summarized cut segment ships in
  **v0.10.5**.
- **Automatic threshold trigger.** v0.10.5 adds the token-budget
  watchdog that fires `cm.compact()` on its own.
- **Configurable cut ratio.** Hardcoded at 50 %; `/compact` takes no
  arguments. Tunable ratio is v0.10.5 territory.
- **Recovery / undo.** Compaction is destructive to the live message
  list; a user who wants a fallback uses `/save` before `/compact`.
  Rewind from a compacted point is the separate **v0.9.7** rewind work.
- **Special marker rendering.** The `<system-compact>` placeholder
  renders as a plain user turn for v0.9.6; dim/labelled rendering is
  picked up by **v0.10.2** visibility-parity.
- **Task-reminder cadence is intentionally reset by compaction.**
  Dropping the older turns also drops any past `<system-reminder>`
  injections, so the next LLM round may see "no reminder for N turns"
  and re-inject one. This is correct behavior (the reminder counter is
  derived from live history), not a bug — noted here so a future reader
  doesn't try to "fix" it.

## Claude Code reference

See [`docs/splits/v0.9.6-emergency-compact.html`](../splits/v0.9.6-emergency-compact.html)
for the full survey with alternatives and recommendations. CC source
at `/datafs/users/dingxf/anthropic/cc2/src`.

**Headline finding:** CC has *no* proactive "drop the oldest N
messages" path. Its compaction is summary-based
(`services/compact/compact.ts:387-763`); the only *mechanical* moves
are micro-compact, which *clears tool-result content in place* rather
than removing messages (`microCompact.ts:36,470-492`), and the
reactive prompt-too-long escape hatch `truncateHeadForPTLRetry`, which
drops whole oldest API-round groups (`compact.ts:231-241,257`). v0.9.6
borrows exactly one thing — the **API-round boundary** as the safe cut
granularity — and is otherwise a deliberately small neutrix-shaped
stopgap. All decisions below are **autonomous** (architect-made under a
`/goal`).

### Split point: compression method (drop vs clear vs summarize)
- **CC**: summarize (`compact.ts:387-763`); mechanical moves only
  clear tool-result content (`microCompact.ts:36`) or reactively drop
  round groups (`compact.ts:231-241`).
- **Decision**: Alternative — drop the oldest ~50 % of messages. No-LLM,
  one-release stopgap; content-clear and summarize are both deferred
  (summarize → v0.10.5).

### Split point: cut granularity & direction
- **CC**: cut at API-round boundaries (`grouping.ts:22-63`), drop whole
  oldest groups; round boundary is API-safe because every `tool_use` is
  resolved before the next assistant turn (`grouping.ts:34-42`).
- **Decision**: Follow CC — round-safe boundary, **snap forward**. Naive
  index `1 + floor(L/2)` over the non-system body, then advance past any
  leading `role:tool` so the first kept message is never an orphan
  tool_result.

### Split point: placeholder
- **CC**: typed `CompactBoundaryMessage` with metadata
  (`compact.ts:598-611`).
- **Decision**: Alternative — single `role:user` placeholder,
  `<system-compact>{N} earlier turns removed by /compact (no
  summary)</system-compact>` (text verbatim). `role:user` (not a 2nd
  system message) mirrors the `<system-reminder>` / `[interrupted by
  user]` convention; no schema change. CC's typed boundary is a
  No-CC-analog gap (neutrix has no message-metadata infra).

### Split point: head preservation
- **CC**: regenerates the system prompt fresh post-compact
  (`compact.ts:269-287`).
- **Decision**: Alternative — preserve the leading `system` prefix
  verbatim, cut only the body after it. neutrix's system prompt is
  static and anchored at `messages[0]`; regeneration would diverge from
  a loaded session's stored prompt.

### Split point: report (turns vs tokens; token source)
- **CC**: reports tokens (pre/post, real tokenizer)
  (`compact.ts:637-642`; `commands/compact/compact.ts:230-248`).
- **Decision**: Alternative — `compacted N turns (~K tokens dropped)`.
  "turns" kept verbatim from the skeleton (no unilateral wording edit in
  autonomous mode); token figure reuses the existing whitespace
  `approximate_token_count` rather than adding a tokenizer (the `~`
  signals it's approximate).

### Split point: command arguments
- **CC**: optional custom summary instructions
  (`commands/compact/compact.ts:52`).
- **Decision**: Alternative — no arguments. No summary to steer; ratio
  is deliberately fixed (configurable ratio is a v0.10.5 non-goal).

### Split point: idempotency
- **CC**: no guarantee; re-run summarizes the prior summary
  (`compact.ts:791-800`).
- **Decision**: Alternative — monotonic further-halve with a no-op
  floor. Second call drops the old placeholder + more body and inserts
  one fresh placeholder; when `floor(L/2) == 0` it's a clean no-op
  ("nothing to compact"). Never stacks markers, never crashes.

### Split point: busy behavior
- **CC**: compaction runs inside the query loop; no "user typed
  /compact mid-turn" analog.
- **Decision**: Alternative — refuse while busy. Add `compact` to the
  existing `{fast, strong, save, load, onboard}` guard set
  (`terminal_chat.py:977`). The CM routine therefore only runs against
  settled history.

### Split point: live rendering
- **CC**: full-screen Ink app re-renders the whole view.
- **Decision**: Alternative — notice-only + suppress reprint. After
  compaction set `_rendered_message_count = len(store.messages)` (both
  prevents tail duplication and re-aligns the index for future appends);
  print one dim notice. Diverges from `/load` (which reprints *unseen*
  content) on purpose — compaction's kept tail is already in scrollback.

### Split point: store + payload, and tasks
- **CC**: single message model; no payload/store split. No analog.
- **Decision**: No CC analog — compact both `cm.messages` and the store
  so `/save` persists the compacted state; **preserve tasks** (save →
  `store.reset()` → append compacted records → `replace_tasks`).
  Compaction trims context, not the live work list.

### Split point: save/load & marker rendering
- **CC**: JSONL-on-disk; typed boundary record. No format analog.
- **Decision**: No CC analog — transcript v2 unchanged; the
  `<system-compact>` placeholder round-trips as a `role:user` message.
  Dim/special rendering deferred to v0.10.2 visibility-parity (renders
  raw for v0.9.6).

### Split point: programmatic entry point
- **CC**: compaction is a service; trigger (auto-compact) and mechanism
  are separable (`autoCompact.ts`).
- **Decision**: No CC analog — direct `async cm.compact() ->
  CompactionOutcome` (mirrors the value-returning `cancel()` method;
  the event surface stays for fire-and-forget UI events). Boundary +
  placeholder logic lives in a pure `compaction.compact_messages()` so
  v0.10.5's auto-trigger reuses the cut computation and swaps the
  placeholder step for a summarizer. No `CompactEvent` until a
  fire-and-forget caller needs one.

## Design

### New module `src/neutrix/compaction.py`

```python
@dataclass(frozen=True)
class CompactionOutcome:
    did_compact: bool
    turns_dropped: int
    approx_tokens_dropped: int

COMPACT_MARKER_OPEN = "<system-compact>"
COMPACT_MARKER_CLOSE = "</system-compact>"

def compact_messages(
    messages: list[dict],
    *,
    keep_ratio: float = 0.5,
) -> tuple[list[dict], CompactionOutcome]:
    """Pure function. Preserve the leading system prefix, drop the
    oldest (1 - keep_ratio) of the body snapped forward to a tool-round
    boundary, insert one <system-compact> placeholder. Returns the new
    list and the outcome (did_compact False ⇒ list returned unchanged)."""
```

- **System prefix**: `k = count of leading role=="system" messages`
  (normally 1). Body = `messages[k:]`.
- **Naive cut** within the body: `drop = floor(len(body) * (1 -
  keep_ratio))`. If `drop == 0`, return unchanged with
  `did_compact=False`.
- **Forward snap**: while the body element at the cut is a `role:tool`
  message, advance the cut (drop one more). Guard: if the snap consumes
  the entire body, return unchanged (`did_compact=False`).
- **Rebuild**: `messages[:k] + [placeholder] + body[cut:]` where the
  placeholder is `{"role":"user", "content": f"{OPEN}{n} earlier turns
  removed by /compact (no summary){CLOSE}"}` and `n` = number of dropped
  body messages.
- **Token estimate**: sum of a local whitespace word-count over the
  dropped messages' `content` (same heuristic as
  `terminal_chat.approximate_token_count`; `~` prefix signals
  approximate).
- `is_compact_marker(content)` helper (mirrors `is_task_reminder`) for
  v0.10.2 to pick up later; unused by the renderer in v0.9.6.

### `ContextManager.compact()` (`context_manager.py`)

```python
async def compact(self) -> CompactionOutcome:
    await self._cancel_and_wait()        # defensive; command refuses while busy
    new_messages, outcome = compact_messages(self.messages)
    if not outcome.did_compact:
        return outcome
    tasks = self.store.tasks
    self.messages = new_messages
    self.store.reset()
    for msg in new_messages:
        self.store.append_message(_record_from_openai(msg))
    self.store.replace_tasks(tasks)
    return outcome
```

Direct method (not an event) so the command gets the counts back —
same pattern as the existing `cancel()`. `_cancel_and_wait()` is a
no-op when IDLE (the command already refuses while busy) but keeps the
method safe for a future event-driven caller.

### `/compact` handler (`terminal_chat.py`)

- Add `compact` to the busy-guard set at `terminal_chat.py:977`.
- Add `_cmd_compact`:
  ```python
  async def _cmd_compact(self, args):
      outcome = await self.ctx.compact()
      if not outcome.did_compact:
          await self.view.print_notice(
              "nothing to compact (conversation too short)", style="dim")
          return
      self._rendered_message_count = len(self.store.messages)
      await self.view.print_notice(
          f"compacted {outcome.turns_dropped} turns "
          f"(~{_fmt_tokens(outcome.approx_tokens_dropped)} tokens dropped)",
          style="dim")
  ```
- Add the `/compact` line to `_cmd_help`.
- `_fmt_tokens(n)`: `f"{n/1000:.1f}k"` when `n >= 1000` else `str(n)`.

### Forward-compatibility with v0.10.5

The cut computation (system-prefix + forward-snap boundary) is isolated
inside `compact_messages`; v0.10.5 reuses it and replaces the
placeholder-insertion step with a summarizer call against the cut slice,
and adds a token-threshold watchdog that calls `cm.compact()` (the same
method this PRD introduces) instead of the `/compact` command.

## Acceptance

### Manual
- Run a session past ~30 messages. `/compact`. Transcript shows
  `compacted N turns (~K tokens dropped)`; the kept tail is **not**
  reprinted (already in scrollback). Subsequent LLM rounds succeed and
  the payload is ~half size (`/status` `msgs:` count drops).
- `/compact` while the assistant is working prints
  "/compact waits for the assistant to finish" and does nothing.
- Cancel + steer still works after a `/compact` (the pairing layer
  handles the synthetic compact message and any orphan tool_use).
- `/save` after `/compact`, then `/load` the file: the compacted state
  (placeholder + kept tail + tasks) is restored intact.
- `/compact` on a 2-message conversation prints "nothing to compact".

### Automated (`tests/test_compaction.py`)
- `test_drops_oldest_half`: 20-message body fixture (+system); assert
  ~10 dropped, system prompt intact at `[0]`, newest messages intact,
  one `<system-compact>` placeholder at `[1]`,
  `outcome.turns_dropped == 10`.
- `test_respects_tool_round_boundary`: fixture where the naive cut lands
  on a `role:tool`; assert the actual cut snapped forward so the first
  kept message is **not** `role:tool` (no leading orphan tool_result).
- `test_pairing_survives_compaction`: fixture where the kept tail's
  **last** message is an assistant-with-`tool_calls` and no
  `[interrupted by user]` follows; run `_ensure_tool_result_pairing` on
  the compacted list and assert `MISSING_TOOL_RESULT` synthesis fires
  and the payload is API-valid (every `tool_use` has a matching
  `tool_result`, no leading orphan `tool_result`).
- `test_kept_tail_tooluse_keeps_results`: fixture where the kept tail
  *starts* with an assistant-with-`tool_calls`; assert its matching
  `role:tool` results are inside the kept region.
- `test_idempotent`: `/compact` twice in a row; assert no crash, the
  second call further-halves or no-ops, and there is never more than one
  `<system-compact>` placeholder.
- `test_no_op_when_too_short`: 1-message body; assert
  `did_compact=False` and the list is returned unchanged.
- `test_compact_preserves_tasks` (CM-level): seed tasks, run
  `cm.compact()`, assert `store.tasks` is unchanged and
  `store.messages` is the compacted set; assert a subsequent
  `store.add_task` gets a non-colliding id.
- `test_token_estimate_monotonic`: dropping more messages never reports
  fewer approx tokens.
