neutrix roadmap

A terminal agentic chat client for OpenAI-compatible LLM APIs.

This page is the user-visible map of where the project is going. Per CLAUDE.md, every PRD acceptance updates this page before implementation starts. Entries link to their PRDs under docs/PRDs/.

v1.0 destination: a TUI Claude-Code-shaped agentic client — cancel-as-steer, lifecycle-driven store, visibility parity, smart Advisor, conversation compaction, subagent dispatch, rewind / branch. Sealed surface at v1.0. Markdown rendering, cost tracking, web frontend, MCP, skills, hooks are post-v1.0 (v1.x).

Render order is by priority tier, then by logical sequence within each tier: P0 blocks daily use right now; P1 important for the v1.0 lock; P2 nice to have pre-v1.0; P3 deferred to post-v1.0 (v1.x).

P0 — daily-blocking / user-reported urgent

v0.9.5 LLM error surface (stalled / timeout / inline error) User-reported during v0.9.4 dogfooding: when the LLM API hangs silently, the heartbeat keeps breathing but neither neutrix nor the LLM is making progress, and the user has no signal until the SDK's ~10-minute default timeout fires. Two layers: ~50 s of silence → heartbeat palette swap to red + ● LLM (stalled) label (UI hint, no abort); ~300 s → background watchdog fires cm.cancel(reason='timeout'), transcript records [LLM timeout after 300s]. Per-slot tunable via a new llm_timeout_s YAML field (default 300 s); the stall hint is derived as max(10, timeout/6). Forward-compatible with v0.10.1 streaming (12 split-point decisions; 9 Alternative / 3 Follow CC; thresholds revised at acceptance for slow models). shipped
v0.9.6 Emergency mechanical /compact User-reported urgent. A slash command that drops the oldest 50 % of ContextManager.messages mechanically — no LLM summarization, no automatic trigger, no configurable ratio. Tool-round-boundary aware. The bridge that keeps users working until smart compaction lands at v0.10.5. (12 split-point decisions; 1 Follow CC / 8 Alternative / 3 No CC analog.) shipped
v0.9.7 Rewind / branch from prior turn User-reported urgent. /rewind N destructively drops the last N user turns (Follow CC) — tool-round-boundary safe — and prints a forward ↶ rewound notice (the append-only renderer can't restyle past turns, so the skeleton's retroactive [rewound] marker became a forward notice — split #7 scope change). Up/Down recall prior user turns into the draft, decoupled from rewind (recall only edits the buffer — split #2). Implemented + tested (suite 224); 4 split forks flagged for review. (10 split-point decisions.) shipped
v0.9.8 Liveness motion — blink, not brightness User-reported: the v0.9.4/v0.9.5 breathing dot still read as stuck at ~5 Hz on a tmux-256color terminal. Root cause — 256-color exposes only ~22 distinct grays (the xterm 24-step ramp #080808..#eeeeee), so a brightness fade bands regardless of refresh; the v0.9.5 jump to 120 Hz couldn't help because the palette, not the clock, is the ceiling. Fix: animate the dot's presence, not its brightness — a single ● that winks on/off every 600 ms, following CC's tool-use loader (ToolUseLoader + useBlink). Red when stalled; strict period; the turn always opens on a visible dot. Re-affirms v0.9.4 #2, replaces #13, reverses v0.9.5's 120 Hz. (6 split-point decisions, all converging on CC's tool-loader wink.) shipped
v0.10.0 Subagent framework (Agent tool) Context-explode structural fix. Agent(description, prompt) spawns a fresh-context worker that reuses ContextManager, runs to completion, and returns only its final text — the parent's context grows by one tool-result, not the worker's transcript. Worker tools omit Agent (no recursion); 100k result cap; 25-round runaway cap; Esc stops the burn via a cross-loop threading.Event (cancel reaches the parent through the v0.9.3 path). Renders as a normal tool result — the dedicated folded block moves to v0.10.2 (split #5). Implemented + tested (suite 248). (12 split-point decisions.) shipped

P1 — important for the v1.0 lock

v0.10.1 Re-enable streaming with cancel-as-steer stream_response back to stream=True (restored from v0.9.2, merged with v0.9.3's pairing layer): token deltas + index-keyed tool_calls + eager stop() stream close. On Esc the partial assistant text is kept — stashed during the round and committed before [interrupted by user] so a steer carries prior intent. Live token rendering deferred to v1.x (append-only scrollback; the heartbeat stays the liveness signal — split #3). Suite 253. (6 split-point decisions.) shipped
v0.10.2 Visibility parity — show what the LLM sees Ground-truthing showed only one channel was hidden: tool schemas → now a folded [tools] block at session start (/show tools expands). Long system prompts fold ([system]); subagent results get a [subagent] label. Adds .claude/rules/visibility-parity.md, an LLMRoundBundle snapshot, and an invariant test iterating the bundle's channels. Fold = expand-by-append (no in-place Ctrl+E toggle — append-only renderer, split #1). Suite 261. (5 split-point decisions.) shipped
v0.10.3 TUIView as pure renderer + FakeView swap-test The folded-tool-result tray moved off the view into ChatStore.folded_tool_results; ToolRecord is now pure data in store (rendering moved to view functions). A store-only FakeView + AST import-boundary test prove a view needs nothing but the store — de-risking v0.10.4's store-only Advisor. The skeleton's module-level import grep was infeasible (orchestrator lives in the module) → scoped to FakeView (split #1). Suite 270. (4 split-point decisions.) shipped
v0.10.4 Smart Advisor — third actor with LLM-driven task judgment A third actor (own cheap LLM call on the fast slot) consulted at turn-end (every 5 turns, or /advise): revises the task list via the Task tools, or injects a judged <advisor> suggestion the main LLM sees next turn. run_once is side-effect-free; injection routes through a ContextManager method (single-mutator preserved). Run-lock + session cap; skipped after a cancelled turn. Idle trigger deferred (split #1). Suite 283. (7 split-point decisions.) shipped
v0.10.5 Smart conversation compaction /compact + an automatic 75%-of-window threshold now summarize the older segment (one LLM call) into a folded <system-summary> (/show summary), keeping the recent ~50% verbatim, round-safe. Plus the >1M hardening from the planning memory: token-budget cut, reactive prompt-too-long catch+retry, and in-message tool-result truncation. Slot.max_context_tokens gates the auto-trigger. The v1.0 gate. Suite 296. (7 split-point decisions.) shipped

v1.0 lock

v1.0.0 Sealed v1.0 surface The cumulative result of the v0.9.x lifecycle arc + the v0.10.x band (subagents, streaming, visibility parity, renderer purity, Smart Advisor, smart compaction), tagged as the v1.0 product. SemVer frozen: post-1.0 behavior changes to the stable surface bump appropriately; internal refactors stay free. Polish + post-1.0 scope (markdown, cost tracking, skills, MCP, hooks, web frontend, parallel/recursive subagents) land as v1.x. Suite 297. (lock PRD.) shipped

v2 — toward complete Claude Code parity

A band of minor releases (numbers settle per release) building to a v2.0.0 CC-parity lock, mirroring how v0.10.x built to v1.0.0. Goal: a product that can completely replace Claude Code for coding. Lead with real coding tools; replace the crude tool set with CC-shaped ones; full parity incl. MCP (user-directed).
Core principle: .claude/-compatible. "If it replaces Claude Code it must be compatible with it." neutrix auto-discovers CC's config — CLAUDE.md, .claude/skills·commands·agents, .claude/settings.json (hooks + permissions) — and auto-writes CC-shaped files — per-session usage JSONL to ~/.claude/projects/… (so ccusage runs on neutrix), permission updates to .claude/settings.local.json. Drop neutrix into a CC project and it inherits the config. Caveat: co-exist safely — additive writes, never clobber CC's files.

v1.1.0 Coding tools (CC-shaped) Replace read_file/write_file/list_dir/run_shell with Read/Edit/Write/Grep/Glob/Bash: line-range Read, surgical str-replace Edit (read-before-edit enforced via the Executor), ripgrep-or-Python Grep, Glob, Bash (timeout + tree-kill). (7 split-point decisions.) shipped
v1.2.0 Project context — CLAUDE.md / AGENTS.md + @-mentions + /init CC's killer coding feature, currently absent: auto-load project + user memory files into the system prompt, @path file-mentions pull files into context, /init generates a CLAUDE.md by surveying the repo. .claude/-compatible — drops into a CC project and inherits its CLAUDE.md. (5 split-point decisions.) Suite 321. shipped
v1.3.0 Skills + slash-command framework + hot-reload Markdown skills/commands from .claude/skills + .claude/commands (frontmatter, a Skill invocation), watchdog hot-reload, and custom subagent types from .claude/agents (extends the v0.10.0 general-purpose-only set). User #4. P1planned
v1.4.0 Permissions + plan mode allow/ask/deny per tool (settings) + read-only plan mode — the safety layer that makes a real Edit/Bash safe and lets Bash hard-block the dedicated-tool commands (v1.1.0 deferred the block here). P1planned
v1.5.0 /goal + conversation resume A neutrix-native goal loop (set a goal, keep driving until a condition holds, /goal clear) — user #6 — plus --continue / /resume to reload the last session (gap-fill beyond /save·/load). P1planned
v1.6.0 Hooks A settings.json hook system (PreToolUse / PostToolUse / Stop / SessionStart) — block a tool, inject context, gate stop. /goal can be reframed on the Stop hook. P2planned
v1.7.0 MCP Load external mcp__server__tool tools from configured MCP servers, namespaced and permission-gated. P2planned
v1.8.0 Usage, cost & timing Capture the API usage (cache read / miss / output; cache-write where the gateway exposes it); per-message tokens + running cumulative; wall time, active time (wall − idle), and cost via LiteLLM's pricing JSON (which also auto-populates max_context_tokens); /cost + a live status-line readout; optional ccusage-compatible JSONL emission. P2planned
v1.9.0 Parallel tools + synchronized heartbeat Run a round's multiple tool_use blocks concurrently (asyncio.gather, ordered results) instead of sequentially; the heartbeat beats the in-flight parallel set in unison. Plus background Bash (run_in_background) + a Monitor-style poll. P2planned
v1.10.0 Web fetch + search tools WebFetch (URL → markdown → query) and WebSearch so the agent can look up docs/APIs mid-task. (Promoted from stretch — needed for a complete replacement.) P2planned
v1.11.0 Multimodal input Paste/attach images, and Read images / PDFs / notebooks into context via the provider's multimodal API. (Deferred in v1.1.0; promoted.) P2planned
v1.12.0 LSP — code intelligence Go-to-definition, find-references, hover, symbols via a language server — refactoring confidence beyond text search. P2planned
v1.13.0 Anthropic-native backend + thinking display A second ChatLLM backend talking Anthropic's API directly (alongside the OpenAI-compat one, selectable per slot): native prompt-cache control, server tools (web/computer-use), the thinking-budget API + an extended-thinking/reasoning display. Closes the closeable part of "model parity" (raw model quality stays whatever model you point at). P1planned
v1.14.0 Sandbox — filesystem / network restrictions Allow/deny lists for paths + network hosts on Bash (CC's sandbox model), with an explicit override. Pairs with v1.4.0 permissions. P2planned
v1.15.0 Cross-session memory Consolidate durable memory across sessions (CC's auto-memory / dream): the Advisor (v0.10.4) extended from in-session judgment to persistent .claude/-stored memory. P3planned
v1.16.0 Background & scheduled agents (long-tail) Background Bash + Monitor (folded from v1.9.0), plus scheduled / remote / multi-agent surfaces (cron, remote triggers, teammates) for full coverage of CC's agent-trigger tools. Niche; last before the lock. P3planned
v2.0.0 Claude Code parity lock Seal the v2 surface: a TUI coding agent at functional parity with Claude Code across the whole closeable surface — tools, project context, skills, permissions/plan, goal, hooks, MCP, web, multimodal, LSP, Anthropic-native backend + thinking, sandbox, cost, parallel, cross-session memory — all .claude/-compatible. SemVer re-locked for the 2.x line. Out of scope by nature (not a release): the model's raw quality (= whatever model the slot points at) and non-TUI surfaces (IDE extensions / desktop / web — a separate product track if ever wanted). P1target

v3 — vision

v3.x RL-framework compatibility (tunable system prompt) Make neutrix's agent loop usable as an RL environment so the system prompt (and policy) can be tuned by an agent-RL framework: a headless, seedable run interface (reset / step / reward hook) decoupled from the TUI — the v0.10.3 store-purity + FakeView swap-test are the groundwork — with the system prompt a first-class, externally-injected parameter. Captured now so v2 architecture choices keep the loop cleanly headless. P3vision

Shipped

v0.9.4 Heartbeat liveness indicator (+ bundled TaskCreate prompt fix) A single ● at the top of the message-area while the ContextManager is in AWAITING_LLM (● LLM), AWAITING_EXECUTOR (● tool: {name}), or CANCELLING (● cancelling…). The dot breathes trough→peak→trough on a 40-frame truecolor gradient at ~100 ms/tick with ±10% jitter — a 4.0-second cycle (~15 BPM, middle of the human resting-calm respiratory range). Reads the v0.9.3 state machine and store.pending_tool_calls; pure-renderer addition, no new store fields (14 split-point decisions, 11 original + 3 Phase-2 reopen on cycle period / smoothing / jitter after the first 720 ms implementation was rejected as "beating too fast"). Bundles a small user-reported fix: TaskCreate tool description rewritten to push full-upfront task planning. shipped
v0.9.3 Cancel as steer + ContextManager state machine Switches cancel cleanup from rollback (v0.9.2) to append [interrupted by user], so the LLM can be steered ("instead, just ls") with the prior tool intent in context. Reshapes the v0.9.2 Controller into a ContextManager that owns messages and drives the LLM/tool loop as an explicit state machine (IDLE / AWAITING_LLM / AWAITING_EXECUTOR / CANCELLING). Agent class is dissolved into CM. A pairing layer in llm.py repairs orphan tool_use blocks at API-send time. stream=False again — streaming re-enables in a later PRD. Legacy tui.py Textual app removed. shipped
v0.9.2 Esc / Ctrl+C cancellation + Executor / LLM / Controller decomposition Adds the in-process service-oriented architecture (Controller broadcasts cancel to LLM.stop() + Executor.cancel()). Esc cancels the in-flight turn; run_shell's subprocess gets tree-killed; conversation history rolls back atomically. OpenAIChatLLM switched back to stream=True so the SDK's AsyncStream.close() is what stop() hangs on. shipped
v0.9.1 Keyboard ergonomics — two-tap Ctrl+C / Ctrl+D, Ctrl+Z suspend, backslash line continuation Codex-style independent-timer arming for Ctrl+C and Ctrl+D; Ctrl+Z backgrounds the process via suspend_to_background; trailing-backslash + Enter inserts a newline. The keyboard half of "cancel and idle-state." shipped
v0.9.0 Lifecycle events + store reducer Adds llm_request_start / llm_request_end and a single ChatStore.apply(event) reducer. Makes the store the canonical "what's in flight" surface, unblocks v0.9.4 heartbeat and v0.10.1 FakeView. Invisible to the user. shipped
