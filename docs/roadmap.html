neutrix roadmap

A terminal agentic chat client for OpenAI-compatible LLM APIs.

This page is the user-visible map of where the project is going. Per CLAUDE.md, every PRD acceptance updates this page before implementation starts. Entries link to their PRDs under docs/PRDs/.

v1.0 destination: a TUI Claude-Code-shaped agentic client — cancel-as-steer, lifecycle-driven store, visibility parity, smart Advisor, conversation compaction, subagent dispatch, rewind / branch. Sealed surface at v1.0. Markdown rendering, cost tracking, web frontend, MCP, skills, hooks are post-v1.0 (v1.x).

Render order is by priority tier, then by logical sequence within each tier: P0 blocks daily use right now; P1 important for the v1.0 lock; P2 nice to have pre-v1.0; P3 deferred to post-v1.0 (v1.x).

P0 — daily-blocking / user-reported urgent

v0.9.5 LLM error surface (stalled / timeout / inline error) User-reported during v0.9.4 dogfooding: when the LLM API hangs silently, the heartbeat keeps breathing but neither neutrix nor the LLM is making progress, and the user has no signal until the SDK's ~10-minute default timeout fires. Two layers: ~50 s of silence → heartbeat palette swap to red + ● LLM (stalled) label (UI hint, no abort); ~300 s → background watchdog fires cm.cancel(reason='timeout'), transcript records [LLM timeout after 300s]. Per-slot tunable via a new llm_timeout_s YAML field (default 300 s); the stall hint is derived as max(10, timeout/6). Forward-compatible with v0.10.1 streaming (12 split-point decisions; 9 Alternative / 3 Follow CC; thresholds revised at acceptance for slow models). shipped
v0.9.6 Emergency mechanical /compact User-reported urgent. A slash command that drops the oldest 50 % of ContextManager.messages mechanically — no LLM summarization, no automatic trigger, no configurable ratio. Tool-round-boundary aware. The bridge that keeps users working until smart compaction lands at v0.10.5. (12 split-point decisions; 1 Follow CC / 8 Alternative / 3 No CC analog.) shipped
v0.9.7 Rewind / branch from prior turn User-reported urgent. /rewind N destructively drops the last N user turns (Follow CC) — tool-round-boundary safe — and prints a forward ↶ rewound notice (the append-only renderer can't restyle past turns, so the skeleton's retroactive [rewound] marker became a forward notice — split #7 scope change). Up/Down recall prior user turns into the draft, decoupled from rewind (recall only edits the buffer — split #2). Implemented + tested (suite 224); 4 split forks flagged for review. (10 split-point decisions.) P0 awaiting accept
v0.10.0 Subagent framework (Agent tool) User-reported urgent for v1.0 (context-explode structural fix). CC analog: the Agent tool that spawns a fresh-context worker, runs it to completion, returns the final assistant text. Each subagent owns its own ContextManager; the parent's context grows by one tool-result line, not by the subagent's full transcript. Visible in the parent transcript as a folded [subagent] block; cancellable via Esc. P0 skeleton

P1 — important for the v1.0 lock

v0.10.1 Re-enable streaming with cancel-as-steer Switch OpenAIChatLLM.stream_response back to stream=True (rolled back in v0.9.3). On Esc: close the stream eagerly, keep the partial assistant text in history, append [interrupted by user], let the LLM-side pairing layer repair any orphan tool_use. Closes the cancel arc. P1 skeleton
v0.10.2 Visibility parity — show what the LLM sees Renders the system prompt, tool schemas, and <system-reminder> injections in the transcript (folded). Adds a .claude/rules/visibility-parity.md rule, an LLMRoundBundle snapshot, a Ctrl+E fold/expand affordance, and an invariant test asserting no channel sent to the LLM is omitted from the user view. P1 skeleton
v0.10.3 TUIView as pure renderer + FakeView swap-test Closes the v0.7-arc onion refactor. View becomes a pure reader of ChatStore; residual writes (_tool_records folded-tray) move onto the store; a FakeView in tests/ proves the renderer is swappable. Architectural hygiene that makes the v0.10.4 Advisor's "store is the only mutator" assumption defensible. P1 skeleton
v0.10.4 Smart Advisor — third actor with LLM-driven task judgment A second LLM that watches the store and revises the task list with judgment instead of mechanical bookkeeping. Plugs into the v0.9.3 ContextManager event surface as an additional event consumer / cancel target. P1 skeleton
v0.10.5 Smart conversation compaction CC analog: manageConversation / /compact with LLM-summarized older turns. Builds on v0.9.6's mechanical /compact by adding an automatic threshold trigger and a summarizer call against the cut segment. Required before v1.0 — Advisor without compaction hits the cliff faster, not slower. P1 skeleton

v1.0 lock

v1.0.0 Sealed v1.0 surface Tag the cumulative result of v0.8.3–v0.10.5 as the v1.0 product. Freeze SemVer (post-v1.0 breaking changes require a major bump). Polish items (markdown rendering, cost / token tracking on /status, skills, MCP, hooks, web frontend, parallel / recursive subagents) all land as v1.x. P1 target

Shipped

v0.9.4 Heartbeat liveness indicator (+ bundled TaskCreate prompt fix) A single ● at the top of the message-area while the ContextManager is in AWAITING_LLM (● LLM), AWAITING_EXECUTOR (● tool: {name}), or CANCELLING (● cancelling…). The dot breathes trough→peak→trough on a 40-frame truecolor gradient at ~100 ms/tick with ±10% jitter — a 4.0-second cycle (~15 BPM, middle of the human resting-calm respiratory range). Reads the v0.9.3 state machine and store.pending_tool_calls; pure-renderer addition, no new store fields (14 split-point decisions, 11 original + 3 Phase-2 reopen on cycle period / smoothing / jitter after the first 720 ms implementation was rejected as "beating too fast"). Bundles a small user-reported fix: TaskCreate tool description rewritten to push full-upfront task planning. shipped
v0.9.3 Cancel as steer + ContextManager state machine Switches cancel cleanup from rollback (v0.9.2) to append [interrupted by user], so the LLM can be steered ("instead, just ls") with the prior tool intent in context. Reshapes the v0.9.2 Controller into a ContextManager that owns messages and drives the LLM/tool loop as an explicit state machine (IDLE / AWAITING_LLM / AWAITING_EXECUTOR / CANCELLING). Agent class is dissolved into CM. A pairing layer in llm.py repairs orphan tool_use blocks at API-send time. stream=False again — streaming re-enables in a later PRD. Legacy tui.py Textual app removed. shipped
v0.9.2 Esc / Ctrl+C cancellation + Executor / LLM / Controller decomposition Adds the in-process service-oriented architecture (Controller broadcasts cancel to LLM.stop() + Executor.cancel()). Esc cancels the in-flight turn; run_shell's subprocess gets tree-killed; conversation history rolls back atomically. OpenAIChatLLM switched back to stream=True so the SDK's AsyncStream.close() is what stop() hangs on. shipped
v0.9.1 Keyboard ergonomics — two-tap Ctrl+C / Ctrl+D, Ctrl+Z suspend, backslash line continuation Codex-style independent-timer arming for Ctrl+C and Ctrl+D; Ctrl+Z backgrounds the process via suspend_to_background; trailing-backslash + Enter inserts a newline. The keyboard half of "cancel and idle-state." shipped
v0.9.0 Lifecycle events + store reducer Adds llm_request_start / llm_request_end and a single ChatStore.apply(event) reducer. Makes the store the canonical "what's in flight" surface, unblocks v0.9.4 heartbeat and v0.10.1 FakeView. Invisible to the user. shipped
