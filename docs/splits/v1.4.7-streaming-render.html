v1.4.7 splits — live streaming render

Render assistant tokens live — completes the v0.10.1 deferral. User-asked "come here, not future".

Decided autonomously under the build-to-v2.0.0 /goal; design validated by the advisor (unify the mirror; strict channel separation; clear-in-the-commit-beat). Regression guards: the v0.10.1 cancel-keep + v0.10.5 shrink-cursor tests stay green.

Split points

1.One mirror, not two — retire _streaming_partialdecided
The risk
v0.10.1 stashed in-flight text on ContextManager._streaming_partial (for cancel-keep). A new render source (store.pending_assistant_text) would be a second mirror — two holders to keep in sync across 4 exit paths (the v0.10.1 reconcile-the-append-sites landmine, doubled).
Decision
Unify on store.pending_assistant_text (the single state holder — v0.10.3). v0.10.1 used a CM field because there was no render consumer; that inverts now. _call_llm routes token deltas to store.extend_assistant_stream; _do_cancel commits from store.pending_assistant_text; clear-pending replaces clear-stash on every path. Deletes the divergence rather than managing it.
2.Channel separation + commit timingdecided
Decision
Strictly disjoint channels. The pending preview renders ONLY in the _above_input live region (ephemeral, re-rendered); committed text renders ONLY via _render_record→console.print (append-only scrollback). No token batch ever flows through console.print (else permanent duplication). On finalize: _append_assistant_message(full) then clear_pending_assistant_text() with no await between — store mutations are synchronous and the watcher only runs at an await, so it never sees both the record and a stale preview (the v0.10.5 desync class).
3.Preview shapedecided
Decision
Pending holds the full partial; the preview shows the last STREAM_PREVIEW_LINES=8 lines (dim italic, … marks elided), so a long growing response can't overflow the bottom region. Commit/cancel use the full pending text, not the truncated view. The per-token extend→notify→invalidate is coalesced by prompt_toolkit (acceptable; tick-decoupling is a later optimization).
v1.4.7 live streaming render · 3 split points, decided autonomously (advisor-validated) · single store-owned pending source · disjoint preview/scrollback channels · clear-in-the-commit-beat. Completes the v0.10.1 deferral.
