Split points

1.Default mode — user-directed: autodecided
CC
default mode is interactive ask (a prompt per risky call); an auto mode uses an AI classifier.
Decision
Two modes, default auto (user: "allow all or auto, by default auto"). neutrix has no approve-dialog yet, so auto = allow normal ops but block clearly-destructive Bash via a deterministic heuristic (rm -rf, git push --force, dd, mkfs, fork-bombs, curl|sh, git reset --hard, …) with a block-with-notice telling the user/model to add an allow rule or switch to allow-all. allow-all (/allow) disables all checks. The CC LLM-classifier auto is a later refinement.
2.Plan mode — removeddecided
Decision
User-directed: removed. "no, no plan mode" / "remove plan mode". The read-only-gating plan mode is not built (and dropped from the band + lock). If wanted later the user will ask.
3.Rule source + shapedecided
Decision
Follow CC. permissions.{allow,deny,ask} rule lists merged from ~/.claude/settings.json + project .claude/settings.json + .claude/settings.local.json. Rules are ToolName or ToolName(glob) matched against the tool's primary arg (command for Bash, path for Read/Edit/Write/Glob, pattern for Grep). Precedence: deny > explicit-allow (overrides auto danger) > auto-danger > ask > allow.
4.Decision point + ask handlingdecided
Decision
The gate lives in Executor.dispatch_all before the to_thread dispatch (it holds policy + permission_mode; defaults allow-all-safe so existing tests are unaffected). A non-allow verdict yields a tool_finished with ok=False + a block reason — the model sees it and adapts. ask → block-with-notice (the interactive approve-dialog is deferred to the AskUserQuestion release, which builds the cross-thread input channel that would also upgrade this to a real prompt).
