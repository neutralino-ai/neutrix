v1.3.0 splits — skills + commands + hot-reload + custom agents

Markdown skills/commands from .claude/, /name dispatch, hot-reload, and custom subagent types. User #4.

Decided autonomously under the build-to-v2.0.0 /goal. CC survey (v2 kickoff): skills loader (skills/loadSkillsDir.ts), command framework (commands.ts), hot-reload watcher (skillChangeDetector.ts), agents (loadAgentsDir.ts).

Split points

1.Skill content model — tool vs slash commanddecided
CC
skills are PromptCommands invoked via a Skill tool (model-invocable) or /name; body is a prompt expanded inline.
Decision
User-invocable slash commands for v1.3.0: /<name> enqueues the markdown body (with $ARGUMENTS/$1.. substituted) as a user turn. A model-invocable Skill tool is deferred — the slash form is the high-value core and reuses the existing command dispatch.
2.Discovery dirs + override orderdecided
Decision
Follow CC. .claude/skills/<name>/SKILL.md + flat .claude/commands/*.md, scanned user (~/.claude) then project (project overrides on name collision). Frontmatter (name/description/argument-hint) parsed; name defaults to dir/file stem.
3.Hot-reload mechanism — file watcher vs polldecided
CC
chokidar FSWatch, 300 ms debounce, clears caches.
Decision
Alternative — mtime-signature poll (no new dependency). A 2 s background task compares a cheap signature (dir + *.md mtimes) and re-discovers only on change. Keeps neutrix lean (the same ethos as keeping the openai SDK over LiteLLM); a real watcher (watchdog) is a later option if 2 s feels slow.
4.Custom subagent typesdecided
Decision
Follow CC. .claude/agents/*.md → AgentDef (body = system prompt). The Agent tool reads them at dispatch time: subagent_type = general-purpose or a custom name; unknown → an error listing the available types. Extends v0.10.0 (general-purpose only) by passing the agent's system prompt to run_subagent.
